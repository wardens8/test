## CSS 简介

CSS 指层叠样式表，**C**ascading **S**tyle **S**heets，是一种用来为结构化文档（如 HTML 文档或 XML 应用）添加样式（字体、间距和颜色等）的计算机语言，文件扩展名为 `.css`。

样式定义如何显示 HTML 元素，通常存储在样式表中。把样式添加到 HTML 4.0 中，是为了解决内容与表现分离的问题。外部样式表可以极大提高工作效率，通常存储在 CSS 文件中，多个样式定义可层叠为一个。

## CSS 基础

三种引入方式

- 内联样式（Inline style）：在相关的标签内使用样式（style）属性书写样式

```html
<p style="color:sienna; margin-left:20px">这是一个段落。</p>
```

- 内部样式表（Internal style sheet）：直接在头部标签内进行书写样式

```html
<style>
  body {
    background-color: red;
  }
  p {
    color: blue;
  }
</style>
```

- 外部样式表（External style sheet）：先在外部新建 `index.css`文件，再在 html 文件中引入（type属性可写可不写，默认css）

```html
<link rel="stylesheet" type="text/css" href="index.css" />
```

- 多重样式优先级：样式表允许以多种方式规定样式信息。样式可以规定在单个的 HTML 元素中，在 HTML 页的头元素中，或在一个外部的 CSS 文件中。甚至可以在同一个 HTML 文档内部引用多个外部样式表。一般情况下，优先级如下：内联样式 > 内部样式 > 外部样式 > 浏览器默认样式

基础语法规范

- CSS 规则由两个主要的部分构成：选择器，一条或多条声明
- 选择器通常是您需要改变样式的 HTML 元素
- 每条声明由一个属性和一个值组成
- 属性（property）是您希望设置的样式属性（style attribute），每个属性有一个值，属性和值被冒号分开
- CSS 声明总是以分号 ; 结束，声明总以大括号 {} 括起来
- 为了让 CSS 可读性更强，你可以每行只描述一个属性

```css
p {
  color: red;
  text-align: center;
}
```

CSS 注释：用来解释代码的信息，浏览器会忽略注释内容

- 以 `/*` 开始, 以 `*/` 结束, 比较长的注释可以在 `/*` 和 `*/` 中分行写，长注释第一个字符缩进两个空格中加一个星号，更易于阅读
- 快捷键：`ctrl + /`

```css
/* 这是注释 */

/*
 * 这是一个较长注释。这是一个较长注释。这是一个较长注释。
 * 这是一个较长注释。这是一个较长注释。这是一个较长注释。
 */
```

## CSS 选择器

基础选择器：这类选择器直接匹配元素本身，是 CSS 最基础的用法

- 元素/标签选择器 `body { ... }`
- id 选择器 `#name { ... }`
- class 类选择器 `.name { ... }`
- 通配符选择器 `* { ... }`

复合选择器：通过符号组合多个基础选择器，匹配元素间的关系

- 并集选择器（,） `A, B { ... }`：匹配多个选择器对应的所有元素
- 交集选择器（无符号） `A.B { ... }`：同时满足多个条件的所有元素
- 子元素选择器（>） `A > B { ... }`：匹配 A 元素的直接子元素 B（一级嵌套）
- 后代选择器（空格） `A B { ... }`：匹配 A 元素下所有层级的 B 后代元素
- 相邻兄弟选择器（+） `A + B { ... }`：匹配 A 元素紧挨着的下一个同级别 B 元素
- 普通兄弟选择器（~） `A ~ B { ... }`：匹配 A 元素后面所有同级别 B 元素

属性选择器：匹配拥有特定属性/属性值的元素，灵活度高

- `[attr] { ... }` 匹配有 attr 属性的元素
- `[attr="value"] { ... }` 匹配有 attr 属性完全等于 value 的元素
- `[attr~="value"] { ... }` 匹配有 attr 属性包含独立单词 value 的元素
- `[attr|="value"] { ... }` 匹配有 attr 属性等于 value 或以 value- 开头的元素

伪类选择器：前缀为 `:`，匹配元素的动态状态、位置关系

- 状态伪类选择器
  - `:link { ... }` 匹配所有未被访问的状态，常用于超链接
  - `:visited { ... }` 匹配所有已被访问的状态（隐私原因只能改文字颜色），常用于超链接
  - `:hover { ... }` 匹配鼠标悬浮时的状态，所有元素都可用
  - `:active { ... }` 匹配鼠标按下时的状态，所有元素都可用
  - `:focus { ... }` 匹配被聚焦的表单元素

> 注意：超链接 4 个状态伪类「铁律顺序」：必须遵循 LVHA 原则，顺序错了会导致样式覆盖失效：`a:link` → `a:visited` → `a:hover` → `a:active`。`a:focus` 通常写在 `a:hover` 前面，避免焦点样式被 hover 覆盖（适配键盘导航），推荐顺序：`a:link` → `a:visited` → `a:focus` → `a:hover` → `a:active`

> 注意：非超链接的状态伪类（比如 `input:focus`、`input:hover`、`input:active`），无强制顺序，但建议 focus 优先（兼顾可访问性）

伪元素选择器：前缀为 `::`（单冒号也兼容，但推荐双冒号区分伪类），匹配元素的“虚拟子元素”

- `::before { content: ''; }` 在元素内容前面插入虚拟内容
- `::after { content: ''; }` 在元素内容后面插入虚拟内容
- `::first-letter { ... }` 表示元素中的第一个字母
- `::first-line { ... }` 表示元素中的第一行
- `::placeholder { ... }` 表示输入框元素中的占位文本
- `::selection { ... }` 表示文档中的高亮的部分（所选部分）
- `::marker { ... }` 表示列表项标记

`!important`：用于增加样式的权重

- `!important` 与优先级无关，但它与最终的结果直接相关，使用一个 `!important` 规则时，此声明将覆盖任何其他声明
- 重要说明：使用 `!important` 是一个坏习惯，应该尽量避免，因为这破坏了样式表中的固有的级联规则，使得调试找 bug 变得更加困难了。当两条相互冲突的带有 `!important` 规则的声明被应用到相同的元素上时，拥有更大优先级的声明将会被采用
- 使用建议：一定要优先考虑使用样式规则的优先级来解决问题而不是 `!important`，只有在需要覆盖全站或外部 CSS 的特定页面中使用 `!important`，永远不要在你的插件中使用 `!important`，永远不要在全站范围的 CSS 代码中使用 `!important`

选择器权重计算规则

- `!important` > ID 选择器 > 类/属性/伪类选择器 > 元素/伪元素选择器 > 通配符/继承，组合选择器的优先级是各部分优先级叠加

## CSS 核心样式属性

### CSS 单位

长度单位

- 绝对单位：`px` 像素，`cm` 厘米，`mm` 毫米，`in` 英寸，`pt` 点，`pc` 派卡
- 相对单位：`%` 百分比，`em` 相对当前元素的字体大小（若当前元素未设置字体大小，则继承父元素的字体大小，且会嵌套增加），`rem` 相对根元素（html）的字体大小，`vw` 相对视口宽度的百分比，`vh` 相对视口高度的百分比，`vmin` 相对视口宽度和高度中较小的那个百分比，`vmax` 相对视口宽度和高度中较大的那个百分比

颜色单位

- `currentColor`：表示当前元素的文本颜色
- `transparent`：表示完全透明
- `rgb(r, g, b)`：通过RGB值定义颜色，r是红色，g是绿色，b是蓝色，取值范围都是0-255
- `rgba(r, g, b, a)`：通过RGBA值定义颜色，r是红色，g是绿色，b是蓝色，取值范围都是0-255，a是不透明度，取值范围是0-1
- `hsl(h, s, l)`：通过HSL值定义颜色，h是色相，取值范围是0-360，s是饱和度，取值范围是0%-100%，l是明度，取值范围是0%-100%
- `hsla(h, s, l, a)`：通过HSLA值定义颜色，h是色相，取值范围是0-360，s是饱和度，取值范围是0%-100%，l是明度，取值范围是0%-100%，a是不透明度，取值范围是0-1
- `#rrggbb`：通过6位十六进制值定义颜色，rr是红色，gg是绿色，bb是蓝色，当数值相同时可简写为 `#rgb`
- `#rrggbbaa`：通过8位十六进制值定义颜色，rr是红色，gg是绿色，bb是蓝色，aa是不透明度，当数值相同时可简写为 `#rgba`
- 还有很多方式，此处只说常用方式，不再过多赘述

### 基本尺寸相关属性

- `width` 设置元素的宽度
- `max-width` 设置元素的最大宽度
- `min-width` 设置元素的最小宽度
- `height` 设置元素的高度
- `max-height` 设置元素的最大高度
- `min-height` 设置元素的最小高度

### 文本样式相关属性

- 字体相关
  - `font-family` 指定文本的字体系列：`family-name` 字体名称（可多个）
  - `font-size` 指定文本的字体大小
  - `font-weight` 指定字体的粗细：`normal` 默认值 400，`bold` 粗体 700，`bolder` 更粗，`lighter`更细，`number`定义由细到粗的字符
  - `font-style` 指定文本的字体样式：`normal` 默认值，`italic` 斜体，`oblique` 倾斜
  - `font-variant` 以小型大写字体或者正常字体显示文本：`normal` 默认值，`small-caps` 浏览器会显示小型大写字母的字体
  - `font` 在一个声明中设置所有的字体属性

```css
h1 {
  /* serif 衬线字体 sans-serif 非衬线字体 monospace 等宽字体 */
  font-family: 'Times New Roman', Times, serif, sans-serif, monospace;
  font-size: 16px;
  line-height: 24px;
  font-weight: bold;
  font-style: italic;
  font-variant: normal;
  font:
    italic normal bold 16px/24px 'Times New Roman',
    Times,
    serif,
    sans-serif,
    monospace;
}
/* font 可设置的属性是（按顺序）："font-style font-variant font-weight font-size/line-height font-family"。font-size和font-family的值是必需的。如果缺少了其他值，默认值将被插入，如果有默认值的话 */
```

- 文本排版
  - `color` 设置文本颜色
  - `line-height` 设置行高
  - `text-align` 对齐元素中的文本：`left` 默认值左边，`right` 右边，`center` 中间，`justify` 两端对齐
  - `vertical-align` 设置元素的垂直对齐：`baseline` 默认值基线，`sub` 下标，`super` 上标，`top` 顶部，`text-top` 文本顶部，`middle` 中部，`bottom` 底部，`text-bottom` 文本底部，`length` 用长度值指定，允许使用负值，`%` 用百分比指定，允许使用负值
  - `text-decoration` 向文本添加修饰：`none` 默认标准，`underline` 下划线，`overline` 上划线，`line-through` 中间线
  - `text-indent` 缩进元素中文本的首行
  - `direction` 设置文本方向：`ltr` 从左向右，`rtl` 从右向左
  - `letter-spacing` 设置字符间距：`normal` 默认，可直接填数字
  - `text-transform` 控制元素中的字母：`none` 默认标准，`capitalize`大写字母开头，`uppercase` 全部大写，`lowercase` 全部小写
  - `text-overflow` 设置文本溢出时的处理方式：`clip`：默认直接裁剪文本，`ellipsis` 显示省略号（仅限单行文本），`string`自定义文本（仅 Firefox 支持）
  - `white-space` 设置元素中空白的处理方式：`normal` 默认，`nowrap` 文本不会换行
  - `word-spacing` 设置字间距

```css
h1 {
  color: red;
  line-height: 24px;
  text-align: center;
  vertical-align: middle;
  text-decoration: underline;
  text-indent: 2em;
  text-shadow: 10px 10px 5px red;
  direction: ltr;
  letter-spacing: normal;
  text-transform: none;
  text-overflow: ellipsis;
  white-space: nowrap;
  word-spacing: 10px;
}
```

```css
/* CSS 设置文字单行显示的方法 */
.view-text {
  /* 阻止文本换行 */
  white-space: nowrap;
  /* 隐藏超出部分 */
  overflow: hidden;
  /* 显示省略号 */
  text-overflow: ellipsis;
}

/* CSS 设置文字多行显示的方法 */
.view-text {
  overflow: hidden;
  text-overflow: ellipsis;
  /* 弹性伸缩盒子模型显示 */
  display: -webkit-box;
  /* 限制在一个块元素显示的文本的行数 */
  -webkit-line-clamp: 2;
  /* 设置或检索伸缩盒对象的子元素的排列方式 */
  -webkit-box-orient: vertical;
}
```

- 补充：文本域拖拽 `resize` 属性：`both`横向纵向拖拽，`horizontal` 横向拖拽，`vertical` 纵向拖拽，`none` 不允许拖拽

```css
textarea {
  resize: none;
}
```

### 背景样式相关属性

- `background-color` 背景颜色：默认 transparent 透明
- `background-image` 背景图像
- `background-repeat` 背景平铺：`repeat` 默认向垂直和水平方向重复，`no-repeat` 不重复，`repeat-x` 水平重复，`repeat-y` 垂直重复
- `background-attachment` 背景固定：`scroll` 默认随着页面的滚动而滚动，`fixed` 不随着页面的滚动而滚动
- `background-position` 背景位置：可指定两个方向，第一个是 X 值，第二个是 Y 值
- `background` 简写属性，作用是将背景属性设置在一个声明中（各值之间用空格分隔，不分先后顺序）

```css
div {
  background-color: red;
  background-image: url(...);
  background-repeat: no-repeat;
  background-attachment: fixed;
  background-position: top left;
  background: #ffffff url('img_tree.png') no-repeat fixed top left;
}
```

### 列表与表格样式相关属性

- 列表样式
  - `list-style` 简写属性。用于把所有用于列表的属性设置于一个声明中
  - `list-style-type` 设置列表项标志的类型
  - `list-style-position` 设置列表中列表项标志的位置
  - `list-style-image` 将图像设置为列表项标志
- 表格样式
  - `border-collapse` 设置表格的边框是否被折叠成一个单一的边框或隔开：`collapse`
  - `text-align` 对齐元素中的文本：`left` 默认值左边，`right` 右边，`center` 中间
  - `vertical-align` 设置元素的垂直对齐
  - `border-spacing` 设置相邻单元格的边距：`length` 用长度值指定，允许使用负值，`%` 用百分比指定，允许使用负值
  - `display: table/table-cell` 将元素设置为表格，表格单元格，就可以使用表格相关属性

### 元素显示与隐藏相关属性

- `display` 显示：`block` 块元素，`inline` 内联元素，`inline-block` 行内块元素，`none` 隐藏
- `visibility` 可见性：`visible` 可见，`hidden` 隐藏
- `opacity`：透明度：0-1
- `overflow`溢出处理：`visible` 默认显示溢出，`hidden` 隐藏溢出，`scroll` 显示滚动条，`auto`根据内容是否溢出动态显示滚动条，溢出时显示，否则不显示

### 盒模型相关属性

所有 HTML 元素可以看作盒子，在 CSS 中，“box model”这一术语是用来设计和布局时使用。CSS 盒模型本质上是一个盒子，封装周围的 HTML 元素，它包括：Margin（外边距），Border（边框），Padding（内边距），Content（内容）。盒模型允许我们在其它元素和周围元素边框之间的空间放置元素

- `margin`：外边距，可以写 1-4 个值，可单独书写也可合并书写

```css
div {
  margin-top: 10px;
  margin-right: 10px;
  margin-bottom: 10px;
  margin-left: 10px;

  /* 所有的4个边距都是25px */
  margin: 25px;
  /* 上下边距为25px，左右边距为50px */
  margin: 25px 50px;
  /* 上边距为25px，左右边距为50px，下边距为75px */
  margin: 25px 50px 75px;
  /* 上边距为25px，右边距为50px，下边距为75px，左边距为100px */
  margin: 25px 50px 75px 100px;
  /* 设置水平居中 */
  margin: 0 auto;
}
```

- `padding`：内边距，可以写 1-4 个值，可单独书写也可合并书写

```css
div {
  padding-top: 10px;
  padding-right: 10px;
  padding-bottom: 10px;
  padding-left: 10px;

  /* 所有的4个边距都是25px */
  padding: 25px;
  /* 上下边距为25px，左右边距为50px */
  padding: 25px 50px;
  /* 上边距为25px，左右边距为50px，下边距为75px */
  padding: 25px 50px 75px;
  /* 上边距为25px，右边距为50px，下边距为75px，左边距为100px */
  padding: 25px 50px 75px 100px;
}
```

- `border`：边框，四个方向可单独书写也可合并书写
  - `border-width`：设置元素所有边框的宽度，可以写 1-4 个值，顺序同内外边距，可单独书写也可合并书写
  - `border-style`：设置元素所有边框的样式，可以写 1-4 个值，顺序同内外边距，可单独书写也可合并书写：`dotted` 定义点状，`dashed` 定义虚线，`solid` 定义实线，`double` 定义双线，`groove` 定义 3D 沟槽，`ridge` 定义 3D 脊边，`inset` 定义 3D 嵌入，`outset` 定义 3D 突出
  - `border-color`：设置元素所有边框中可见部分的颜色，可以写 1-4 个值，顺序同内外边距，可单独书写也可合并书写
  - `border`：用于把针对四个边的属性设置在一个声明。可以设置的属性分别（按顺序）：`border-width`，`border-style` 和 `border-color`
  - 不同方向的边框属性可以单独书写：
    - `border-top`、`border-top-width`、`border-top-style`、`border-top-color`
    - `border-right`、`border-right-width`、`border-right-style`、`border-right-color`
    - `border-bottom`、`border-bottom-width`、`border-bottom-style`、`border-bottom-color`
    - `border-left`、`border-left-width`、`border-left-style`、`border-left-color`

```css
div {
  border-width: 25px;
  border-width: 25px 50px;
  border-width: 25px 50px 75px;
  border-width: 25px 50px 75px 100px;

  border-style: solid;
  border-style: solid dotted;
  border-style: solid dotted dashed;
  border-style: solid dotted dashed double;

  border-color: red;
  border-color: red green;
  border-color: red green blue;
  border-color: red green blue yellow;

  border: 25px solid red;
  border-top: 25px solid red;
  border-right: 25px solid red;
  border-bottom: 25px solid red;
  border-left: 25px solid red;
}
```

- `outline`：轮廓，是绘制于元素周围的一条线，位于边框边缘的外围，可起到突出元素的作用，outline 并不会占盒子位置
  - `outline-color` 设置轮廓的颜色
  - `outline-style` 设置轮廓的样式：具体内容同 `border-style`
  - `outline-width` 设置轮廓的宽度
  - `outline-offset` 设置轮廓与边框的距离
  - `outline` 在一个声明中设置所有的轮廓属性

```css
input {
  outline-color: red;
  outline-style: solid;
  outline-width: 5px;
  outline-offset: 5px;
  outline: red solid 5px;
}
```

标准盒模型 vs 怪异盒模型

- 标准盒模型 `box-sizing: content-box`
  - 总宽度计算公式：`总元素的宽度=宽度+左边框+右边框+左填充+右填充`
  - 总高度计算公式：`总元素的高度=高度+上边框+下边框+上填充+下填充`
- 怪异盒模型 `box-sizing: border-box`，该模型属于 CSS3 盒子属性

设置 margin 的两个问题

- margin 塌陷问题
  - 原因：父元素和相邻的第一个/最后一个子元素的垂直 margin 重叠，子元素 margin 穿透父元素生效
  - 解决方案
    - 用 padding 代替 margin
    - 开启 BFC，给父元素加 `overflow: hidden`（最推荐，无副作用）
    - 开启 BFC（除 overflow，还可设 `display: flow-root`）
    - 给父元素加 1px 边框 `border-top: 1px solid transparent`
    - 给父元素加内边距 `padding: 1px`
- margin 重叠问题
  - 原因：两个相邻的块级元素，垂直 margin 不叠加，只取较大值生效（水平 margin 无此问题）
  - 解决方案
    - 只给一个元素设置 margin，另一个元素不设置
    - 给其中一个元素外层包一个容器，开启 BFC
    - 将其中一个元素转为行内块元素 `display: inline-block`

> BFC，Block Formatting Context，块级格式化上下文，是一个在网页渲染过程中重要的概念。它决定了元素如何对其内容进行布局，以及与其他元素之间的关系和相互作用。BFC 可以看作是一个独立的容器，容器内的元素布局不会影响外部元素，反之亦然。常见的触发条件包括使用 `display: flow-root` 或 `overflow: hidden` 等属性，浮动也会开启 BFC。理解 BFC 对于解决 CSS 中的布局问题（如边距重叠、浮动定位等）至关重要。

## CSS 三大特性

层叠性

- 给相同选择器设置相同的样式，此时一个样式就会覆盖（层叠）另一个冲突的样式
- 层叠性主要解决样式冲突的问题
- 层叠性原则：样式冲突，后面的样式会覆盖前面的样式；样式不冲突，不会层叠

继承性

- 设置给祖先元素的样式，会同时应用在其后代元素上，这被称为样式的继承
- 继承的存在大大简化了样式的编写，如文本、字体等属性会继承父元素的样式，但是也有某些属性不能被继承，比如布局、背景等相关属性
- 可以使用 `inherit` 关键字显式地设置一个属性应从父元素继承其值

优先级

- 当同一个元素指定多个选择器，就会有优先级产生
- 选择器越具体，优先级越高。选择器相同，后写的样式会层叠前写的样式
  - 继承权重：0,0,0,0
  - 通配符选择器权重：0,0,0,0
  - 元素选择器权重：0,0,0,1
  - 类选择器、属性选择器、伪类选择器权重：0,0,1,0
  - ID 选择器权重：0,1,0,0
  - 内联样式权重：1,0,0,0
  - `!important` 权重：1,0,0,0,0
  - 注意：权重会叠加，但是不会进位

## CSS 三大基础布局

### 普通流（标准流）布局

- 正常布局，一部分一部分进行书写

### 浮动（float）布局

- 浮动：元素的水平方向浮动，意味着元素只能左右移动而不能上下移动。一个浮动元素会尽量向左或向右移动，直到它的外边缘碰到包含框或另一个浮动框的边框为止。浮动元素之后的元素将围绕它，浮动元素之前的元素将不会受到影响，如果图像是右浮动，下面的文本流将环绕在它左边。如果你把几个浮动的元素放到一起，如果有空间的话，它们将彼此相邻。可以填入的内容：`left` 向左，`right` 向右
- 注意：设置浮动的行内元素会变为行内块，可以设置宽度和高度

```css
/* 浮动 */
.name {
  float: left;
  width: 110px;
  height: 90px;
  margin: 5px;
}
```

- 浮动溢出问题
  - 核心原因：元素开启浮动（float: left/right）后，会脱离标准文档流，父元素无法感知浮动子元素的高度，导致父元素高度为 0，后续元素错位
- 清除浮动
  - 父元素添加固定高度（不推荐，不适配动态内容）
  - 开启 BFC：父元素添加 `overflow: hidden` 或 `display: flow-root`
  - 额外标签法也称为隔墙法，是 W3C 推荐的做法。元素浮动之后，周围的元素会重新排列，为了避免这种情况，使用 clear 属性。在浮动元素末尾添加一个空的标签，例如`<div style="clear:both"></div>`。clear 属性指定元素两侧不能出现浮动元素。可以填入的内容：`left` 左侧，`right` 右侧，`both` 全部
  - 伪元素清除法：父元素添加 after 伪元素
  - 双伪元素清除法：父元素添加 before after 双伪元素

```css
/* 伪元素清除法 */
.clearfix::after {
  content: '';
  display: block;
  clear: both; /* 清除左右浮动 */
  visibility: hidden; /* 隐藏伪元素 */
  height: 0;
}
.clearfix {
  *zoom: 1; /* 兼容IE低版本 */
}
```

```css
/* 双伪元素清除法 */
.clearfix::before,
.clearfix::after {
  content: '';
  display: table;
}
.clearfix::after {
  clear: both;
}
.clearfix {
  *zoom: 1;
}
```

### 定位（position）布局

- `static` 默认没有定位，遵循正常的文档流对象
- `relative` 相对定位（占原位置）：相对定位的元素的定位是相对其正常位置
- `absolute` 绝对定位（不占原位置）：绝对定位的元素的位置相对于最近的已定位父级元素，如果元素没有已定位的父级元素，那么它的位置相对于 html【子绝父相】
- `fixed` 固定定位（不占原位置）：元素的位置相对于浏览器可视窗口是固定位置，即使窗口是滚动的它也不会移动
- `sticky` 粘性定位（占原位置）：sticky 英文字面意思是粘，粘贴，所以可以把它称之为粘性定位。sticky 是基于用户的滚动位置来定位
- 注意：设置定位的行内元素会变为行内块，可以设置宽度和高度

```css
/* static */
.name {
  position: static;
}
/* relative */
.name {
  position: relative;
  top: 50px;
  left: 50px;
}
/* absolute */
.name {
  position: absolute;
  top: 50px;
  left: 50px;
}
/* fixed */
.name {
  position: fixed;
  top: 50px;
  left: 50px;
}
/* sticky */
.name {
  position: sticky;
  top: 50px;
  left: 50px;
}
```

- 利用定位实现居中效果
  - 开启定位后，左右的距离 = left + margin-left + content + margin-right + right = width
  - 开启定位后，上下的距离 = top + margin-top + content + margin-bottom + bottom = height
  - 将定位的距离全部为0时，元素可以使用auto来居中

```css
/* 父元素 */
.parent {
  position: relative;
  width: 200px;
  height: 200px;
  background-color: red;
}
/* 子元素 */
.child {
  position: absolute;
  top: 0;
  bottom: 0;
  left: 0;
  right: 0;
  margin: auto;
  width: 100px;
  height: 100px;
  background-color: blue;
}
```

- 定位层级错乱问题
  - 核心原因：元素开启定位（position: relative/absolute/fixed）后，默认按渲染顺序排列层级，后渲染的元素覆盖先渲染的，导致需要显示的元素被遮挡
  - 万能解决方案
    - 核心属性：`z-index`，只对「开启定位的元素」生效，值越大，层级越高
    - 基础用法：`z-index: 1; z-index: 10;`（数值无上限，推荐用 10 的倍数分层）
    - 补充注意：父元素层级再高，也无法覆盖子元素的层级（子元素层级相对于父元素）
  - 避坑点
    - 不要用 `z-index: 999999` 这种极端值，后续难以维护
    - 未开启定位的元素，`z-index` 无效

# CSS3

CSS3 是最新的 CSS 标准，用于控制网页的样式和布局。

## CSS3 新增选择器

新增属性选择器

- `[attr^="value"] { ... }` 匹配有 attr 属性以 value 开头的元素
- `[attr$="value"] { ... }` 匹配有 attr 属性以 value 结尾的元素
- `[attr*="value"] { ... }` 匹配有 attr 属性包含 value 的元素

新增伪类选择器

- 状态伪类选择器
  - `:enabled { ... }` 匹配可用状态的表单元素
  - `:disabled { ... }` 匹配被禁用的表单元素（带 disabled 属性）
  - `:checked { ... }` 匹配被选中的单选框或复选框
  - `:selected { ... }` 匹配被选中的下拉列表选项
  - `:required { ... }` 匹配设置了 required 属性的表单元素
- 结构伪类选择器
  - `:root { ... }` 匹配根元素（相当于html），常用于设置全局样式
  - `:empty { ... }` 匹配没有子元素（包括文本）的空元素
  - `:first-child { ... }` 匹配其父元素的第一个子元素
  - `:last-child { ... }` 匹配其父元素的最后一个子元素
  - `:nth-child(n) { ... }` 匹配其父元素的第n个子元素，n可填数字/odd奇数/even偶数/n全部
  - `:nth-last-child(n) { ... }` 匹配其父元素的倒数第n个子元素
  - `:only-child { ... }` 匹配其父元素的唯一的子元素
  - `:first-of-type { ... }` 匹配其父元素中同类型标签的第一个子元素
  - `:last-of-type { ... }` 匹配其父元素中同类型标签的最后一个子元素
  - `:nth-of-type(n) { ... }` 匹配其父元素中同类型标签的第n个（比 :nth-child 更精准，推荐优先用）
  - `:nth-last-of-type(n) { ... }` 匹配其父元素中同类型标签的倒数第n个
  - `:only-of-type { ... }` 匹配其父元素中同类型标签唯一的子元素
  - 结构伪类选择器一般会复合后代选择器或子元素选择器使用以提升性能
- 否定伪类选择器
  - `A:not(B) { ... }` 匹配所有除了B类元素的A类元素（A和B可以是任意选择器）
- 目标伪类选择器
  - `:target { ... }` 匹配当前活动的目标元素（URL包含该元素的id时生效）

## CSS3 背景、边框、渐变

CSS3 背景

- `background-image` 添加背景图片，不同的背景图像和图像用逗号隔开，所有的图片中显示在最顶端的为第一张
- `background-size` 指定背景图像的大小尺寸：可以直接写大小，`contain` 缩放（背景会不全），`cover` 铺满（图片会不全）
- `background-origin` 指定背景图像定位的位置区域：`padding-box` 默认、`border-box`、`content-box`
- `background-clip` 指定背景图片从指定位置开始绘制：`border-box` 默认、`padding-box`、`content-box`

```css
.name {
  background-image: url(img_flwr.gif), url(paper.gif);
  background-size: 80px 60px;
  background-origin: padding-box;
  background-clip: border-box;
}
```

CSS3 边框

- `border-radius` 用于创建圆角：该属性可以填入 1-4 个值，如果你在 `border-radius` 属性中只指定一个值，那么将生成 4 个圆角
- `box-shadow` 用来添加盒子阴影：`box-shadow: h-shadow v-shadow blur spread color inset;`
- `border-image` 使用图像创建一个边框

```css
.name {
  border-radius: 10px;
  box-shadow: 10px 10px 5px red;

  /* 第一个值为左上角与右下角，第二个值为右上角与左下角 */
  border-radius: 15px 50px;
  /* 第一个值为左上角，第二个值为右上角和左下角，第三个值为右下角 */
  border-radius: 15px 50px 30px;
  /* 第一个值为左上角，第二个值为右上角，第三个值为右下角，第四个值为左下角 */
  border-radius: 15px 50px 30px 5px;
}
```

CSS3 渐变

- 线性渐变 `Linear Gradients`：方向+颜色
  - 方向：默认从上向下/向上/向左/向右/对角方向
  - 颜色：输入多个颜色进行渐变
- 径向渐变 `Radial Gradients`
  - 形状：默认是 ellipse 椭圆形，circle 圆形
  - 颜色：输入多个颜色进行渐变

```css
/* 线性渐变 */
.name {
  /* 一般渐变 */
  background-image: linear-gradient(red, yellow, green);
  /* 带方向渐变 */
  background-image: linear-gradient(to right, red, yellow, green);
  /* 对角渐变 */
  background-image: linear-gradient(to bottom right, red, yellow, green);
  /* 带角度渐变 */
  background-image: linear-gradient(-30deg, red, yellow);
  /* 带透明度渐变 */
  background-image: linear-gradient(
    to right,
    rgba(255, 0, 0, 0),
    rgba(255, 0, 0, 1)
  );
  /* 重复渐变 */
  background-image: repeating-linear-gradient(red, yellow 10%, green 20%);
}
```

```css
/* 径向渐变 */
.name {
  /* 一般渐变 */
  background-image: radial-gradient(red, yellow, green);
  /* 颜色不均匀渐变 */
  background-image: radial-gradient(red 5%, yellow 15%, green 60%);
  /* 不同形状渐变 */
  background-image: radial-gradient(ellipse, red, yellow, green);
  /* 带有不同尺寸大小关键字的径向渐变 */
  background-image: radial-gradient(
    closest-side at 60% 55%,
    red,
    yellow,
    black
  );
  background-image: radial-gradient(
    farthest-side at 60% 55%,
    red,
    yellow,
    black
  );
  /* 重复渐变 */
  background-image: repeating-radial-gradient(red, yellow 10%, green 15%);
}
```

## CSS3 2D/3D 转换

> 该部分属性需要兼容性处理

2D 转换：`transform` 属性可以让某个元素改变形状，大小和位置

- 平移
  - `translate(x,y)` 定义 2D 平移，沿着 X、Y 轴移动元素
  - `translateX(n)` 定义 2D 平移，沿着 X 轴移动元素
  - `translateY(n)` 定义 2D 平移，沿着 Y 轴移动元素
- 旋转
  - `rotate(angle)` 定义 2D 旋转，在参数中规定角度进行旋转，deg度，turn圈
- 缩放
  - `scale(x,y)` 定义 2D 缩放，改变元素沿着 X、Y 轴的缩放倍数
  - `scaleX(n)` 定义 2D 缩放，改变元素沿着 X 轴的缩放倍数
  - `scaleY(n)` 定义 2D 缩放，改变元素沿着 Y 轴的缩放倍数
- 倾斜
  - `skew(x-angle,y-angle)` 定义 2D 倾斜转换，沿着 X、Y 轴
  - `skewX(angle)` 定义 2D 倾斜转换，沿着 X 轴
  - `skewY(angle)` 定义 2D 倾斜转换，沿着 Y 轴
- 自定义变换
  - `matrix(scaleX, skewY, skewX, scaleY, translateX, translateY)` 将 2D 变换方法合并成一个。matrix 方法有六个参数，包含旋转，缩放，移动（平移）和倾斜功能
- 其他
  - `transform-origin` 定义 3D 转换元素的基点（位置），可以写方位，也可以写具体指
  - `transform` 属性可以同时添加多个转换，用**空格**隔开即可，注意书写顺序：位移、旋转、缩放（因为会影响基准点）

```css
.name {
  transform: translate(100px, 100px);
  transform: translateX(100px);
  transform: translateY(100px);

  transform: rotate(30deg);

  transform: scale(2, 3);
  transform: scale(2);
  transform: scaleX(3);
  transform: scaleY(3);

  transform: skew(30deg, 20deg);
  transform: skewX(30deg);
  transform: skewY(30deg);

  transform-origin: top left;
  transform-origin: 80px 80px;

  transform: translate(100px, 100px) rotate(45deg) scale(2);
}
```

- 巧妙利用 translate 和绝对定位实现居中：`transform: translate(-50%, -50%);`

```css
.name {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
}
```

3D 转换：`transform` 属性可以让某个元素改变形状，大小和位置

- 透视
  - `perspective(n)` 定义 3D 转换元素的透视视图，该属性需要加在父元素上，子元素才能有 3D 效果，n 为透视距离，200-1000 之间为佳
  - `transform-style` 规定被嵌套元素如何在 3D 空间中显示：`flat` 默认不开启 3D 立体空间、`preserve-3d` 开启 3D 立体空间
- 平移
  - `translate3d(x,y,z)` 定义 3D 平移，沿着 X、Y、Z 轴移动元素
  - `translateX(n)` 定义 3D 平移，沿着 X 轴移动元素
  - `translateY(n)` 定义 3D 平移，沿着 Y 轴移动元素
  - `translateZ(n)` 定义 3D 平移，沿着 Z 轴移动元素
- 旋转（左手转则）
  - `rotate3d(x,y,z,angle)` 定义 3D 旋转，在参数中规定角度进行 3D 旋转，前三个代表不同方向的数量，最后一个代表旋转角度
  - `rotateX(angle)` 定义 3D 旋转，在参数中规定角度沿 X 轴进行旋转
  - `rotateY(angle)` 定义 3D 旋转，在参数中规定角度沿 Y 轴进行旋转
  - `rotateZ(angle)` 定义 3D 旋转，在参数中规定角度沿 Z 轴进行旋转
- 缩放
  - `scale3d(x,y,z)` 定义 3D 缩放，改变元素沿着 X、Y、Z 轴的缩放倍数
  - `scaleX(n)` 定义 3D 缩放，改变元素沿着 X 轴的缩放倍数
  - `scaleY(n)` 定义 3D 缩放，改变元素沿着 Y 轴的缩放倍数
  - `scaleZ(n)` 定义 3D 缩放，改变元素沿着 Z 轴的缩放倍数
- 自定义变换（很少用）
  - `matrix3d(n,n,n,n,n,n,n,n,n,n,n,n,n,n,n,n)` 定义 3D 转换，使用 16 个值的 4x4 矩阵

```css
/* 3D 转换：用法与 2D 转换相差不大 */
.name {
  transform: translate3d(100px, 100px, 100px);
  transform: translateX(100px);
  transform: translateY(100px);
  transform: translateZ(100px);

  transform: rotate3d(1, 1, 1, 30deg);
  transform: rotateX(30deg);
  transform: rotateY(30deg);
  transform: rotateZ(30deg);

  transform: scale3d(2, 3, 4);
  transform: scaleX(3);
  transform: scaleY(3);
  transform: scaleZ(3);
}
```

CSS3 过渡：`transition` 属性可以让某个元素从一种样式逐渐改变为另一种的效果

- 要实现这一点，必须规定两项内容：指定要添加效果的 CSS 属性，指定效果的持续时间。
- `transition-property` 规定应用过渡的 CSS 属性的名称：可以直接写属性，也可以写 all 表示全部
- `transition-duration` 定义过渡效果花费的时间。默认是 0
- `transition-timing-function` 规定过渡效果的时间曲线
  - `ease`：默认值，缓慢加速，然后缓慢减速的过渡效果
  - `linear`：匀速过渡效果
  - `ease-in`：缓慢加速的过渡效果
  - `ease-out`：缓慢减速的过渡效果
  - `ease-in-out`：先缓慢加速，再缓慢减速的过渡效果
  - `steps(n)` 分步动画，可以设置步数
  - `cubic-bezier(x1,y1,x2,y2)`：自定义的贝塞尔曲线函数，通过四个值来定义曲线的控制点
- `transition-delay` 规定过渡效果何时开始。默认是 0
- `transition` 简写属性，用于在一个属性中设置四个过渡属性；也可以逗号分隔同时设置多个属性

```css
.name {
  transition-property: width;
  transition-duration: 1s;
  transition-timing-function: linear;
  transition-delay: 2s;

  transition: width 1s linear 2s;

  transition:
    width 1s linear 2s,
    height 1s linear 2s;
}
```

## CSS3 动画

> 该部分属性需要兼容性处理

动画是使元素从一种样式逐渐变化为另一种样式的效果。您可以改变任意多的样式任意多的次数。请用百分比来规定变化发生的时间，或用关键词 "from" 和 "to"，等同于 0% 和 100%。0% 是动画的开始，100% 是动画的完成。为了得到最佳的浏览器支持，您应该始终定义 0% 和 100% 选择器。

- @keyframes 规则：@keyframes 规则是创建动画，规则内指定一个 CSS 样式和动画将逐步从目前的样式更改为新的样式。

```css
@keyframes myfirst {
  0% {
    background: red;
  }
  100% {
    background: yellow;
  }
}
/* Safari 与 Chrome */
@-webkit-keyframes myfirst {
  0% {
    background: red;
  }
  100% {
    background: yellow;
  }
}
```

- 当在 @keyframes 创建动画，把它绑定到一个选择器，否则动画不会有任何效果。指定至少这两个 CSS3 的动画属性绑定向一个选择器：
  - `animation-name` 规定动画的名称
  - `animation-duration` 规定动画的时长
  - `animation-timing-function` 规定动画的时间曲线
    - `ease`：默认值，缓慢加速，然后缓慢减速的过渡效果
    - `linear`：匀速过渡效果
    - `ease-in`：缓慢加速的过渡效果
    - `ease-out`：缓慢减速的过渡效果
    - `ease-in-out`：先缓慢加速，再缓慢减速的过渡效果
    - `steps(n)` 分步动画，可以设置步数
    - `cubic-bezier(n,n,n,n)`：自定义的贝塞尔曲线函数，通过四个值来定义曲线的控制点，分别对应 x1、y1、x2、y2
  - `animation-delay` 规定动画的延时时间
  - `animation-iteration-count` 规定动画的播放次数：`infinite` 表示无限循环
  - `animation-direction` 规定动画的播放方向：`normal` 正常播放，`reverse` 反向播放，`alternate` 交替播放，`alternate-reverse` 反向交替播放
  - `animation-fill-mode` 设置动画结束后的样式：`none` 不改变默认行为，`forwards` 动画完成定格，`backwards` 动画开始定格，`both` 向前和向后填充模式都被应用
  - `animation-play-state` 设置动画是运行还是暂停：`running` 默认值，动画正在运行，`paused` 动画暂停
  - `animation` 简写属性：用于设置上述八个动画属性，前两个必须要有。当然可以书写多个动画，逗号分隔

```css
div {
  animation-name: myfirst;
  animation-duration: 5s;
  animation-timing-function: linear;
  animation-delay: 2s;
  animation-iteration-count: infinite;
  animation-direction: alternate;
  animation-fill-mode: forwards;
  animation-play-state: running;
  animation: myfirst 5s linear 2s infinite alternate forwards running;

  animation: myfirst 5s;
}
```

## CSS3 鼠标指针

`cursor` 属性可以控制鼠标指针悬浮在元素上时显示的样式，核心作用是给用户直观的交互提示

- 可以填入一个值，用来定义鼠标指针
  - `default` 默认箭头（系统默认样式）：用于普通文本、无交互的元素
  - `pointer` 手型（最常用）：用于按钮、链接、可点击元素
  - `text` 文本输入光标：用于输入框、可编辑文本
  - `move` 十字移动光标：用于可拖拽的元素（如弹窗、滑块）
  - `not-allowed` 禁止/不可用光标（带斜杠的圆）：用于禁用的按钮、无权限操作的元素
  - `wait` 加载中光标（转圈/沙漏）：用于页面加载、异步请求过程中
  - `help` 帮助光标（箭头+问号）：用于帮助提示按钮、说明文本
  - `progress` 加载中（箭头+转圈）：用于后台加载但仍可交互的场景
  - `crosshair` 十字准星：用于绘图工具、选区定位
  - `grab/grabbing` 可抓取/已抓取：用于拖拽列表、滚动容器
  - `zoom-in/zoom-out` 放大/缩小：用于图片缩放、地图缩放按钮
  - `url()` 自定义图标
- 也可以填入多个值，支持多值回退，如果第一个值不支持，会用第二个

```css
img {
  cursor: default;
  cursor: pointer;
  cursor: text;
  cursor: move;
  cursor: not-allowed;
  cursor: wait;
  cursor: help;
  cursor: progress;
  cursor: crosshair;
  cursor: grab;
  cursor: grabbing;
  cursor: zoom-in;
  cursor: zoom-out;
  cursor: url(custom-cursor.png);

  cursor: url(custom-cursor.png), pointer;
}
```

## CSS3 滤镜属性、计算属性

> 该部分也需要添加浏览器标识

`filter` 属性定义了元素的可视效果

- `none`：默认值，没有效果。
- `blur(px)` 高斯模糊：给图像设置高斯模糊。如果没有设定值，则默认是 0
- `brightness(%)` 亮度：给图片应用一种线性乘法，使其看起来更亮或更暗。如果值是 0%，图像会全黑。值是 100%，则图像无变化。其他的值对应线性乘数效果。值超过 100%也是可以的，图像会比原来更亮。如果没有设定值，默认是 1
- `contrast(%)` 对比度：调整图像的对比度。值是 0%的话，图像会全黑。值是 100%，图像不变。值可以超过 100%，意味着会运用更低的对比。若没有设置值，默认是 1
- `saturate(%)` 饱和度：转换图像饱和度。值定义转换的比例。值为 0%则是完全不饱和，值为 100%则图像无变化。其他值，则是效果的线性乘子。超过 100%的值是允许的，则有更高的饱和度。 若值未设置，值默认是 1
- `opacity(%)` 透明度：转化图像的透明程度。值定义转换的比例。值为 0%则是完全透明，值为 100%则图像无变化。值在 0%和 100%之间，则是效果的线性乘子，也相当于图像样本乘以数量。 若值未设置，值默认是 1。该函数与已有的 opacity 属性很相似，不同之处在于通过 filter，一些浏览器为了提升性能会提供硬件加速
- `grayscale(%)` 灰度：将图像转换为灰度图像。值定义转换的比例。值为 100%则完全转为灰度图像，值为 0%图像无变化。值在 0%到 100%之间，则是效果的线性乘子。若未设置，值默认是 0
- `drop-shadow(h-shadow v-shadow blur spread color)` 阴影：给图像设置一个阴影效果。阴影是合成在图像下面，可以有模糊度的，可以以特定颜色画出的遮罩图的偏移版本。注意：前两个值是必须写的，后三个值可写可不写，尤其注意第四个值在 Chrome、Firefox 等主流浏览器暂不支持该参数（写了也不会生效）
- `hue-rotate(deg)` 色相旋转：给图像应用色相旋转。"angle"一值设定图像会被调整的色环角度值。值为 0deg，则图像无变化。若值未设置，默认值是 0deg。该值虽然没有最大值，超过 360deg 的值相当于又绕一圈
- `invert(%)` 反相：反转输入图像。值定义转换的比例。100%的价值是完全反转。值为 0%则图像无变化。值在 0%和 100%之间，则是效果的线性乘子。 若值未设置，值默认是 0
- `sepia(%)` 深褐色：将图像转换为深褐色。值定义转换的比例。值为 100%则完全是深褐色的，值为 0%图像无变化。值在 0%到 100%之间，则是效果的线性乘子。若未设置，值默认是 0

```css
.name {
  /* 高斯模糊 */
  filter: blur(5px);
  /* 亮度 */
  filter: brightness(200%);
  /* 对比度 */
  filter: contrast(200%);
  /* 饱和度 */
  filter: saturate(800%);
  /* 透明度 */
  filter: opacity(30%);
  /* 灰度 */
  filter: grayscale(100%);
  /* 阴影 */
  filter: drop-shadow(8px 8px 10px red);
  /* 色相旋转 */
  filter: hue-rotate(90deg);
  /* 反相 */
  filter: invert(100%);
  /* 深褐色 */
  filter: sepia(100%);
}
```

计算属性 `calc` 函数，在声明 CSS 属性值时执行一些计算

```css
.name {
  width: calc(100% - 20px);
  height: calc(100% - 20px);
}
```

## CSS3 现代布局（Flex + Grid）

### 流式布局（百分比布局）

- 流式布局，就是百分比布局，也称非固定像素布局
- 通过盒子的宽度设置成百分比来根据屏幕的宽度来进行伸缩，不受固定像素的限制，内容向两侧填充
- 流式布局方式是移动 web 开发使用的比较常见的布局方式
  - max-width 最大宽度（max-height 最大高度）
  - min-width 最小宽度（min-height 最小高度）
- 其实流式布局和 PC 布局相差不大，只是一些地方用百分比来表示了

### Flex 弹性布局（Flexible Box）

弹性盒子是 CSS3 的一种新的布局模式，是一种当页面需要适应不同的屏幕大小以及设备类型时确保元素拥有恰当的行为的布局方式。引入弹性盒布局模型的目的是提供一种更加有效的方式来对一个容器中的子元素进行排列、对齐和分配空白空间。

- 用于父元素
  - `display` 指定 HTML 元素盒子类型：`flex` 块级弹性盒子、`inline-flex` 行内弹性盒子
  - `flex-direction` 指定弹性容器中子元素的排列方式：`row` 默认横向左对齐，`row-reverse` 横向右对齐，`column` 纵向排列，`column-reverse` 反转纵向排列
  - `flex-wrap` 设置弹性盒子的子元素超出父容器时是否换行：`nowrap` 默认不换行、`wrap` 换行、`wrap-reverse` 反向换行
  - `flex-flow` 是 `flex-direction` 和 `flex-wrap` 两个属性的简写
  - `justify-content` 设置弹性盒子元素在主轴（横轴）方向上的对齐方式：`flex-start` 默认从头部开始、`flex-end` 从尾部开始、`center` 居中、`space-between` 两边对齐靠边、`space-around` 两边对齐平分边距、`space-evenly` 两边对齐平分间距
  - `align-items` 设置弹性盒子元素在侧轴（纵轴）方向上的对齐方式（单行）：`stretch` 默认拉伸、`flex-start` 从头部开始、`flex-end` 从尾部开始、`center` 居中、`baseline` 基线
  - `align-content` 设置弹性盒子元素在侧轴（纵轴）方向上的对齐方式（多行）：`flex-start` 默认从头部开始、`flex-end` 从尾部开始、`center` 居中、`space-between` 两边对齐靠边、`space-around` 两边对齐平分边距、`space-evenly` 两边对齐平分间距、`stretch` 拉伸
- 用于子元素
  - `flex-basis` 设置弹性盒子的子元素的基础尺寸，根据主轴的方向来定义
  - `flex-shrink` 设置弹性盒子的子元素的收缩系数，如何分配空间，缩小空间
  - `flex-grow` 设置弹性盒子的子元素的放大系数，如何分配空间，放大空间
  - `flex` 设置弹性盒子的子元素如何分配空间，分配剩余空间（上述三个属性的简写grow shrink basis）
  - `order` 设置弹性盒子的子元素排列顺序
  - `align-self` 设置单个弹性盒子元素在侧轴（纵轴）方向上的对齐方式：`stretch` 默认拉伸、`flex-start` 从头部开始、`flex-end` 从尾部开始、`center` 居中、`baseline` 基线

```css
.name {
  display: flex;
  flex-direction: row;
  flex-wrap: wrap;
  flex-flow: row wrap;
  justify-content: space-between;
  align-items: center;
  align-content: space-between;
}

.name .item {
  flex-basis: auto;
  flex-shrink: 1;
  flex-grow: 1;
  flex: 1; /* 1 1 0% */
  order: 1;
  align-self: center;
}
```

### Grid 网格布局（Grid Layout）

Grid 网格布局是一组相交的水平线和垂直线，它定义了网格的列和行。CSS 提供了一个基于网格的布局系统，带有行和列，可以让我们更轻松地设计网页，而无需使用浮动和定位。

网格的使用

- 用于父元素
  - `display` 指定 HTML 元素盒子类型：`grid`/`inline-grid` 块级网格布局、`subgrid` 子网格布局
  - `grid-template-rows` 指定网格布局中行的大小，以及网格布局中设置行的数量
  - `grid-template-columns` 指定网格布局中列的大小，以及网格布局中设置列的数量
  - `row-gap` 指定网格元素的行间距（旧属性已废弃）
  - `column-gap` 指定列之间的间隙（旧属性已废弃）
  - `gap` 是 `row-gap` 和 `column-gap` 的简写属性（旧属性已废弃）
- 用于子元素
  - `grid-row-start` 指定网格元素行的开始位置
  - `grid-row-end` 指定网格元素行的结束位置
  - `grid-row` 是 `grid-row-start` 和 `grid-row-end` 的简写属性
  - `grid-column-start` 指定网格元素列的开始位置
  - `grid-column-end` 指定网格元素列的结束位置
  - `grid-column` 是 `grid-column-start` 和 `grid-column-end` 的简写属性
  - `grid-area` 是 `grid-row-start`、`grid-column-start`、`grid-row-end`、`grid-column-end` 的简写属性
  - `z-index` 指定网格元素的堆叠顺序

```css
.name {
  display: grid;
  grid-template-rows: 1fr 2fr; /* 2行 */
  grid-template-columns: 1fr 2fr 3fr 4fr; /* 4列 */
  grid-template-rows: auto auto auto auto; /* 自动分行 */
  grid-template-columns: auto auto auto auto; /* 自动分列 */
  row-gap: 10px;
  column-gap: 10px;
  gap: 10px;
}

.name .item1 {
  grid-row-start: 1;
  grid-row-end: 3;
  grid-row: 1 / 3;
  grid-row: 1 / span 3;
  grid-column-start: 1;
  grid-column-end: 3;
  grid-column: 1 / 3;
  grid-column: 1 / span 3;
  grid-area: 1 / 1 / 3 / 3;
}
```

网格的增强

- 用于父元素
  - `grid-template-areas` 指定如何显示行和列，使用命名的网格元素
  - `grid` 是 `grid-template-rows`, `grid-template-columns`, `grid-template-areas`, `grid-auto-rows`, `grid-auto-columns`, `grid-auto-flow` 的简写属性
- 用于子元素
  - `grid-area` 指定网格元素的名称，或者也可以是 `grid-row-start`、`grid-column-start`、`grid-row-end`、`grid-column-end` 的简写属性

```css
.name {
  display: grid;
  grid-template-rows: [a z] auto [b] auto [c] auto [d] auto; /* 自动分行并命名网格线 */
  grid-template-columns: [a] auto [b] auto [c] auto [d] auto; /* 自动分列并命名网格线 */
  /* 用模版时需要先定义各部分的 grid-area 名称 */
  grid-template-areas:
    'header header header header header header'
    'menu main main main right right'
    'menu footer footer footer footer footer';
}

/* 用模版时定义各部分的 grid-area 名称 */
.name .item1 {
  grid-area: header;
}
.name .item2 {
  grid-area: menu;
}
.name .item3 {
  grid-area: main;
}
.name .item4 {
  grid-area: right;
}
.name .item5 {
  grid-area: footer;
}
```

网格的对齐

- `justify-items` 指定网格元素在水平方向上的对齐方式
- `align-items` 指定网格元素在垂直方向上的对齐方式
- `place-items` 是 `align-items` 和 `justify-items` 的简写属性
- `justify-content` 指定网格元素整体在水平方向上的对齐方式
- `align-content` 指定网格元素整体在垂直方向上的对齐方式
- `place-content` 是 `align-content` 和 `justify-content` 的简写属性

```css
.name {
  display: grid;
  justify-items: center;
  align-items: center;
  place-items: center;
  justify-content: center;
  align-content: center;
  place-content: center;
}
```

### Flex 和 Grid 两种布局的关系

- 核心联系
  - 二者均属于 CSS3 推出的现代布局方案，都旨在替代传统的 float/position 布局的局限性
  - 都支持容器对子元素的空间分配、对齐方式、排序等灵活控制，都能实现自适应布局
  - 都默认让子元素摆脱文档流的一些束缚，无需手动清除浮动，布局更简洁、可维护性更强
- 核心区别
  - Flex 布局为一维布局，只能同时控制「行」或「列」中的一个方向，从「轴线」出发，依赖主轴/交叉轴，子元素的排列的会相互影响（比如一个元素宽了，其他元素变窄）
  - Grid 布局为二维布局，可同时控制「行」和「列」两个方向，从「网格容器」出发，依赖行轨道 / 列轨道，子元素基于网格单元格排列，相互独立性更强
  - 一句话总结：Flex 管一行一列，Grid 管一片网格
- 各自核心应用场景（落地层面，必会）
  - Flex 布局，优先用这些场景（一维适配）
    - 单行 / 单列元素对齐：比如导航栏的文字水平居中、按钮垂直居中（最常用场景）
    - 元素均匀分布：比如卡片底部的按钮组，自适应分配剩余空间
    - 响应式换行适配：比如标签列表，屏幕变窄时自动换行，换行后仍保持单行内的对齐
    - 子元素灵活伸缩：比如页面头部（左侧 logo，右侧导航自适应拉伸）
  - Grid 布局，优先用这些场景（二维适配）
    - 复杂网格布局：比如仪表盘、相册、电商商品列表（多行多列均匀排列）
    - 页面整体框架搭建：比如页面分为「头部 + 侧边栏 + 主体 + 底部」的经典布局（一行多列 + 多行多列）
    - 子元素精准定位：比如某个元素需要跨 2 行 3 列，Grid 的 grid-row/grid-column 一键实现
    - 不规则网格排版：比如图文混排的杂志式布局，无需嵌套多层容器
- 实用总结
  - 优先用 Flex：只要是「单行 / 单列」的布局需求，Flex 写起来更简单、兼容性更好
  - 果断用 Grid：只要涉及「多行多列」的复杂排版，Grid 比 Flex 更简洁（避免嵌套多层 Flex）
  - 二者可以嵌套使用：比如 Grid 控制页面整体框架，每个网格内部用 Flex 控制元素的单行对齐

## 自定义变量

CSS 自定义变量，官方名称为 CSS 自定义属性（Custom Properties），本质是开发者为 CSS 数值定义的 “可复用标识符”。不同于传统 CSS 重复书写固定值（比如反复写同一个主色调、间距），自定义变量可以一次定义、多处复用，修改时只改一处就能批量生效，是 CSS 原生的 “复用解决方案”，也是响应式、主题切换的核心工具之一。

- 定义语法：`--变量名: 变量值`
  - 必须以 两道短横线（--） 开头，这是 CSS 变量的语法标识，不可省略。变量名可以包含字母、数字、下划线（\_）、短横线（-）、中文，但区分大小写
  - 变量值可以是任意合法的 CSS 值（颜色、尺寸、字体、路径、甚至是 calc 计算式等）
- 调用语法：`var(--变量名, 备用值1, 备用值2...)`
  - `var()` 是专门调用 CSS 变量的函数，不可省略
  - 备用值是可选参数，当变量未定义、定义错误或不兼容时，会自动启用备用值。可以设置多个备用值，优先级从左到右

```css
:root {
  --主色调: #165dff; /* 中文变量名可行 */
  --text-size-base: 16px; /* 基础字号 */
  --spacing-sm: 8px;
  --spacing-md: 16px;
  --spacing-lg: 24px;
  --border-radius: 6px;
  --box-shadow: 0 2px 8px rgba(0, 0, 0, 0.1);
}

body {
  margin: 0;
  padding: var(--spacing-md);
  font-size: var(--text-size-base);
  color: #333;
  background-color: #f5f7fa;
}

.btn {
  padding: var(--spacing-sm) var(--spacing-md);
  background-color: var(--主色调);
  border: none;
  border-radius: var(--border-radius);
  color: #fff;
  cursor: pointer;
  box-shadow: var(--box-shadow);
}

/* 备用值测试：--accent-color 未定义，启用 #FF7D00 */
.btn:hover {
  background-color: var(--accent-color, #ff7d00);
}
```

作用域深度解析

- 全局变量（最常用）
  - 定义在 `:root` 选择器中，root 是 CSS 的伪类，指向文档的根元素
  - 作用范围：整个 HTML 文档，所有选择器都能调用
  - 优势：适合定义全局通用的样式（主色调、基础字号、通用间距等）
- 局部变量
  - 定义在普通选择器中（比如 .card、nav、p 等）
  - 作用范围：仅当前选择器及其后代选择器可用，外部无法调用
  - 优势：适合定义某个组件专属的样式，避免全局污染
- 作用域优先级 & 继承规则
  - 优先级：局部变量 > 全局变量（后代选择器中定义的变量，会覆盖祖先的变量）
  - 继承性：后代选择器会自动继承祖先选择器中定义的变量，无需手动调用

CSS 变量的高级用法

- 结合 calc () 进行数值计算
  - 这是 CSS 变量最常用的高级用法，calc() 函数可以对变量进行加减乘除运算，完美解决 “动态尺寸适配” 问题 `calc(var(--size) * 2)`
  - 结合媒体查询（响应式适配）：媒体查询中可以重写 CSS 变量，实现 “不同屏幕尺寸下，批量修改样式”，比传统响应式写法更简洁、更易维护。
  - 结合 JavaScript 动态修改（核心优势）：这是 CSS 变量 碾压 Less/Sass 变量 的核心亮点 ——运行时可动态修改，而 Less/Sass 变量是编译后固定的，无法修改。通过 JS 可以轻松实现「一键切换主题」「动态调整字体大小」等交互效果，语法非常简单
  - 变量的嵌套使用：CSS 变量可以嵌套调用其他变量，让变量定义更灵活，适合构建层级化的样式体系（比如主色调 → 浅主色调 → 深主色调）

```css
/* 高级用法1 */
:root {
  --base-width: 200px;
}

.name {
  width: calc(var(--base-width) * 1.5); /* 300px */
  height: calc(var(--base-width) / 2); /* 100px */
}
```

```css
/* 高级用法2 */
:root {
  --grid-cols: 2; /* 默认2列网格 */
  --spacing: 16px;
}

/* 平板屏幕（768px+） */
@media (min-width: 768px) {
  :root {
    --grid-cols: 4; /* 重写为4列网格 */
    --spacing: 24px; /* 重写间距 */
  }
}

/* 桌面屏幕（1200px+） */
@media (min-width: 1200px) {
  :root {
    --grid-cols: 6; /* 重写为6列网格 */
  }
}

.grid {
  display: grid;
  grid-template-columns: repeat(var(--grid-cols), 1fr);
  gap: var(--spacing);
}
```

```javascript
// 高级用法3
name.getPropertyValue('--变量名')
name.style.setProperty('--变量名', '新值')
```

```css
/* 高级用法4 */
:root {
  --main-color-rgb: 22, 93, 255;
  --main-color-opacity: 0.5;
  --main-color: rgba(var(--main-color-rgb), var(--main-color-opacity));
}

.name {
  background-color: var(--main-color);
}
```

兼容性处理：CSS 变量不支持 IE 11 及以下浏览器，若需要兼容这些旧浏览器，有两种解决方案

- 方案一：降级写法（最简单），在调用变量的同时，写一句普通 CSS 值作为降级，IE 会忽略变量，执行普通值

```css
.btn {
  /* IE 降级值 */
  background-color: #165dff;
  /* 现代浏览器生效 */
  background-color: var(--main-color);
}
```

- 方案二：使用 PostCSS 插件（进阶）
  - 安装 postcss-custom-properties 插件，编译时会自动将 CSS 变量替换为固定值，实现兼容
  - 缺点：编译后变量变为固定值，失去动态修改的能力

## 自定义字体

`@font-face` 是一个字体规则，用来告诉浏览器：「我要引入一个自定义字体，你按照我给的路径加载，之后就能用我定义的字体名调用它」。当然也可以直接使用字体图标（iconfont 用法）来解决问题

> 注意：引入自定义字体会增加页面加载时间，还会引起字体版权问题，慎重使用

```css
/* 1. 定义自定义字体 */
@font-face {
  /* 自定义字体名（后续调用用它）（必选） */
  font-family: 'MyCustomFont';
  /* 字体文件路径（必选） */
  src: url('fonts/我的字体.ttf');
  /* 字体粗细（可选）：normal(400)/bold(700)/100-900 */
  font-weight: normal;
  /* 字体样式（可选）：normal/italic/oblique */
  font-style: normal;
}

/* 2. 调用自定义字体 */
body {
  /* 兜底字体：加载失败时显示默认无衬线字体 */
  font-family: 'MyCustomFont', sans-serif;
}
```

## 兼容性处理

浏览器前缀：浏览器前缀是浏览器厂商针对未标准化的 CSS 属性，推出的私有前缀，用来适配各自浏览器的特殊实现

- 四大核心前缀对应浏览器
  - `-webkit-`：谷歌内核，适配 Chrome、Safari、Edge 新版、移动端所有浏览器（最常用）
  - `-moz-`：火狐内核，适配 Firefox（火狐浏览器）
  - `-ms-`：微软内核，适配 IE 浏览器（目前基本淘汰，但旧项目可能残留）
  - `-o-`：欧朋内核，适配 Opera（欧朋浏览器，现在用得极少）
- 使用规范：私有前缀写在前面，标准属性写在最后（优先级最低，保证后续浏览器支持标准语法时生效），浏览器具体版本号可以写在最前面

```css
/* 示例：圆角属性的前缀写法 */
.box {
  -webkit-border-radius: 8px;
  -moz-border-radius: 8px;
  -ms-border-radius: 8px;
  -o-border-radius: 8px;
  /* 标准属性 */
  border-radius: 8px;
}
```

> 补充：现在很多 CSS 属性已经标准化，不需要写前缀（比如 color、font-size），只针对部分新属性 / 特殊属性需要添加（如 backdrop-filter、scrollbar-width 等）。

降级方案编写技巧

- 降级方案的核心原则：「高级特性适配现代浏览器，低级特性兜底旧浏览器」，保证旧浏览器不报错、页面不崩，现代浏览器体验更好。
- 技巧 1：属性降级（最基础）。当一个高级属性旧浏览器不支持时，给它写一个旧浏览器支持的替代属性。

```css
/* 示例：渐变属性降级 */
.box {
  background: #f0f0f0; /* 旧浏览器兜底：纯色背景 */
  background: linear-gradient(#f0f0f0, #ccc); /* 现代浏览器：渐变背景 */
}
```

- 技巧 2：条件判断降级（针对 IE 浏览器）。主要用于 IE 浏览器的特殊适配（IE 不支持很多现代 CSS 特性），常用 HTML 条件注释。
  - 注：IE 已经被微软淘汰，现在这种写法只用于极古老的项目维护。

```html
<!-- 只有 IE 浏览器会加载这个样式文件 -->
<!--[if IE]>
  <link rel="stylesheet" href="ie.css" />
<![endif]-->
```

- 技巧 3：能力检测降级（最推荐）
  通过简单的 JS 检测浏览器是否支持某个属性，再动态加载对应的样式，适配性更灵活。

```javascript
运行
// 检测浏览器是否支持 flex 布局
if (!('flex' in document.documentElement.style)) {
  // 不支持 flex，给 html 加一个类名，加载降级样式
  document.documentElement.classList.add('no-flex')
}
```

```css
/* 现代浏览器：flex 布局 */
.father {
  display: flex;
}
/* 旧浏览器：降级为浮动布局 */
.no-flex .father {
  overflow: hidden;
}
.no-flex .son {
  float: left;
}
```

## 移动端基础补充

视口：视口（viewport）就是浏览器显示页面内容的屏幕区域。视口可以分为布局视口、视觉视口和理想视口

- 布局视口（layout viewport）
  - 一般移动设备的浏览器都默认设置了一个布局视口，用于解决早期的 PC 端页面在手机上显示的问题
  - iOS,Android 基本都将这个视口分辨率设置为 980px，所以 PC 上的网页大多都能在手机上呈现，只不过元素看上去很小，一般默认可以通过手动缩放网员
- 视觉视口（visual viewport）
  - 字面意思，它是用户正在看到的网站的区域。注意：是网站的区域
  - 我们可以通过缩放去操作视觉视口，但不会影响布局视口，布局视口仍保持原来的宽度
- 理想视口（ideal viewport）
  - 为了使网站在移动端有最理想的浏览和阅读宽度而设定
  - 理想视口，对设备来讲，是最理想的视口尺寸
  - 需要手动添写 meta 视口标签通知浏览器操作
  - meta 视口标签的主要目的：布局视口的宽度应该与理想视口的宽度一致，简单理解就是设备有多宽，我们布局的视口就多宽
- 总结
  - 视口就是浏览器显示页面内容的屏幕区域，视口分为布局视口、视觉视口和理想视口
  - 我们移动端布局想要的是理想视口就是手机屏幕有多宽，我们的布局视口就有多宽
  - 想要理想视口，我们需要给我们的移动端页面添加 meta 视口标签

物理像素&物理像素比

- 物理像素点指的是屏幕显示的最小颗粒，是物理真实存在的。这是厂商在出厂时就设置好了，比如苹果 6\7\8 是 750\*1334
- PC 端页面，1 个 px 等于 1 个物理像素的。但是移动端就不尽相同。我们开发时候的 1px 不是一定等于 1 个物理像素的
- 一个 px 的能显示的物理像素点的个数，称为物理像素比或屏幕像素比
- PC 端和早前的手机屏幕/普通手机屏幕：1CSS 像素=1 物理像素的
- Retina（视网膜屏幕）是一种显示技术，可以将把更多的物理像素点压缩至一块屏幕里，从而达到更高的分辨率，并提高屏幕显示的细腻程度

二倍图/多倍图

- 对于一张 50px\*50px 的图片，在手机 Retina 屏中打开，按照刚才的物理像索比会放大倍数，这样会造成图片模糊
- 在标准的 viewport 设置中，使用倍图来提高图片质量，解决在高清设备中的模糊问题
- 通常使用二倍图，因为 iPhone 6\7\8 的影响，但是现在还存在 3 倍图 4 倍图的情况，这个看实际开发公司需求
- 背景图片注意缩放问题

```css
/* 在iphone8 下面 */
img {
  /* 原始图片100*100px */
  width: 50px;
  height: 50px;
}
.box {
  /* 原始图片100*100px */
  background-size: 50px 50px;
  background-size: contain;
  background-size: cover;
}
```

### BEM 命名规范（块-元素-修饰符）

- 这是最常用、最规范的命名方式，核心是结构清晰，语义唯一，解决多人协作时命名混乱的问题。
- 核心语法：`块__元素--修饰符`（注意：双下划线分隔元素，双连字符分隔修饰符，单连字符连接单词）
  - 块（Block）：独立的、可复用的整体组件（如头部、卡片、按钮）
  - 元素（Element）：块的组成部分，不能脱离块单独存在
  - 修饰符（Modifier）：块或元素的状态、样式变体

```css
/* 块：卡片（独立组件） */
.card {
}
/* 元素：卡片的标题、内容（依赖card存在） */
.card__title {
}
.card__content {
}
/* 修饰符：卡片的高亮状态、小尺寸；标题的加粗状态 */
.card--highlight {
}
.card--small {
}
.card__title--bold {
}
```

### OOCSS 理念（面向对象 CSS）

- 核心是面向复用、分离结构和样式，把通用样式抽离成 “公共类”，实现一次编写，多次复用（类比 JS 的面向对象思想）
- 两大核心原则
  - 分离结构与皮肤：结构（宽高、间距、定位）和皮肤（颜色、背景、边框）分开写
  - 分离容器与内容：内容的样式不依赖于父容器，确保内容可移植

```css
/* 公共结构类（复用） */
.container {
  width: 1200px;
  margin: 0 auto;
}
.flex-center {
  display: flex;
  align-items: center;
  justify-content: center;
}
/* 公共皮肤类（复用） */
.bg-red {
  background: #ff0000;
}
.text-gray {
  color: #666;
}
/* 具体组件调用公共类+自身特殊样式 */
.btn {
  padding: 4px 12px;
}
```

### CSS 命名禁忌与最佳实践

- 命名禁忌（绝对不要做）
  - 禁止使用无语义的随机字符/数字（如 `.box1`、`.a123`、`.abc`）
  - 禁止使用浏览器默认标签名作为类名（如 `.div`、`.span`、`.p`）
  - 禁止使用样式属性名作为类名（如 `.red`、`.big`、`.left`）—— 样式变了，类名就失效
  - 禁止使用中文 / 特殊符号（如 `.头部`、`.card*`），允许单连字符 `-` 连接单词
  - 禁止嵌套过深的命名（如 `.header-nav-list-item-link`，建议用 BEM 替代）
- 最佳实践（一定要遵守）
  - 语义优先：类名要见名知意，优先使用英文单词（如 `.header`、`.footer`、`.nav`、`.aside`）
  - 小写为主：所有类名统一小写，单词之间用 - 连接（如 `.user-info`，而非 `.userInfo` 或 `.UserInfo`）
  - 简洁适度：避免过长的类名，BEM 命名也不要过度嵌套（最多三级）
  - 统一前缀：通用组件或页面模块加前缀（如页面组件 `.page-header`，公共组件 `.com-btn`）
  - 避免冗余：不要重复父级语义（如 `.header .header-nav` 可简化为 `.header__nav`，符合 BEM）

## CSS 优化

- 选择器权重优化（避免过度嵌套）
  - 核心原则：权重越低、层级越少，浏览器渲染速度越快，且后期维护越简单
  - 避免过度嵌套：很多新手会写 `div.container ul li a` 这种多层嵌套，最优写法是给目标元素加类，直接用类选择器 `.link-btn` 定位，层级 ≤3 层
  - 拒绝 `!important` 滥用：`!important` 会强制覆盖所有同权重样式，后续修改只能再加 `!important` 形成恶性循环，仅在紧急修复全局样式时临时使用
  - 优先使用轻量化选择器：优先级 ID 选择器(#) > 类选择器(.) > 元素选择器(div) ，优先用类选择器（权重适中、复用性强），少用 ID 选择器（权重太高，无法复用）
- 样式去重与复用
  - 核心原则：不要写重复代码，减少冗余，提升样式可维护性，改一处同步所有关联元素。
  - 提取公共样式，封装通用类：把页面中重复出现的样式（如居中、清除浮动、文字省略）提取成全局通用类，直接在 HTML 元素上复用。
  - 利用 CSS 继承特性：字体类（font-size/font-family）、颜色类（color）等可继承样式，写在父容器上，子元素自动继承，无需重复书写。
  - 巧用复合属性：把多个单一属性合并成复合属性，简化代码。
