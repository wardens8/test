## Less 概述

Less（Leaner Style Sheets 精简样式表）是一种动态样式语言，属于 CSS 预处理器的范畴，它扩展了 CSS 语言，增加了变量、Mixin、函数等特性，使 CSS 更易维护和扩展，Less 既可以在 客户端上运行 ，也可以借助 Node.js 在服务端运行。

Less（Leaner Style Sheets 的缩写）是一种向后兼容的 CSS 语言扩展。这是 Less 语言和 Less.js 的官方文档，Less.js 是将 Less 样式转换为 CSS 样式的 JavaScript 工具。

因为 Less 看起来就像 CSS，所以学习起来很容易。Less 只对 CSS 语言做了一些方便的补充，这也是它可以学得这么快的原因之一。

## Less 环境搭建

安装 Less 编译器（需要安装 Nodejs）

```bash
# 全局安装
npm install -g less
# 项目安装
npm install -D less

# 编译单个文件
lessc src/index.less dist/index.css
# 压缩输出（生产）
lessc src/index.less dist/index.min.css --clean-css
# 监听文件自动编译（开发）
lessc src/index.less dist/index.css --watch
```

工程化集成

- Vite：内置 Less 支持，直接安装 less 即可
- Webpack：配置 less-loader、css-loader、style-loader
- Vue/React：脚手架默认支持，只需安装 less 依赖

浏览器安装

```html
<link rel="stylesheet/less" type="text/css" href="styles.less" />
<style type="text/less"></style>
<script src="https://cdn.jsdelivr.net/npm/less"></script>
<!-- 完成后需要用 LiveServer 打开 web 服务器查看，本地文件会出现跨域问题 -->
```

使用 `.less` 文件 ✅️

```html
<!-- VSCode 安装 Easy LESS 插件，可以帮助我们在同一目录下将 Less 文件编译成 CSS 文件，并且可以实时预览。 -->
<link rel="stylesheet" href="styles.css" />
```

- 安装 `Easy LESS` 插件后的配置：可在 `settings.json` 文件中添加配置来修改生成的css的位置，或在单独less文件顶部写入单独配置来修改生成的css的位置，路径写相对路径或绝对路径

```json
// settings.json
{
  "less.compile": {
    "out": "../css/[index.css]"
  }
}
```

```less
// 单独配置
// out: ../css/[index.css] 生成css文件至指定位置
// out: false 不生成css文件
```

## Less 核心语法

### 注释 Comments

Less 支持两种注释方式：

- `//` 单行注释，不会被编译到 CSS 文件中，快捷键：`Ctrl + /`
- `/* */` 多行注释，会被编译到 CSS 文件中，快捷键：`Ctrl + Shift + /`

```less
// Get in line!

/* One heck of a block
 * style comment! */
```

### 变量 Variables

Less 可以使用 `@` 符号来定义变量，变量名由字母、数字、下划线组成，并且不能以数字开头

- 定义变量：用 `@` 符号定义变量，使用时引用 `@变量名`

```less
// Variables
@width: 10px;
@height: @width + 10px;
// Usage
#header {
  width: @width;
  height: @height;
}
// 编译为：
#header {
  width: 10px;
  height: 20px;
}
```

可变插值：使用变量来控制 CSS 规则中的值，但它们也可以用在其他地方，例如选择器名称、属性名称、URL 和 @import 语句。变量名用 `@{变量名}` 的形式引用

- 定义属性：用 `@` 符号定义属性，使用时引用 `@{属性名}`
- 定义选择器：用 `@` 符号定义选择器，使用时引用 `@{选择器名}`
- 定义 URLs：用 `@` 符号定义 URLs，使用时引用 `@{URLs}`

```less
// Variables
@property: color;
// Usage
.widget {
  @{property}: #0ee;
  background-@{property}: #999;
}
// 编译为：
.widget {
  color: #0ee;
  background-color: #999;
}
```

```less
// Variables
@selector: banner;
// Usage
.@{selector} {
  font-weight: bold;
  line-height: 40px;
  margin: 0 auto;
}
// 编译为：
.banner {
  font-weight: bold;
  line-height: 40px;
  margin: 0 auto;
}
```

```less
// Variables
@images: '../img';
// Usage
body {
  color: #444;
  background: url('@{images}/white-sand.png');
}
// 编译为：
body {
  color: #444;
  background: url('../img/white-sand.png');
}
```

可变变量：在 Less 中，你可以使用另一个变量定义一个变量的名称

```less
// Variables
@primary: green;
@secondary: blue;
// Usage
.section {
  @color: primary;
  .element {
    color: @@color;
  }
}
// 编译为：
.section .element {
  color: green;
}
```

延迟评估（延迟加载）：变量在使用前不必声明，使用时才解析（类似于变量提升）

```less
// 有效的 Less 片段：
.lazy-eval {
  width: @var;
}
@var: @a;
@a: 9%;

// 这也是有效的 Less 片段：
.lazy-eval {
  width: @var;
  @a: 9%;
}
@var: @a;
@a: 100%;

// 都编译成：
.lazy-eval {
  width: 9%;
}
```

作用域规则：两次定义变量时，使用变量的最后定义，从当前作用域向上搜索。这类似于 css 本身，其中定义中的最后一个属性用于确定值。局部变量会覆盖全局变量

```less
// 例如：
@var: 0;
.class {
  @var: 1;
  .brass {
    @var: 2;
    three: @var;
    @var: 3;
  }
  one: @var;
}
// 编译为：
.class {
  one: 1;
}
.class .brass {
  three: 3;
}
// 本质上，每个作用域都有一个 "final" 值，类似于浏览器中的属性，就像这个使用自定义属性的例子：
.header {
  --color: white;
  color: var(--color); // the color is black
  --color: black;
}
// 这意味着，与其他 CSS 预处理语言不同，Less 变量的行为与 CSS 非常相似
```

属性作为变量（新!）：你可以使用 `$prop` 语法轻松地将属性视为变量。有时这可以使你的代码更轻一些。请注意，与变量一样，Less 将选择当前/父作用域内的最后一个属性作为 "final" 值

```less
.widget {
  color: #efefef;
  background-color: $color;
}
// 编译为：
.widget {
  color: #efefef;
  background-color: #efefef;
}
```

```less
.block {
  color: red;
  .inner {
    background-color: $color;
  }
  color: blue;
}
// 编译为：
.block {
  color: red;
  color: blue;
}
.block .inner {
  background-color: blue;
}
```

### 嵌套 Nesting

Less 使你能够使用嵌套代替级联，或与级联结合使用

```css
/* 假设我们有以下 CSS */
#header {
  color: black;
}
#header .navigation {
  font-size: 12px;
}
#header .logo {
  width: 300px;
}
```

```less
// 在 Less 中，我们也可以这样写：
#header {
  color: black;
  .navigation {
    font-size: 12px;
  }
  .logo {
    width: 300px;
  }
}
// 生成的代码更加简洁，并且模仿了 HTML 的结构
```

在嵌套规则中， `&` 表示父选择器，常用于给现有选择器添加伪类。你可以使用此方法将伪选择器与混入打包在一起。这是经典的 `clearfix hack`

```less
.clearfix {
  display: block;
  zoom: 1;
  &:after {
    content: ' ';
    display: block;
    font-size: 0;
    height: 0;
    clear: both;
    visibility: hidden;
  }
}
```

嵌套的 `@` 规则和冒泡：诸如 `@media` 或 `@supports` 之类的 `@` 规则可以以与选择器相同的方式嵌套。`@` 规则放在最前面，与同一规则集中其他元素的相对顺序保持不变。这称为冒泡。

```less
.component {
  width: 300px;
  @media (min-width: 768px) {
    width: 600px;
    @media (min-resolution: 192dpi) {
      background-image: url(/img/retina2x.png);
    }
  }
  @media (min-width: 1280px) {
    width: 800px;
  }
}
// 编译为：
.component {
  width: 300px;
}
@media (min-width: 768px) {
  .component {
    width: 600px;
  }
}
@media (min-width: 768px) and (min-resolution: 192dpi) {
  .component {
    background-image: url(/img/retina2x.png);
  }
}
@media (min-width: 1280px) {
  .component {
    width: 800px;
  }
}
```

### 混入 Mixins

混入是一种将一组属性从一个规则集中包含（混合进入）到另一个规则集中的方法

- 普通混入：会直接编译至 CSS 文件中

```less
// Variables
.bordered {
  border-top: dotted 1px black;
  border-bottom: solid 2px black;
}
// Usage
#menu a {
  color: #111;
  .bordered();
}
.post a {
  color: red;
  .bordered();
}
// .bordered 类的属性现在将出现在 #menu a 和 .post a 中。（请注意，你也可以将 #ids 用作混入）
// 编译为：
.bordered {
  border-top: dotted 1px black;
  border-bottom: solid 2px black;
}
#menu a {
  color: #111;
  border-top: dotted 1px black;
  border-bottom: solid 2px black;
}
.post a {
  color: red;
  border-top: dotted 1px black;
  border-bottom: solid 2px black;
}
```

- 带括号的混入：如果你想创建一个 mixin 但你不希望那个 mixin 出现在你的 CSS 输出中，请在 mixin 定义之后加上括号

```less
// Variables
.my-mixin {
  color: black;
}
.my-other-mixin() {
  background: white;
}
// Usage
.class {
  .my-mixin();
  .my-other-mixin();
}
// 编译为：
.my-mixin {
  color: black;
}
.class {
  color: black;
  background: white;
}
```

- 带参数的混入
  - Mixins 可以接受参数，这些参数是在混合时传递给选择器块的变量
  - Mixins 可以为其参数设置默认值
  - Mixins 可以带入多个参数
  - Mixins 可以通过名称而不是位置来提供参数值

```less
// Variables
.border-radius(@radius) {
  -webkit-border-radius: @radius;
  -moz-border-radius: @radius;
  border-radius: @radius;
}
// Usage
#header {
  .border-radius(4px);
}
.button {
  .border-radius(6px);
}
// 编译为：
#header {
  -webkit-border-radius: 4px;
  -moz-border-radius: 4px;
  border-radius: 4px;
}
.button {
  -webkit-border-radius: 6px;
  -moz-border-radius: 6px;
  border-radius: 6px;
}
```

```less
// Variables
.border-radius(@radius: 5px) {
  -webkit-border-radius: @radius;
  -moz-border-radius: @radius;
  border-radius: @radius;
}
// Usage
#header {
  .border-radius();
}
// 编译为：
#header {
  -webkit-border-radius: 5px;
  -moz-border-radius: 5px;
  border-radius: 5px;
}
```

```less
// Variables
.mixin(@color; @margin; @padding) {
  color: @color;
  margin: @margin;
  padding: @padding;
}
// Usage
.class {
  .mixin(black; 10px; 20px);
}
// 编译为：
.class {
  color: black;
  margin: 10px;
  padding: 20px;
}
```

```less
// Variables
.mixin(@color: black; @margin: 10px; @padding: 20px) {
  color: @color;
  margin: @margin;
  padding: @padding;
}
// Usage
.class1 {
  .mixin(@margin: 20px; @color: #33acfe);
}
.class2 {
  .mixin(#efca44; @padding: 40px);
}
// 编译为：
.class1 {
  color: #33acfe;
  margin: 20px;
  padding: 20px;
}
.class2 {
  color: #efca44;
  margin: 10px;
  padding: 40px;
}
```

- 匹配模式：在多个相同的混合中（参数个数必须相同），匹配特定的混合

```less
// Variables
.mixin(dark, @color) {
  color: darken(@color, 10%);
}
.mixin(light, @color) {
  color: lighten(@color, 10%);
}
// @_ 表示匹配所有
.mixin(@_, @color) {
  display: block;
}
// Usage
.class1 {
  .mixin(dark, #888);
}
.class2 {
  .mixin(light, #888);
}
// 编译为：
.class1 {
  color: #6f6f6f;
  display: block;
}
.class2 {
  color: #a2a2a2;
  display: block;
}
```

- `@arguments` 变量：`@arguments` 变量包含了传入的所有参数

```less
// Variables
.box-shadow(@x: 0, @y: 0, @blur: 1px, @color: #000) {
  -webkit-box-shadow: @arguments;
  -moz-box-shadow: @arguments;
  box-shadow: @arguments;
}
// Usage
.big-block {
  .box-shadow(2px, 5px);
}
// 编译为：
.big-block {
  -webkit-box-shadow: 2px 5px 1px #000;
  -moz-box-shadow: 2px 5px 1px #000;
  box-shadow: 2px 5px 1px #000;
}
```

### 运算 Operations

算术运算 `+`、`-`、`*`、`/` 可以对任何数字、颜色或变量进行运算。如果可能，数学运算会考虑单位并在加、减或比较之前转换数字。结果在最左边有明确说明的单位类型。如果转换不可能或没有意义，则忽略单位。不可能转换的例子：px 到 cm 或 rad 到 %。

```less
// numbers are converted into the same units
@conversion-1: 5cm + 10mm; // result is 6cm
@conversion-2: 2 - 3cm - 5mm; // result is -1.5cm

// conversion is impossible
@incompatible-units: 2 + 5px - 3cm; // result is 4px

// example with variables
@base: 5%;
@filler: @base * 2; // result is 10%
@other: @base + @filler; // result is 15%
```

乘法和除法不转换数字。大多数情况下没有意义 - 长度乘以长度得出面积，而 css 不支持指定面积。Less 将按原样对数字进行运算，并将明确声明的单位类型分配给结果。

```less
@base: 2cm * 3mm; // result is 6cm
```

你还可以对颜色进行算术运算：

```less
@color: (#224488 / 2); // result is #112244
background-color: #112244 + #111; // result is #223355
```

但是，你可能会发现 Less 的**颜色函数**更有用。

从 4.0 开始，不会使用 `/` 运算符在括号外执行除法。

```less
@color: #222 / 2; // results in `#222 / 2`, not #111
background-color: (#ffffff / 16); //results is #101010
```

你可以改变**数学**设置，如果你想让它一直有效，但不推荐。

`calc()` 异常：发布于 v3.0.0

为了 CSS 兼容性，`calc()` 不计算数学表达式，但会计算嵌套函数中的变量和数学。

```less
@var: 50vh/2;
width: calc(50% + (@var - 20px)); // result is calc(50% + (25vh - 20px))
```

### 转义 Escaping

转义允许你使用任意字符串作为属性或变量值。`~"anything"` 或 `~'anything'` 中的任何内容都按原样使用，除了插值之外没有任何变化。

```less
@min768: ~'(min-width: 768px)';
.element {
  @media @min768 {
    font-size: 1.2rem;
  }
}
// 结果是：
@media (min-width: 768px) {
  .element {
    font-size: 1.2rem;
  }
}
```

注意，从 Less 3.5 开始，你可以简单地写。在 3.5+ 中，许多以前需要 "引号转义" 的案例不再需要了。

```less
@min768: (min-width: 768px);
.element {
  @media @min768 {
    font-size: 1.2rem;
  }
}
// 结果是：
@media (min-width: 768px) {
  .element {
    font-size: 1.2rem;
  }
}
```

## Less 进阶语法

### 函数 Functions

Less 提供了许多函数，可以转换颜色、操作字符串和进行数学运算。具体内容可以参考官网内容。

使用它们非常简单。以下示例使用百分比将 0.5 转换为 50%，将基色的饱和度增加 5%，然后将背景颜色设置为亮 25% 并旋转 8 度的颜色：

```less
@base: #f04615;
@width: 0.5;
.class {
  width: percentage(@width); // returns `50%`
  color: saturate(@base, 5%);
  background-color: spin(lighten(@base, 25%), 8);
}
```

常用颜色函数

```less
lighten(#000, 10%); // 变亮
darken(#fff, 10%); // 变暗
saturate(@color, 5%); // 饱和
desaturate(@color,5%); // 降饱和
fade(@color, 50%); // 透明度
mix(#f00, #00f, 50%); // 混合
```

数学/字符串/列表函数

```less
round(3.14); // 3
ceil(3.1); // 4
floor(3.9); // 3
percentage(0.5); // 50%
replace("a/b", "/", "-"); // a-b
length(@list); // 列表长度
extract(@list, 2); // 取第2项
```

自定义函数（混合 + 返回）

```less
.add(@a, @b) {
  @result: @a + @b;
}
.box {
  .add(10px, 20px);
  width: @result; // 30px
}
```

### 命名空间和访问器 Namespaces and Accessors

有时，出于组织目的或只是为了提供一些封装，你可能希望对混入进行分组。你可以在 Less 中非常直观地做到这一点。假设你想在 #bundle 下打包一些混入和变量，以供以后重用或分发。现在如果我们想在我们的 `#header a` 中混合 `.button` 类，我们可以这样做：

```less
// Variables
#bundle() {
  .button {
    display: block;
    border: 1px solid black;
    background-color: grey;
    &:hover {
      background-color: white;
    }
  }
  .tab {
    ...;
  }
  .citation {
    ...;
  }
}
// Usage
#header a {
  color: orange;
  #bundle.button(); // can also be written as #bundle > .button
}
// 编译为：
#header a {
  color: orange;
  display: block;
  border: 1px solid black;
  background-color: grey;
}
#header a:hover {
  background-color: white;
}
```

### 条件判断 Guards

Less 提供了条件判断，类似于编程语言中的 if 语句。

- 基础语法：`.mixin(@width) when (@width > 200px) { width: @width; }`
- 条件逻辑（修正）：
  - 或条件：`when (@a = 1), (@b = 2)`
  - 且条件：`when (@a = 1) and (@b = 2)`
  - 非条件：`when not (@a = 1)`
- 应用场景：根据变量值动态生成样式（如不同尺寸的组件样式）

```less
.mixin (@a) when (lightness(@a) >= 50%) {
  background-color: black;
}
.mixin (@a) when (lightness(@a) < 50%) {
  background-color: white;
}
.mixin (@a) {
  color: @a;
}
// Usage
.class1 {
  .mixin(#ddd);
}
.class2 {
  .mixin(#555);
}
// 编译为：
.class1 {
  background-color: black;
  color: #ddd;
}
.class2 {
  background-color: white;
  color: #555;
}
```

### 循环 Loops

Less 提供了循环，类似于编程语言中的 for 语句。

- 基础语法：`.loop (@i) when (@i > 0) { .loop(@i - 1); }`
- 应用场景：根据变量值动态生成样式（如不同尺寸的组件样式）

```less
.generate-columns(4);
.generate-columns(@n, @i: 1) when (@i <= @n) {
  .column-@{i} {
    width: (@i * 100% / @n);
  }
  .generate-columns(@n, (@i + 1));
}
// 编译为：
.column-1 {
  width: 25%;
}
.column-2 {
  width: 50%;
}
.column-3 {
  width: 75%;
}
.column-4 {
  width: 100%;
}
```

### 映射 Maps

从 Less 3.5 开始，你还可以使用混入和规则集作为值映射。

```less
@colors: {
  primary: blue;
  secondary: green;
};
.button {
  color: @colors[primary];
  border: 1px solid @colors[secondary];
}
// 如预期的那样输出：
.button {
  color: blue;
  border: 1px solid green;
}
```

```less
#colors() {
  primary: blue;
  secondary: green;
}
.button {
  color: #colors[primary];
  border: 1px solid #colors[secondary];
}
// 如预期的那样输出：
.button {
  color: blue;
  border: 1px solid green;
}
```

### 继承 Extend

继承可让多个选择器应用同一组属性，最终编译为并集选择器，其性能比混合高，但灵活性比混合低

```less
nav ul {
  &:extend(.inline);
  background: blue;
}
.inline {
  color: red;
}
// 编译为：
nav ul {
  background: blue;
}
.inline,
nav ul {
  color: red;
}
```

Extend "all"：可理解为把 all 前的选择器出现的地方全都替换为 extend 前的选择器。即把 .test 替换为 .replacement 生成一个新的选择器应用样式，且不破坏原有的选择器

```less
.a.b.test,
.test.c {
  color: orange;
}
.test {
  &:hover {
    color: green;
  }
}
.replacement:extend(.test all) {
}
// 编译为：
.a.b.test,
.test.c,
.a.b.replacement,
.replacement.c {
  color: orange;
}
.test:hover,
.replacement:hover {
  color: green;
}
```

### 导入 Importing

导入（Import）允许你将一个 Less 文件导入到另一个文件中。Less 文件导入时可以不加扩展名，Less 将会按顺序查找扩展名为 `.less`、`.css` 的文件。

```less
@import 'library'; // library.less
@import 'typo.css';
```

导入修饰符

- `reference`：只引用，不编译，不生成 css
- `inline`：直接将内容插入，不生成 css
- `once`：默认，只编译一次
- `multiple`：可多次编译

```less
@import (reference) 'library'; // library.less
@import (inline) 'library'; // library.less
@import (once) 'library'; // library.less
@import (multiple) 'library'; // library.less
```

### 避免编译

通过加**引号**可以避免 Less 编译，直接把内容输出到 CSS 中

```less
.banner .inline .header {
  width: '100px + 100px';
  width: ~'100px + 100px';
  height: 100px + 100px;
}
// 编译为：
.banner .inline .header {
  width: '100px + 100px';
  width: 100px + 100px;
  height: 200px;
}
```

## Less 工程化目录

```plaintext
src/less/
├── variables.less // 全局变量（颜色、间距、字体、z-index）
├── mixins.less // 通用混合（居中、尺寸、文本截断）
├── reset.less // 样式重置
├── base.less // 基础样式（html/body/链接）
├── components/ // 组件样式（button/card/modal）
├── pages/ // 页面级样式
└── index.less // 入口（统一导入）
```

入口文件统一管理

```less
// index.less
@import 'variables';
@import 'mixins';
@import 'reset';
@import 'base';
@import 'components/**';
@import 'pages/**';
```

// 颜色
