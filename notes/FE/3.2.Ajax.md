# Ajax

Ajax：Asynchronous JavaScript and XML，异步的 JavaScript 和 XML，是一种用于创建快速动态网页的技术，通过在后台与服务器进行少量数据交换，Ajax 可以使网页实现异步更新。这意味着可以在不重新加载整个网页的情况下，对网页的某部分进行更新。

- MDN 官网：https://developer.mozilla.org/zh-CN/
- Ajax MDN 官网：https://developer.mozilla.org/zh-CN/docs/Web/Guide/AJAX

后续开发中，Ajax 负责的是前后端数据交互，前端框架负责的是页面渲染和交互。

## XMLHttpRequest

XMLHttpRequest 对象用于在后台与服务器交换数据。这意味着可以在不重新加载整个页面的情况下，对网页的某部分进行更新，简称为 XHR/xhr。但是，XMLHttpRequest 的方式已经过时，不推荐使用。

- MDN 官网：https://developer.mozilla.org/zh-CN/docs/Web/API/XMLHttpRequest

### XHR 基础

- `const xhr = new XMLHttpRequest()` 创建一个 XMLHttpRequest 对象
- `xhr.open(method, url, async)` 打开一个异步请求，method 是请求方法，url 是请求地址，async 是是否异步
- `xhr.setRequestHeader(header, value)` 设置请求头，value 只能写一个字符串
  - Access-Control-Allow-Origin：允许跨域请求的源
  - Access-Control-Allow-Methods：允许跨域请求的方法
  - Access-Control-Allow-Headers：允许跨域请求的头部
  - Content-Type：请求体的类型
- `xhr.onreadystatechange` 监听请求状态变化
  - xhr.readyState：请求状态，0-4，0 初始化，1-2 发送请求，3-4 请求完成
  - xhr.status：响应状态码，**200-299 表示请求成功**，**3xx 表示重定向**（301永久重定向、302临时重定向），**4xx 表示客户端错误**（400请求错误、401未授权、402需要付费、403禁止访问、404资源不存在），**5xx 表示服务器错误**（500服务器错误、501服务器不支持请求的功能、502网关错误、503服务器超载）
  - xhr.response：响应体，类型灵活
  - xhr.responseText：响应体，类型是字符串
- `xhr.send(body)` 发送请求，body 是请求体

```javascript
// 创建 XMLHttpRequest 对象
const xhr = new XMLHttpRequest()
// 打开一个异步请求
xhr.open('GET', 'https://api.example.com/data', true)
// 设置请求头
xhr.setRequestHeader('Content-Type', 'application/json')
// 监听请求状态变化
xhr.onreadystatechange = function () {
  if (xhr.readyState === 4 && xhr.status === 200) {
    const data = JSON.parse(xhr.responseText)
    console.log(data)
  }
}
// 发送请求
xhr.send()
```

直接使用

```javascript

const url = 'https://api.example.com/data'
xhr.open('GET', url, true)

xhr.send()
```

函数方式使用

```javascript

  const xhr = new XMLHttpRequest()
  xhr.open('GET', url, true)
  xhr.onreadystatechange = function () {
    if (xhr.readyState === 4 && xhr.status === 200) {
      const data = JSON.parse(xhr.responseText)

    }
  }
  xhr.send()
}
```

## Fetch API

Fetch 提供了一个更现代、更灵活的方式来处理网络请求，属于 XMLHttpRequest 的升级版。它返回一个 Promise 对象，可以使用 `.then()` 和 `.catch()` 方法来处理请求的结果和错误。

- MDN 官网：https://developer.mozilla.org/zh-CN/docs/Web/API/Fetch_API

### Fetch 基础

- `fetch(url, options)` 发送请求，url 是请求地址，options 是请求配置
- `fetch(url, options).then(res => res.xxxx()).then(data => console.log(data))` 发送请求，解析响应体格式，然后处理数据

```javascript
fetch('https://api.example.com/data')
  .then((res) => {
    if (res.status === 200) {
      // 返回一个 Promise 对象，解析响应体为 JSON 格式
      return res.json()
    } else {
      // 手动抛出一个错误，可以在 catch 中统一捕获处理
      throw new Error('请求失败')
    }
  })
  .then((data) => {
    if (data.status === 200) {
      console.log(data)
    }
  })
  .catch((error) => console.error(error))
```

```javascript
// 发送请求，并解析响应体为 JSON 格式，然后处理数据
fetch(url, options)
  .then((res) => res.json())
  .then((data) => console.log(data))
  .catch((error) => console.error(error))

// 发送请求，并解析响应体为文本格式，然后处理数据
fetch(url, options)
  .then((res) => res.text())
  .then((data) => console.log(data))
  .catch((error) => console.error(error))

// 发送请求，并解析响应体为二进制格式，然后处理数据
fetch(url, options)
  .then((res) => res.blob())
  .then((data) => console.log(data))
  .catch((error) => console.error(error))

// 发送请求，并解析响应体为二进制格式，然后处理数据
fetch(url, options)
  .then((res) => res.arrayBuffer())
  .then((data) => console.log(data))
  .catch((error) => console.error(error))
```

options 配置项

- method：请求方法，默认是 GET，可选值：GET、POST、PUT、DELETE、PATCH 等
- headers：请求头，默认是 {}
- body：请求体，默认是 null
- mode：请求模式，默认是 cors，可选值：cors、no-cors、same-origin
- credentials：请求凭证，默认是 same-origin
- cache：请求缓存，默认是 default
- redirect：请求重定向，默认是 follow
- referrer：请求来源，默认是 client
- referrerPolicy：请求来源策略，默认是 no-referrer-when-downgrade
- signal：请求取消，默认是 null

```javascript
fetch('https://api.example.com/data', {
  method: 'GET',
  headers: {
    'Content-Type': 'application/json',
    Authorization: 'Bearer token',
  },
  body: JSON.stringify({ key: 'value' }),
  mode: 'cors',
  credentials: 'same-origin',
  cache: 'default',
  redirect: 'follow',
  referrer: 'client',
  referrerPolicy: 'no-referrer-when-downgrade',
  signal: null,
})
  .then((res) => res.json())
  .then((data) => console.log(data))
  .catch((error) => console.error(error))
```

### Fetch 快速使用

```javascript
const url = 'https://api.example.com/data'
const options = {}
fetch(url, options)
  .then((res) => res.json())
  .then((data) => console.log(data))
  .catch((error) => console.error(error))
```

## Axios

Axios（ajax – I/O – system）是一个基于 Promise 的网络请求库，可以用于浏览器和 node.js。它支持多种请求方法，如 GET、POST、PUT、DELETE 等，支持多种请求头和请求体类型，支持请求拦截器和响应拦截器，支持自动转换 JSON 数据，支持取消请求等。

- Axios 英文官网：https://axios-http.com/
- Axios 中文官网：https://axios-http.com/zh/
- Axios 中文文档：https://axios.org.cn/
- Axios 中文文档：https://www.axios-http.cn/

### Axios 基础

CDN 安装

```html
<!-- 使用 jsDelivr CDN -->
<script src="https://cdn.jsdelivr.net.cn/npm/axios/dist/axios.min.js"></script>
<!-- 使用 unpkg CDN -->
<script src="https://unpkg.com/axios/dist/axios.min.js"></script>
```

Node 安装

- 使用 npm：`npm i axios`
- 使用 yarn：`yarn add axios`
- 使用 bower：`bower install axios`

基本使用

- `axios.get(url, config)` 发送 GET 请求

```javascript
const axios = require('axios')
axios
  .get(url)
  .then((res) => console.log(res))
  .catch((error) => console.error(error))

// 向给定ID的用户发起请求
axios
  .get('/user?ID=12345')
  .then((res) => console.log(res))
  .catch((error) => console.error(error))
// 上述请求也可以按以下方式完成（可选）
axios
  .get('/user', { params: { ID: 12345 } })
  .then((res) => console.log(res))
  .catch((error) => console.error(error))
```

- `axios.post(url, data, config)` 发送 POST 请求

```javascript
const axios = require('axios')
axios
  .post(url, data)
  .then((res) => console.log(res))
  .catch((error) => console.error(error))

axios
  .post('/user', {
    firstName: 'Fred',
    lastName: 'Flintstone',
  })
  .then((res) => console.log(res))
  .catch((error) => console.error(error))
```

- `axios({ config })` 发送任意请求
  - method 请求方法
  - url 请求地址
  - baseURL 基础地址（可以写baseURL或baseURL+url，也可以全部写url里面）
  - headers 请求头
  - params 请求参数
  - data 请求体
  - timeout 超时时间
  - responseType 响应类型
  - transformRequest 请求转换
  - transformResponse 响应转换

```javascript
const axios = require('axios')
axios({
  method: 'post',
  url: 'https://api.example.com/user/12345',
  data: { firstName: 'Fred', lastName: 'Flintstone' },
})
  .then((res) => console.log(res))
  .catch((error) => console.error(error))
```

axios 拦截器

- 请求拦截器

```javascript
const axios = require('axios')
axios.interceptors.request.use((config) => {
  // token 统一添加

})
```

- 响应拦截器

```javascript
const axios = require('axios')
axios.interceptors.response.use(
  (res) => {
    return res.data
  },
  (err) => {},
)
```

取消请求

- CancelToken
- 用于切换页面取消请求、防止重复请求

并发请求

- axios.all([])
- axios.spread()

错误处理

- err.response
- err.request
- err.message

## Token 和 JWT

- Token 是一种身份验证机制，用于验证用户身份。Token 通常是一个字符串，可以包含用户信息、过期时间、签名等。Token 可以存储在客户端，每次请求时携带 Token，服务器验证 Token 是否有效，从而验证用户身份。
- JWT（JSON Web Token）是一种基于 Token 的身份验证机制，用于验证用户身份。JWT 是一个 JSON 对象，包含三个部分：头部（Header）、载荷（Payload）和签名（Signature）。

JWT 使用步骤：

- 安装：`npm i jsonwebtoken`、`yarn add jsonwebtoken`
- 引入：`const jwt = require('jsonwebtoken')`
- 生成 Token：`const token = jwt.sign(payload, secret, options)`
- 验证 Token：`const decoded = jwt.verify(token, secret, options)`

```javascript
// 引入 jsonwebtoken 模块
const jwt = require('jsonwebtoken')
// 生成 Token
const payload = { username: 'admin', password: '123456' } // 需要加密的内容
const secret = 'mysecret' // 密钥：用于加密和解密
const options = { expiresIn: '1h' } // 配置：过期时间
const token = jwt.sign(payload, secret, options)
console.log(token)
// 验证 Token
const decoded = jwt.verify(token, secret, options)
console.log(decoded)
```

## 跨域问题

同源策略

- 同源策略：同源策略是一种安全策略，它限制了从一个源加载的文档或脚本如何与来自另一个源的资源进行交互。它是一种用于隔离潜在恶意文件的关键安全机制。
- 跨域请求：跨域请求是指从一个源向另一个源发送请求。由于同源策略的限制，跨域请求可能会被**浏览器**阻止。

解决跨域请求的方法

- CORS：跨域资源共享（Cross-Origin Resource Sharing，简称 CORS）是一种机制，它使用额外的 HTTP 头来告诉浏览器让运行在一个 origin（域）上的 Web 应用被准许访问来自不同源服务器上的指定的资源。当一个资源从与该资源不同的域、协议或端口请求一个资源时，资源会发起一个跨域 HTTP 请求。
  - 简单请求：请求方法为 GET、POST 或 HEAD 等
  - 复杂请求：请求方法为 PUT、DELETE 或 OPTIONS 等
  - OPTIONS 预检：复杂请求会先发送一个 OPTIONS 请求，称为预检请求，服务器会返回一个响应，告诉浏览器是否允许跨域请求。

```javascript
// CORS 服务器端设置响应头
res.setHeader('Access-Control-Allow-Origin', '*')
```

- JSONP：JSONP（JSON with Padding）是一种非官方的跨域数据交换协议。它通过动态创建 `<script>` 标签，并设置其 `src` 属性为跨域请求的 URL，从而实现跨域数据交换。由于 `<script>` 标签不受同源策略的限制，因此可以通过这种方式实现跨域请求。

```javascript
// JSONP
```

- 代理：代理是一种服务器，它接收客户端的请求，并将其转发到目标服务器，然后将目标服务器的响应返回给客户端。通过代理服务器，可以实现跨域请求。
- postMessage：postMessage 是一种 HTML5 提供的 API，用于在不同窗口、标签页或 iframe 之间进行通信。通过 postMessage，可以实现跨域通信。
- iframe：iframe 是一个 HTML 标签，用于在网页中嵌入另一个网页。通过 iframe，可以实现跨域请求。
