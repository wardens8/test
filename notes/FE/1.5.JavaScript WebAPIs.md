## DOM 简介

DOM：文档对象模型（Document Object Model，简称DOM），是W3C组织推荐的处理可扩展标记语言（HTML 或者XML）的标准编程接口。W3C已经定义了一系列的DOM接口，通过这些DOM接口可以改变网页的内容、结构和样式。

DOM 树：DOM树又称为文档树模型，把文档当做一个树形结构，文档的每一个组成部分都是树中的一个节点，并且每一个节点都是对象。

- 文档：一个页面就是一个文档，DOM中使用 `document` 表示
- 元素：页面中的所有标签都是元素，DOM中使用 `element` 表示
- 节点：网页中的所有内容都是节点（标签、属性、文本、注释等），DOM中使用 `node` 表示

网页中的所有内容都是节点（标签、属性、文本、注释等），在DOM中，节点使用node来表示。HTML DOM树中的所有节点均可通过JavaScript进行访问，所有HTML元素（节点）均可被修改，也可以创建或删除。一般地，节点至少拥有`nodeType`（节点类型）、`nodeName`（节点名称）和`nodeValue`（节点值）这三个基本属性：元素节点nodeType为1、属性节点nodeType为2、文本节点nodeType为3（文本节点包含文字、空格、换行等）。我们在实际开发中，节点操作主要操作的是元素节点。

## DOM 节点获取

- `getElementById(id)` 通过ID获取，返回一个元素对象，注意ID节点可以不获取直接使用，但是推荐先获取再使用
- `getElementsByTagName(tagName)` 通过标签名获取，返回一个包含全部元素对象的伪数组，会实时更新
- `getElementsByClassName(className)` 通过类名获取，返回一个包含全部元素对象的伪数组，会实时更新
- `getElementsByName(name)` 通过name属性获取，返回一个包含全部元素对象的伪数组，会实时更新
- `querySelector(selector)` 通过选择器获取，返回第一个匹配的元素对象（可以复合选择器），不会实时更新
- `querySelectorAll(selector)` 通过选择器获取，返回所有匹配的元素对象的伪数组（可以复合选择器），不会实时更新
- 注意：获取节点时，可以从 `document` 获取，也可以从父元素获取
- `document.documentElement` 获取html元素
- `document.head` 获取head元素
- `document.body` 获取body元素
- `document.title` 获取title元素

```javascript
// 获取元素
const div = document.getElementById('box')
const divs = document.getElementsByTagName('div')
const divs = document.getElementsByClassName('box')
const divs = document.getElementsByName('box')

const div = document.querySelector('.box')
const divs = document.querySelectorAll('.box')
```

## DOM 节点操作

父节点

- `elementNode.parentNode` 获取父节点
- `elementNode.parentElement` 获取父元素节点

```javascript
// 获取父节点，两种方法获得的节点完全一致
const div = document.querySelector('div')
console.log(div.parentNode) // 获取父节点
console.log(div.parentElement) // 获取父元素节点
```

子节点

- `parentNode.childNodes` 获取所有子节点（包括文本节点、注释节点等）
- `parentNode.firstChild` 获取第一个子节点
- `parentNode.lastChild` 获取最后一个子节点
- `parentNode.children` 获取所有子元素节点
- `parentNode.firstElementChild` 获取第一个子元素节点（IE9+）
- `parentNode.lastElementChild` 获取最后一个子元素节点（IE9+）

```javascript
// 获取子节点
const ul = document.querySelector('ul')
console.log(ul.childNodes) // 获取所有子节点
console.log(ul.firstChild) // 获取第一个子节点
console.log(ul.lastChild) // 获取最后一个子节点
console.log(ul.children) // 获取所有子元素节点
console.log(ul.firstElementChild) // 获取第一个子元素节点
console.log(ul.lastElementChild) // 获取最后一个子元素节点
```

兄弟节点

- `elementNode.nextSibling` 获取当前元素的下一个兄弟节点
- `elementNode.previousSibling` 获取当前元素上一个兄弟节点
- `elementNode.nextElementSibling` 获取当前元素下一个兄弟元素节点（IE9+）
- `elementNode.previousElementSibling` 获取当前元素上一个兄弟元素节点（IE9+）

```javascript
// 获取兄弟节点
const li = document.querySelector('li')
console.log(li.nextSibling) // 获取当前元素的下一个兄弟节点
console.log(li.previousSibling) // 获取当前元素上一个兄弟节点
console.log(li.nextElementSibling) // 获取当前元素下一个兄弟元素节点
console.log(li.previousElementSibling) // 获取当前元素上一个兄弟元素节点
```

创建节点

- `document.createElement(tagName)` 创建元素节点
- `document.createTextNode(content)` 创建文本节点
- `document.createComment(content)` 创建注释节点
- `document.write(content)` 向文档写入内容，页面会重绘（不推荐使用，了解即可）

```javascript
// 创建元素节点
const div = document.createElement('div')
// 创建文本节点
const text = document.createTextNode('hello world')
// 创建注释节点
const comment = document.createComment('hello world')
```

添加节点

- `parentNode.appendChild(child)` 将一个节点添加到指定父节点的子节点列表的末尾
- `parentNode.append(nodes/strings)` 将节点或字符串添加到指定父节点的子节点列表的末尾
- `parentNode.prepend(nodes/strings)` 将节点或字符串添加到指定父节点的子节点列表的开头
- `parentNode.insertBefore(newChild, refChild)` 在参考节点前插入一个节点
- `parentNode.replaceChild(newChild, oldChild)` 替换一个子节点

```javascript
// 添加元素节点
const ul = document.querySelector('ul')
const li = document.createElement('li')
ul.appendChild(li)

// 添加多个元素节点
const ul = document.querySelector('ul')
const li1 = document.createElement('li')
const li2 = document.createElement('li')
ul.append(li1, li2, 'hello world')

// 添加多个元素节点
const ul = document.querySelector('ul')
const li1 = document.createElement('li')
const li2 = document.createElement('li')
ul.prepend(li1, li2, 'hello world')

// 在参考节点前插入一个节点
const ul = document.querySelector('ul')
const li = document.createElement('li')
ul.insertBefore(li, ul.children[0])

// 替换一个子节点
const ul = document.querySelector('ul')
const li = document.createElement('li')
ul.replaceChild(li, ul.children[0])
```

- `element.insertAdjacentElement(position, element)` 在元素指定位置插入一个元素节点
- `element.insertAdjacentHTML(position, htmlString)` 在元素指定位置插入一个元素
- `element.insertAdjacentText(position, text)` 在元素指定位置插入一个文本

```javascript
// insertAdjacentElement
const div = document.querySelector('div')
const span = document.createElement('span')
span.innerText = 'Hello World'
div.insertAdjacentElement('beforebegin', span)

// insertAdjacentHTML
const div = document.querySelector('div')
div.insertAdjacentHTML('beforebegin', '<span>Hello World</span>')

// insertAdjacentText
const div = document.querySelector('div')
div.insertAdjacentText('beforeend', 'Hello World')
```

删除节点

- `elementNode.remove()` 删除自身节点
- `parentNode.removeChild(child)` 从DOM中删除一个子节点，返回删除的节点

```javascript
// 删除自身节点
const div = document.querySelector('div')
div.remove()

// 删除元素节点
const ul = document.querySelector('ul')
ul.removeChild(ul.children[0])
```

克隆节点（复制节点）

- `elementNode.cloneNode(deep)` 克隆一个节点，返回克隆的节点
  - `deep` 默认为 `false` 浅拷贝，只复制标签不复制里面的内容
  - `deep` 为 `true` 深拷贝，既复制标签又复制里面的内容

```javascript
// 克隆元素节点
const ul = document.querySelector('ul')
const li = ul.children[0].cloneNode(true)
ul.appendChild(li)
```

几种动态创建元素的效率对比

```javascript
const div = document.querySelector('div')
let time1 = +new Date() // 开始时间

// innerHTML 字符串拼串 = 2000ms左右
for (let i = 0; i < 1000; i++) {
  div.innerHTML += '<a href="#">Baidu</a>'
}

// createElement + appendChild 创建添加节点 = 5-6ms
for (let i = 0; i < 1000; i++) {
  const a = document.createElement('a')
  a.href = '#'
  a.innerText = 'Baidu'
  div.appendChild(a)
}

// insertAdjacentHTML = 5-6ms
for (let i = 0; i < 1000; i++) {
  div.insertAdjacentHTML('beforeend', '<a href="#">Baidu</a>')
}

// innerHTML + 数组join拼接 = 3-4ms
const arr = []
for (let i = 0; i < 1000; i++) {
  arr.push('<a href="#">Baidu</a>')
}
div.innerHTML = arr.join('')

let time2 = +new Date() // 结束时间
console.log(time2 - time1)
```

## DOM 元素操作

修改元素内容

- `element.innerHTML` 获取或设置元素内容，保留格式，包括标签（会出现XSS攻击）
- `element.innerText` 获取或设置元素内容，不保留格式，不包括标签（不会出现XSS攻击）
- `element.textContent` 获取或设置元素内容，保留格式，不包括标签（不会出现XSS攻击）

```javascript
// 获取元素内容
const div = document.querySelector('div')
console.log(div.innerHTML) // 获取元素内容，包括标签
console.log(div.innerText) // 获取元素内容，不包括标签
console.log(div.textContent) // 获取元素内容，不包括标签

// 设置元素内容
const div = document.querySelector('div')
div.innerHTML = '<strong>hello world</strong>'
div.innerText = '<strong>hello world</strong>'
div.textContent = '<strong>hello world</strong>'
```

修改元素属性

- `element.hasAttribute(attr)` 检查元素是否有某个属性
- `element.attr` 获取元素属性，主要控制本身的属性
- `element.getAttribute(attr)` 获取元素属性，主要控制自定义属性
- `element.attr = value` 设置元素属性
- `element.setAttribute(attr, value)` 设置元素属性
- `element.removeAttribute(attr)` 移除元素属性
- `element.dataset`/`element.dataset.*`/`element.dataset[*]` 获取 `data-*` 的自定义属性的方法，H5新增

```javascript
// 获取元素属性
const img = document.querySelector('img')
console.log(img.src) // 获取元素属性
console.log(img.getAttribute('src')) // 获取元素属性

// 设置元素属性
const img = document.querySelector('img')
img.src = 'https://www.baidu.com/img/flexible/logo/pc/result.png'
img.setAttribute('src', 'https://www.baidu.com/img/flexible/logo/pc/result.png')

// 移除元素属性
const img = document.querySelector('img')
img.removeAttribute('src')
```

```javascript
// 修改输入框属性
const ipt = document.querySelector('input')
ipt.type = 'checkbox'
ipt.name = 'sex'
ipt.value = 'male'
ipt.checked = true
ipt.disabled = true
ipt.title = 'sex'
ipt.id = 'sex'
ipt.className = 'sex'
```

修改元素样式

- `element.style` 获取元素内联样式或直接设置元素样式
- `getComputedStyle(element)` 获取元素所有生效的样式（包括内联样式、外部样式表样式），但是只读不可修改

```javascript
// 获取元素内联样式
const div = document.querySelector('div')
console.log(div.style) // 获取元素内联样式，但是无法得到css样式表中的样式

// 设置元素样式
const div = document.querySelector('div')
div.style.width = '100px'
div.style.height = '100px'
div.style.backgroundColor = 'red'

// 获取元素所有样式
const div = document.querySelector('div')
console.log(getComputedStyle(div)) // 获取元素所有样式
```

修改元素类名

- `element.className` 获取或设置元素类名
- `element.classList` 获取元素类名列表
- `element.classList.contains(token)` 检查是否包含类名
- `element.classList.add(token1, token2, ...)` 添加类名
- `element.classList.remove(token1, token2, ...)` 移除类名
- `element.classList.replace(oldToken, newToken)` 检查是否包含类名
- `element.classList.toggle(token)` 切换类名

```javascript
// 获取元素类名
const div = document.querySelector('div')
console.log(div.className) // 获取元素类名
console.log(div.classList) // 获取元素类名列表

// 设置元素类名
const div = document.querySelector('div')
div.className = 'box'
div.className = 'box box1 box2'
div.classList.contains('box')
div.classList.add('box1')
div.classList.remove('box1')
div.classList.replace('box1', 'box2')
div.classList.toggle('box1')
```

## DOM 事件绑定

事件

- JavaScript 与 HTML 之间的交互是通过事件实现的
- 事件三要素：事件源、事件类型、事件处理程序
  - 事件源：事件被触发的对象，比如：按钮
  - 事件类型：事件被触发的方式，比如：鼠标点击
  - 事件处理程序：事件触发后要执行的代码，通过函数赋值的形式，比如：弹出一个框

常见的事件绑定（事件注册）

- `onclick` 鼠标点击左键触发
- `onmousedown` 鼠标按下触发
- `onmouseup` 鼠标弹起触发
- `onmousemove` 鼠标移动触发
- `onmouseover` 鼠标经过触发
- `onmouseout` 鼠标离开触发
- `onfocus` 获得鼠标焦点触发
- `onblur` 失去鼠标焦点触发

```javascript
// 事件绑定
const btn = document.querySelector('button')
btn.onclick = function () {
  alert('按钮被点击了')
}
// 事件解绑
btn.onclick = null
```

## DOM 事件高级

由于传统的事件绑定方式存在一些问题，注册的事件具有唯一性（后面的会覆盖前面的），因此现代浏览器推荐使用事件监听器（事件注册）的方式绑定事件

事件绑定（事件注册）和事件解绑（事件删除）

- `elementNode.addEventListener(event, function, [useCapture])` 添加事件监听器
- `elementNode.removeEventListener(event, function, [useCapture])` 移除事件监听器
- `elementNode.attachEvent(on-event, function)` 添加事件监听器（IE8及以下）
- `elementNode.detachEvent(on-event, function)` 移除事件监听器（IE8及以下）
- `event` 有以下方式：click鼠标点击、mousedown鼠标按下、mouseup鼠标弹起、mousemove鼠标移动、mouseover鼠标经过、mouseout鼠标离开、keydown键盘按下、keyup键盘弹起、keypress键盘按下并弹起

```javascript
// 添加事件监听器
const div = document.querySelector('div')
div.addEventListener('click', function () {
  console.log('点击了div')
})

// 移除事件监听器
const div = document.querySelector('div')
const fn = function () {
  console.log('点击了div')
}
div.addEventListener('click', fn)
div.removeEventListener('click', fn)

// IE8及以下添加事件监听器
const div = document.querySelector('div')
div.attachEvent('onclick', function () {
  console.log('点击了div')
})

// IE8及以下移除事件监听器
const div = document.querySelector('div')
const fn = function () {
  console.log('点击了div')
}
div.attachEvent('onclick', fn)
div.detachEvent('onclick', fn)
```

文档的加载

- load 事件：当页面中的所有资源（包括图片、视频等）加载完成时，才会触发 load 事件，因此 window 对象上的 load 事件是页面加载完成时触发的事件
- DOMContentLoaded 事件：当初始的 HTML 文档被完全加载和解析完成时，DOMContentLoaded 事件被触发，不需要等待样式表、图片和子框架的加载完成
- defer 属性：defer 属性规定是否对脚本的内容进行延迟执行，延迟执行意味着将脚本放在文档的底部，在文档解析完成后执行

```javascript
window.onload = function () {
  // 页面加载完成，才执行回调函数
}
window.addEventListener('load', function () {
  // 页面加载完成，才执行回调函数
})
```

```javascript
document.addEventListener('DOMContentLoaded', function () {
  // DOM树加载完成，才执行回调函数
})
```

```html
<!-- 该标签会在页面加载完成后执行 -->
<script defer src="script.js"></script>
```

DOM事件流：事件捕获和事件冒泡

- DOM事件流：事件流描述的是从页面中接收事件的顺序，事件发生时会在元素节点之间按照特定的顺序传播，这个传播过程即DOM事件流
- 事件捕获：`【Document --> Node】` 网景最早提出，事件从最不具体的DOM最顶层节点开始，然后逐级向下传播到最具体的节点
- 事件冒泡：`【Node --> Document】` IE最早提出，事件从最具体的节点开始，然后逐级向上传播到最不具体的DOM最顶层节点
- 事件捕获和事件冒泡是两种事件传播的机制，现代浏览器默认采用事件冒泡机制，但可以通过事件监听器的第三个参数来设置事件捕获，第三个参数默认为 `false`，表示事件冒泡，设置为 `true` 表示事件捕获

```javascript
// 事件冒泡 div --> body --> html --> document
const div = document.querySelector('div')
div.addEventListener('click', function () {})
document.body.addEventListener('click', function () {})
document.documentElement.addEventListener('click', function () {})
document.addEventListener('click', function () {})

// 事件捕获 document --> html --> body --> div
const div = document.querySelector('div')
div.addEventListener('click', function () {}, true)
document.body.addEventListener('click', function () {}, true)
document.documentElement.addEventListener('click', function () {}, true)
document.addEventListener('click', function () {}, true)
```

事件对象

- `event` 事件对象，事件触发时，浏览器会创建一个事件对象，并将事件相关信息放入事件对象中，传递给事件处理函数
- `event.target` 事件目标，事件触发时，被点击的元素
- `event.currentTarget` 事件当前目标，事件触发时，被点击的元素
- `event.type` 事件类型，事件触发时，事件类型（click、mousedown、mouseup、mousemove、mouseover、mouseout、keydown、keyup、keypress）

```javascript
// 事件对象
const div = document.querySelector('div')
div.addEventListener('click', function (event) {
  console.log(event)
})

// 事件目标
const div = document.querySelector('div')
div.addEventListener('click', function (event) {
  console.log(event.target)
})

// 事件当前目标
const div = document.querySelector('div')
div.addEventListener('click', function (event) {
  console.log(event.currentTarget)
})

// 事件类型
const div = document.querySelector('div')
div.addEventListener('click', function (event) {
  console.log(event.type)
})
```

- `event.preventDefault()` 阻止默认行为，比如阻止链接跳转、阻止表单提交
- `event.returnValue` 设置事件返回值阻止默认行为，比如阻止链接跳转、阻止表单提交（IE678）
- `event.stopPropagation()` 阻止事件冒泡，阻止事件冒泡后，当前元素的事件会触发，而父元素的事件不会触发
- `event.cancelBubble` 设置事件返回值阻止事件冒泡，阻止事件冒泡后，当前元素的事件会触发，而父元素的事件不会触发（IE678）

```javascript
// 阻止默认行为
const a = document.querySelector('a')
a.addEventListener('click', function (event) {
  event.preventDefault()
  event.returnValue = false
})

// 阻止事件冒泡
const div = document.querySelector('div')
div.addEventListener('click', function (event) {
  event.stopPropagation()
  event.cancelBubble = true
})
```

事件委托（事件代理、事件委派）

- 事件委托：利用事件冒泡的原理，将事件绑定到父元素上，通过判断事件目标来处理事件
- 事件委托的优点：减少事件绑定次数，提高性能，动态添加的元素也可以绑定事件

```javascript
// 事件委托
const ul = document.querySelector('ul')
ul.addEventListener('click', function (event) {
  // 判断事件目标
  if (event.target.tagName === 'li') {
    console.log('点击了li')
  }
})
```

鼠标事件 MouseEvent / PointerEvent

- `click` 鼠标点击
- `scroll` 滚动条滚动事件
- `wheel` 滚轮事件
- `dblclick` 鼠标双击
- `contextmenu` 右键菜单事件
- `selectstart` 开始选中文字事件
- `mousedown`/`pointerdown` 鼠标按下
- `mouseup`/`pointerup` 鼠标弹起
- `mousemove`/`pointermove` 鼠标移动

## DOM 网页特效

元素偏移量 offset 系列（width+padding+border）

- offset翻译过来就是偏移量，我们使用offset系列相关属性可以动态的得到该元素的位置（偏移）、大小等
- `element.offsetParent` 获取元素的父元素
- `element.offsetWidth`/`element.offsetHeight` 获取元素的宽高
- `element.offsetTop`/`element.offsetLeft` 获取元素距离带定位的父元素的位置

```javascript
// 获取元素的父元素

console.log(element.offsetHeight)

// 获取元素距离父元素的位置
console.log(element.offsetTop)
console.log(element.offsetLeft)
```

> offset和style的区别：offset可以获取所有样式，包括内嵌和外链样式，style只能获取行内样式；offset是只读的，不可更改，style是可读写的，可以更改。所以，如果想要获取样式使用offset，想要更改样式，使用style。

元素可视区 client 系列（width+padding）

- client翻译过来就是客户端，我们使用client系列的相关属性来获取元素可视区的相关信息。通过client系列 的相关属性可以动态的得到该元素可视区的边框大小、元素大小等（可视区指元素被浏览器窗口包裹的区域）
- `element.clientWidth`/`element.clientHeight` 获取元素的宽高
- `element.clientTop`/`element.clientLeft` 获取元素边框的宽高


# JavaScript BOM

## BOM 简介

BOM：浏览器对象模型（Browser Object Model，简称BOM），它提供了独立于内容而与浏览器窗口进行交互的对象，其核心对象是window。BOM由一系列相关的对象构成，并且每个对象都提供了很多方法与属性。BOM缺乏标准，JavaScript语法的标准化组织是ECMA，DOM的标准化组织是W3C，BOM最初是Netscape浏览器标准的一部分。

## Window 对象

- `window` 对象是 BOM 的核心对象，它表示浏览器窗口，并且是 JavaScript 的全局对象，因此所有全局变量和函数都是 window 对象的属性和方法
- `window.onload` 页面加载完成事件，当页面完全加载完成后触发，包括页面上的所有依赖资源（如图片、样式表等）
- `window.onresize` 窗口大小改变事件，当窗口大小改变时触发
- `window.onscroll` 窗口滚动事件，当窗口滚动时触发

```javascript
// 页面加载完成
window.onload = function () {}
window.addEventListener('load', function () {})

// DOM加载完成
document.addEventListener('DOMContentLoaded', function () {})

// 窗口大小改变
window.onresize = function () {}
window.addEventListener('resize', function () {})

// 窗口滚动
window.onscroll = function () {}
window.addEventListener('scroll', function () {})
```

- `window.innerWidth`/`window.innerHeight` 浏览器窗口的宽高
- `window.pageXOffset`/`window.pageYOffset` 浏览器窗口的滚动距离
- `window.screenX`/`window.screenY` 浏览器窗口相对于屏幕的坐标
- `window.screenLeft`/`window.screenTop` 浏览器窗口相对于屏幕的坐标
- `window.screen.width`/`window.screen.height` 屏幕的宽高
- `window.screen.availWidth`/`window.screen.availHeight` 屏幕的可用宽高（除去任务栏等）

```javascript

console.log(window.innerHeight)

console.log(window.pageXOffset)
console.log(window.pageYOffset)

// 获取窗口位置

console.log(window.screenY)

// 获取窗口位置

console.log(window.screenTop)

console.log(window.screen.height)

// 获取窗口分辨率
```

- `window.open(url, target)` 打开一个新窗口或标签页
- `window.close()` 关闭当前窗口或标签页
- `window.moveTo(x, y)` 移动当前窗口或标签页到指定位置
- `window.resizeTo(width, height)` 调整当前窗口或标签页的大小

## Location 对象

URL

- 统一资源定位符（Uniform Resource Locator，简称URL），是互联网上标准资源的地址。互联网上的每个文件都有一个唯一的URL，它包含的信息指出文件的位置以及浏览器应该怎么处理它
- URL的一般语法格式为：
  - `protocol://host[:port]/path/[?query]#fragment`
  - `http://www.itcast.cn/index.html?name=andy&age=18#link`
- protocol：通信协议，常用的http，ftp，maito等
- host：主机（域名），www.itheima.com
- port：端口号，可选，省略时使用方案的默认端口，如http的默认端口为80
- path：路径，由0个或多个'/'符号隔开的字符串，一般用来表示主机上的一个目录或文件地址
- query：参数，以键值对的形式，通过&符号分隔开来
- fragment：片段，#后面内容，常见于链接、锚点

window 对象给我们提供了一个 location 属性用于获取或设置窗体的 URL，并且可以用于解析 URL。因为这个属性返回的是一个对象，所以我们将这个属性也称为 location 对象

- `location.href` 获取或设置整个 URL
- `location.host` 获取或设置主机名和端口号
- `location.pathname` 获取或设置路径
- `location.search` 获取或设置查询字符串
- `location.hash` 获取或设置锚点
- `location.assign(url)` 加载新的文档（也称为重定向）
- `location.replace(url)` 加载新的文档，并且不保留历史记录
- `location.reload([true])` 重新加载当前文档，等同于F5刷新

```javascript
// 获取location对象的所有属性和方法
console.log(location)

// 获取整个URL
console.log(location.href)
// 设置整个URL
location.href = 'https://www.baidu.com'

// 获取主机名和端口号
console.log(location.host)
// 设置主机名和端口号
location.host = 'www.baidu.com'

// 获取路径
console.log(location.pathname)
// 设置路径
location.pathname = '/index.html'

// 获取查询字符串
console.log(location.search)
// 设置查询字符串
location.search = '?name=andy&age=18'

// 获取锚点
console.log(location.hash)
// 设置锚点
location.hash = '#link'

// 加载新的文档
location.assign('https://www.baidu.com')
// 加载新的文档，并且不保留历史记录
location.replace('https://www.baidu.com')
// 重新加载当前文档，且强制清理缓存
location.reload(true)
// 重新加载当前文档，且不清理缓存
location.reload()
```

## History 对象

History 对象与浏览器历史记录进行交互。该对象包含用户（在浏览器窗口中）访问过的URL

- `history.back()` 后退一页
- `history.forward()` 前进一页
- `history.go(n)` 跳转到历史记录指定页，n为正数表示前进n个，n为负数表示后退n个

```javascript

// 后退一页

// 前进一页
history.forward()
// 跳转到历史记录指定页
history.go(-1)
```

## 定时器

> 注意：定时器全部为异步，会进入宏任务队列，不会阻塞代码执行

- `setTimeout(function, delay)` 在指定的毫秒数后执行函数或指定的代码
- `setInterval(function, delay)` 按照指定的时间间隔重复执行代码或函数
- 注意：定时器里面的 `this` 指向 `window`

```javascript
// 1秒后执行
setTimeout(() => {
  console.log('1秒后执行')
}, 1000)

// 每隔1秒执行
setInterval(() => {
  console.log('每隔1秒执行')
}, 1000)
```

- `clearTimeout(timerID)` 清除由 `setTimeout` 设置的定时器
- `clearInterval(timerID)` 清除由 `setInterval` 设置的定时器

```javascript
// 清除定时器
const timer1 = setTimeout(() => {
  console.log('1秒后执行')
}, 1000)
clearTimeout(timer1)

// 清除定时器
const timer2 = setInterval(() => {
  console.log('每隔1秒执行')
}, 1000)
clearInterval(timer2)
```

# Web 存储

## localStorage

- `window.localStorage` 本地存储，用于长久保存整个网站的数据，保存的数据没有过期时间，直到手动去除
- `localStorage.length` 返回数据项的数量
- `localStorage.key(n)` 返回存储中第n个键的名称
- `localStorage.getItem('key')` 获取
- `localStorage.setItem('key', 'value')` 存储
- `localStorage.removeItem('key')` 清除
- `localStorage.clear()` 清空

## sessionStorage

- `window.sessionStorage` 会话存储，用于临时保存同一窗口（或标签页）的数据，在关闭窗口或标签页之后将会删除这些数据
- `sessionStorage.length` 返回数据项的数量
- `sessionStorage.key(n)` 返回存储中第n个键的名称
- `sessionStorage.getItem('key')` 获取
- `sessionStorage.setItem('key', 'value')` 存储
- `sessionStorage.removeItem('key')` 清除
- `sessionStorage.clear()` 清空
