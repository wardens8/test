## Node 基础

### Node 简介

Node.js 是一个构建在V8引擎之上的 JavaScript 运行环境。它使得 JavaScript 可以运行在浏览器以外的地方。相对于大部分的服务端语言来说，Node.js 有很大的不同，它采用了单线程，且通过异步的方式来处理并发的问题。

### 模块化规范

- 早期的网页中，是没有一个实质的模块规范的。我们实现模块化的方式，就是最原始的通过`script`标签来引入多个 JavaScript 文件
- 问题：无法选择要引入模块的哪些内容；在复杂的模块场景下非常容易出错......
- 于是，我们就继续在 JavaScript 中引入一个模块化的解决方案。在 node 中，默认支持的模块化规范叫做 `CommonJS`

#### CommonJS 规范

模块

- `.js` 文件作为模块：在 CommonJS 中，一个 js 文件就是一个模块。
- `.cjs` 文件作为模块：在 CommonJS 中，一个 cjs 文件也是一个模块。`.cjs`为扩展名，表示是一个 CommonJS 标准的模块。注意`.cjs`作为扩展名时，必须写全。
- 文件夹作为模块：当我们使用一个文件夹作为模块时，文件夹中必须有一个模块的主文件。如果文件夹中含有`package.json`文件且文件中设置`main`属性，则`main`属性指定的文件会成为主文件，导入模块时就是导入该文件。如果没有`package.json`，则 node 会按照`index.js`、`index.node`的顺序寻找主文件。

```javascript
require('./m1.js')
require('./m2.cjs')

require('./hello')
require('./hello/index.js') // 此处的index.js可省略
require('./hello/hello.js')
```

导入模块

- 使用 `require("模块的路径")` 函数来导入模块
  - 导入自定义模块时，模块名要以 `./` 或 `../` 开头，扩展名可以省略。如果省略的 js 文件的扩展名，node 会自动为文件补全扩展名，`.js --> .json --> .node（特殊）`
  - 导入核心模块时，直接写核心模块的名字即可，也可以在核心模块前添加`node:xxx`
- 按需导入
  - 直接用属性的方式进行导入
  - 可以使用解构赋值的方法进行按需导入

```javascript
// 正常导入
const m1 = require('./m1.js')
const m1 = require('./m1')

const path = require('path')
const path = require('node:path')

// 按需导入
const obj = require('./m3')
const name = require('./m3').name

const { name, age, gender } = require('./m3')
console.log(name, age, gender)
```

导出模块

- 在定义模块时，模块中的内容默认是不能被外部看到的，可以通过 `exports` 来设置要向外部暴露的内容
- 当我们在其他模块中引入当前模块时，`require` 函数返回的就是 `exports`，可以将希望暴露给外部模块的内容设置为 `exports` 的属性
- 访问 `exports` 的方式有两种：`exports` 和 `module.exports`
  - 可以通过 `exports` 一个一个的导出值
  - 也可以直接通过 `module.exports` 同时导出多个值
- 注意：`module.exports` 和 `exports` 最开始是指向同一个空对象的，当 `module.exports` 被赋值后，二者就不再指向同一个对象了，此时 `module.exports` 指向向外抛出的对象，而 `exports` 仍然指向旧的空对象，所以向外抛出时，最好使用同一种方式，要么都用 `exports`，要么都用 `module.exports`。

```javascript
// 第一种方式
exports.a = '孙悟空'
exports.b = { name: '白骨精' }
exports.c = function fn() {
  console.log('哈哈')
}
//运行结果：{a: '孙悟空', b: {…}, c: ƒ}

// 第二种方式
module.exports = {
  a: '哈哈',
  b: [1, 3, 5, 7],
  c: () => console.log(111),
}
//运行结果：{a: '哈哈', b: Array(4), c: ƒ}
```

原理：每一个 CommonJS 模块在执行时，外层都会被套上一个函数；所以我们之所以能在 CommonJS 模块中使用 `exports`、`require` 并不是因为它们是全局变量。它们实际上以参数的形式传递进模块的

- `exports` 用来设置模块向外部暴露的内容
- `require` 用来引入模块的方法
- `module` 当前模块的引用
- `__filename` 表示当前模块的绝对路径
- `__dirname` 表示当前模块所在目录的绝对路径

```javascript
;(function (exports, require, module, __filename, __dirname) {
  // 模块代码会被放到这里
})
```

#### ES Module（ESM）规范

- 2015年随着ES6标准的发布，ES的内置模块化系统也应运而生，并且在Node.js中同样也支持ES标准的模块化
- 默认情况下，node中的默认模块化标准是CommonJS，要想使用ES模块化，可以采用以下两种方案
  - 使用 `.mjs` 作为扩展名
  - 修改 `package.json` 将模块化规范设置为ES模块。当我们设置 `"type": "module"` 当前项目下所有的js文件都默认为ES module

导入模块

- 使用 `import "模块的路径"` 函数来导入模块，扩展名不可以省略
- 使用 `import { 所需的内容 } from "模块的路径"` 函数来导入模块所需内容
- 使用 `import { a as name } from "模块的路径"` 函数来导入模块所需内容，把a变量重命名为name
- 使用 `import defaultExport from "模块的路径"` 函数来导入模块默认导出内容，命名为defaultExport（可任意起名）。
- 导入核心模块时，直接写核心模块的名字即可，也可以在核心模块前添加 `node:xxx`
- 注意：通过 ES 模块化，`import` 导入的值全部都是常量，无法更改，但是可以更改其属性。ES 模块都是运行在严格模式下的。ES 模块化，在浏览器中同样支持，但是通常我们不会直接使用，通常都会结合打包工具使用。

```javascript
// 导入默认导出
import defaultExport from "module-name.js"

// 将所有模块导入到指定命名空间中
import * as name from "module-name.js"

// 导入模块中的指定内容
import { export1 } from "module-name.js"
import { export1 , export2 } from "module-name.js"

// 以指定别名导入模块中的指定内容
import { export1 as alias1 } from "module-name.js"
import { export1 , export2 as alias2 , [...] } from "module-name.js"

// 导入默认和其他内容
import defaultExport, { export1 [ , [...] ] } from "module-name.js"
import defaultExport, * as name from "module-name.js"

// 导入模块
import "module-name.js"
```

导出模块

- 直接命名导出 `export let xxx;`
- 默认导出 `export default 值;`，一个模块中只有一个默认导出

```javascript
// 导出变量（命名导出）
export let name1, name2, …, nameN
export let name1 = …, name2 = …, …, nameN

// 导出函数（命名导出）
export function functionName(){...}

// 导出类（命名导出）
export class ClassName {...}

// 导出一组
export { name1, name2, …, nameN }

// 重命名导出
export { variable1 as name1, variable2 as name2, …, nameN }

// 解构赋值后导出
export const { name1, name2: bar } = o

// 默认导出
export default expression;
export default function (…) { … } // also class, function
export default function name1(…) { … } // also class, function
export { name1 as default, … }

// 聚合模块
export * from …; // 将其他模块中的全部内容导出（除了default）
export * as name1 from …; // ECMAScript® 2O20 将其他模块中的全部内容以指定别名导出
export { name1, name2, …, nameN } from …; // 将其他模块中的指定内容导出
export { import1 as name1, import2 as name2, …, nameN } from …; // 将其他模块中的指定内容重命名导出
export { default, … } from …;
```

### 全局对象与核心全局变量

全局对象

- 在ES模块中，通用的的全局对象是 globalThis
- 在浏览器中，全局对象是 window
- 在 Node 中，全局对象是 global

process 进程对象

- process 模块用来表示和控制当前的 node 进程。通过该对象可以获取进程的信息，或者对进程做各种操作
- process 是一个全局变量，可以直接使用
- `process.argv` 获取命令行参数，返回一个数组，第一个元素是 node 的执行路径，第二个元素是当前执行的文件路径，剩余元素是运行时传递的参数
- `process.env` 获取环境变量，返回一个对象，包含了当前环境中的所有变量
- `process.pid` 获取当前进程的进程 ID
- `process.platform` 获取当前系统平台：win32 / darwin / linux
- `process.version` 获取当前 node 的版本号
- `process.cwd()` 获取当前工作目录
- `process.on('exit', callback)` 监听进程退出事件，当进程退出时，会触发 exit 事件，并执行 callback 函数
- `process.on('uncaughtException', callback(error))` 监听未捕获异常事件，当进程抛出异常时，会触发 uncaughtException 事件，并执行 callback 函数
- `process.exit()` 结束当前进程，终止 node，0 表示正常退出，非 0 表示异常退出
- `process.memoryUsage()` 获取当前进程的内存使用情况，返回一个对象，包含了 rss、heapTotal、heapUsed、external 等属性
- `process.nextTick(callback[, …args])` 向 tick 任务队列中添加任务。
  - **tick 队列中的代码，会在下一次事件循环之前执行，且会在微任务队列和宏任务队列中任务之前执行**。

定时器

- `setTimeout(callback, delay[, …args])`：在 delay 毫秒后执行 callback 函数
- `setInterval(callback, delay[, …args])`：每隔 delay 毫秒执行一次 callback 函数
- `setImmediate(callback[, …args])`：在当前 tick 队列中的任务执行完毕后，立即执行 callback 函数

常用全局函数

- `console`：用来打印日志信息，类似于浏览器的 console
- `Buffer`：是 Node.js 用来处理二进制数据的类，它能够存储二进制数据，并且提供了处理二进制数据的工具方法。Buffer 类在 Node.js 中是一个全局变量，无需通过 require() 引入。

## Node 核心模块

核心模块是 node 中的内置模块，这些模块有的可以直接在 node 中使用，有的直接引入即可使用。核心模块有很多，不能一一介绍，这里只是简单介绍几个模块：

### path 模块

path 模块可以帮助我们获取文件（夹）的路径。通过 path 可以用来获取各种路径

- 要使用 path，需要先对其进行导入 `const path = require("node:path")`
- `path.join([…paths])`：拼接路径（自动处理斜杠）
- `path.resolve([…paths])` 生成一个绝对路径（遇到斜杠就处理为根目录）
- `path.dirname([path])`：获取文件所在目录
- `path.basename([path])`：获取文件名
- `path.extname([path])`：获取扩展名（.js/.html 等）
- 注意：我们通过不同的方式执行代码时，它的工作目录是有可能发生变化的。如果将一个相对路径作为参数，则 resolve 会自动将其转换为绝对路径，此时根据工作目录的不同，它所产生的绝对路径也不同。所以，一般会将一个绝对路径作为第一个参数，一个相对路径作为第二个参数，这样它会自动计算出最终的路径。`const result = path.resolve(__dirname, "./hello.js")`。此时不必担心因执行代码的方式不同而出现工作目录的差异，也不用担心因工作文件夹位置改变或使用系统改变造成的困扰。所以，以后在使用路径时，尽量通过`path.resolve()`来生成路径。

```javascript
const path = require('node:path')
// 假设当前文件根目录路径为 D:\xxx\xxx\
const result1 = path.join('./a/b/c.js') // \a\b\c.js
const result2 = path.resolve('./a/b/c.js') // D:\xxx\xxx\a\b\c.js

const result1 = path.join(__dirname, '/a/b/c.js') // D:\xxx\xxx\a\b\c.js
const result2 = path.resolve(__dirname, '/a/b/c.js') // D:\a\b\c.js

const result1 = path.join(__dirname, './a/b/c.js') // D:\xxx\xxx\a\b\c.js
const result2 = path.resolve(__dirname, './a/b/c.js') // D:\xxx\xxx\a\b\c.js

const result1 = path.join(__dirname, '../a/b/c.js') // D:\xxx\a\b\c.js
const result2 = path.resolve(__dirname, '../a/b/c.js') // D:\xxx\a\b\c.js
```

### fs 模块

fs 模块（File System）可以帮助我们读取磁盘中的文件。文件操作也就是所谓的 I/O，input/output

- 要使用 fs 模块，需要先对其进行导入 `const fs = require("node:fs")`，或导入 Promise 的 fs 模块 `const fs = require("node:fs/promises")`
- `readFileSync([文件路径])` 同步的读取文件的方法，会阻塞后边代码的执行。当我们通过 fs 模块读取磁盘中的数据时，读取到的数据总会以 Buffer 对象的形式返回。（Buffer）是一个临时用来存储数据的缓冲区。

```javascript
// fs.readFileSync()
const fs = require('node:fs')
const path = require('node:path')
const buffer = fs.readFileSync(path.resolve(__dirname, './hello.txt'))
console.log(buffer.toString())
```

- `fs.readFile([文件路径], [回调函数])`读取文件

```javascript
// fs.readfile()
const fs = require('node:fs')
const path = require('node:path')
fs.readFile(path.resolve(__dirname, './hello.txt'), (err, buffer) => {
  if (err) {
    console.log('出错了~')
  } else {
    console.log(buffer.toString())
  }
})

// Promise版本的fs的方法
const fs = require('node:fs/promises')
const path = require('node:path')
fs.readFile(path.resolve(__dirname, './hello.txt'))
  .then((buffer) => {
    console.log(buffer.toString())
  })
  .catch((error) => {
    console.log('出错了~')
  })

// async和await版本的fs的方法
const fs = require('node:fs/promises')
const path = require('node:path')
;(async () => {
  try {
    const buffer = await fs.readFile(path.resolve(__dirname, './hello.txt'))
    console.log(buffer.toString())
  } catch (error) {
    console.log('出错了~~')
  }
})()
```

- `fs.appendFile([文件路径], [内容])`若没有该文件则创建新文件并将数据写入，若有改文件则将数据添加到已有文件中

```javascript
const fs = require('node:fs/promises')
const path = require('node:path')
fs.appendFile(path.resolve(__dirname, './hello123.txt'), '超哥讲的真不错').then((r) => {
  // 注意此时的then并没有返回值，只是作为提示出现
  console.log('添加成功')
})
```

- `fs.writeFile([文件路径], [内容])`若没有该文件则创建新文件并将数据写入，若有改文件则将数据覆盖到已有文件中

```javascript
const fs = require('node:fs/promises')
const path = require('node:path')
fs.writeFile(path.resolve(__dirname, './hello123.txt'), '超哥讲的真不错').then((r) => {
  // 注意此时的then并没有返回值，只是作为提示出现
  console.log('添加成功')
})
```

- `fs.mkdir([目录名称])`创建一个新目录。mkdir 可以接收一个配置对象作为第二个参数，通过该对象可以对方法的功能进行配置：recursive 默认值为 false，设置 true 以后，会自动创建不存在的上一级目录

```javascript
const fs = require('node:fs/promises')
const path = require('node:path')
fs.mkdir(path.resolve(__dirname, './hello/abc'), { recursive: true })
  .then((r) => {
    // 注意此时的then并没有返回值，只是作为提示出现
    console.log('操作成功~')
  })
  .catch((err) => {
    console.log('创建失败', err)
  })
```

- `fs.rmdir([目录名称])`删除目录。rmdir 也可以接收一个配置对象作为第二个参数，通过该对象可以对方法的功能进行配置：recursive 默认值为 false，设置 true 以后，会删除多存在的下一级目录。注意：该方法在后续可能会删除，后续将使用`fs.rm()`方法来进行删除

```javascript
const fs = require('node:fs/promises')
const path = require('node:path')
fs.rmdir(path.resolve(__dirname, './hello'), { recursive: true }).then((r) => {
  // 注意此时的then并没有返回值，只是作为提示出现
  console.log('删除成功')
})
```

- `fs.rm([文件名称或目录名称])`删除文件。使用方法同`fs.rmdir()`，rm 可以接收一个配置对象作为第二个参数，通过该对象可以对方法的功能进行配置：recursive 默认值为 false，设置 true 以后，会删除多存在的下一级目录。

```javascript
const fs = require('node:fs/promises')
const path = require('node:path')
fs.rm(path.resolve(__dirname, './hello'), { recursive: true }).then((r) => {
  // 注意此时的then并没有返回值，只是作为提示出现
  console.log('删除成功')
})
```

- `fs.rename([旧文件路径文件名], [新文件路径文件名])`重命名（也可以作剪切使用）。

```javascript
const fs = require('node:fs/promises')
const path = require('node:path')
fs.rename(path.resolve(__dirname, '../an.jpg'), path.resolve(__dirname, './an.jpg')).then((r) => {
  // 注意此时的then并没有返回值，只是作为提示出现
  console.log('重命名成功')
})
```

- `fs.copyFile([旧文件路径文件名], [新文件路径文件名])`复制文件（复制）。

```javascript
const fs = require('node:fs/promises')
const path = require('node:path')
fs.copyFile(path.resolve(__dirname, '../an.jpg'), path.resolve(__dirname, './an.jpg')).then((r) => {
  // 注意此时的then并没有返回值，只是作为提示出现
  console.log('复制成功')
})
```

### http 模块

http 模块（HyperText Transfer Protocol）可以帮助我们创建网络服务。

- 要使用 http 模块，需要先对其进行导入 `const http = require("node:http")`
- 模块顶级属性/方法
  - `http.createServer()` 创建 HTTP 服务器
  - `http.request()` 发起客户端请求（可自定义 method、header、body）
  - `http.get()` 快捷发起 GET 请求，自动调用 req.end ()
  - `http.METHODS` 返回所有支持的 HTTP 方法数组
  - `http.STATUS_CODES` 返回所有状态码对应的描述信息
  - `http.globalAgent` 默认的 HTTP 代理，管理连接复用
- server 对象（服务器）
  - `server.listen(port, host, callback)` 启动服务器，监听端口
  - `server.close(callback)` 关闭服务器，停止接收新连接
  - `server.listening` 判断服务器是否正在监听
  - `server.maxHeadersCount` 限制最大请求头数量
- req 请求对象（IncomingMessage）
  - `req.method` 请求方法：GET/POST/PUT…
  - `req.url` 请求路径：/ /api/user…
  - `req.headers` 请求头对象：所有请求头信息
  - `req.httpVersion` HTTP 版本：1.1 / 2.0
  - `req.socket` 底层 socket 对象
  - `req.on('data', chunk => {})` 接收请求体数据
  - `req.on('end', () => {})` 请求体接收完毕
  - `req.destroy()` 销毁请求，释放资源
- res 响应对象（ServerResponse）
  - `res.statusCode` 设置响应状态码：200、404、500…
  - `res.statusMessage` 状态码描述文字
  - `res.headersSent` 判断响应头是否已发送
  - `res.setHeader(name, value)` 设置单个响应头
  - `res.getHeader(name)` 获取已设置的响应头
  - `res.removeHeader(name)` 删除响应头
  - `res.writeHead(statusCode, headers)` 一次性设置状态码 + 多个响应头
  - `res.write(data)` 写入响应内容，可多次调用
  - `res.end(data)` 结束响应，返回数据给客户端（必须调用）
  - `res.addTrailers(headers)` 添加尾部响应头（极少用）
- http.ClientRequest（客户端请求）
  - `req.write(data)` 发送请求体数据
  - `req.end(data)` 结束请求
  - `req.abort()` 取消请求
  - `req.on('response', res => {})` 收到响应
- http.IncomingMessage（响应结果，客户端用）
  - `res.statusCode` 响应状态码
  - `res.headers` 响应头
  - `res.on('data')` 接收数据
  - `res.on('end')` 接收完毕

```javascript

const port = 3000
const server = http.createServer((req, res) => {})
server.listen(port, () => console.log(`Example app listening on http:localhost:${port}!`))
```

## HTTP 协议

### 网络基础

分解一下这个网站：https://lilichao.com/hello/index.html

- `https://` 协议名 http ftp ...
- `lilichao.com` 域名 domain，整个网络中存在着无数个服务器，每一个我服务器都有它自己的唯一标识，这个标识被称为 ip 地址 192.168.1.17，但是 ip 地址不方便记忆。域名就相当于是 ip 地址的别名
- `/hello/index.html` 网站资源路径

【常见面试题】当在浏览器中输入地址以后发生了什么？https://lilichao.com/hello/index.html

1. DNS 解析，获取网站的 ip 地址
2. 浏览器需要和服务器建立连接（tcp/ip）（三次握手）
3. 向服务器发送请求（http 协议）
4. 服务器处理请求，并返回响应（http 协议）
5. 浏览器将响应的页面渲染
6. 断开和服务器的连接（四次挥手）

【常见面试题】客户端如何和服务器建立（断开）连接

- 三次握手（建立连接）：三次握手是客户端和服务器建立连接的过程
  1. 客户端向服务器发送连接请求（请求消息 SYN）
  2. 服务器收到连接请求，向客户端返回消息（同意消息 ACK，请求消息 SYN）
  3. 客户端向服务器发送同意连接的信息（同意消息 ACK）
- 四次挥手（断开连接）：四次挥手是客户端断开和服务器的连接
  1. 客户端向服务器发送请求，通过之服务器数据发送完毕，请求断开连接（FIN）
  2. 服务器向客户端返回数据，知道了（ACK）
  3. 服务器向客户端返回数据，收完了，可以断开连接（FIN ACK）
  4. 客户端向服务器发数据，可以断开了（ACK）
- 请求和响应实际上就是一段数据，只是这段数据需要遵循一个特殊的格式，这个特殊的格式由 HTTP 协议来规定

### 网络通信

TCP/IP 协议族（了解）

- TCP/IP 协议族中包含了一组协议，这组协议规定了互联网中所有的通信的细节
- 网络通信的过程由四层组成（发数据从上到下，收数据反向从下到上）
  - 应用层：软件的层面，浏览器、服务器都属于应用层
  - 传输层：负责对数据进行拆分，把大数据拆分为一个一个小包
  - 网络层：负责给数据包，添加信息
  - 数据链路层：传输信息
- HTTP 协议就是应用层的协议，用来规定客户端和服务器间通信的报文格式的

报文（message）

- 浏览器和服务器之间通信是基于请求和响应的
  - 浏览器向服务器发送请求（request）
  - 服务器向浏览器返回响应（response）
  - 浏览器向服务器发送请求相当于浏览器给服务器写信；服务器向浏览器返回响应，相当于服务器给浏览器回信；这个信在 HTTP 协议中就被称为报文。而 HTTP 协议就是对这个报文的格式进行规定
- 请求报文（request）：客户端发送给服务器的报文称为请求报文
  - 请求报文的格式如下：请求首行，请求头，空行，请求体
  - 请求首行：请求首行就是请求报文的第一行 `GET /index.html?username=sunwukong HTTP/1.1`
    - 第一部分 `GET` 表示请求的方式，get 表示发送的是 get 请求。现在常用的方式就是 get 和 post 请求，get 请求主要用来向服务器请求资源，post 请求主要用来向服务器发送数据。
    - 第二部分 `/index.html?username=sunwukong` 表示请求资源的路径，? 后边的内容叫做查询字符串，查询字符串是一个名值对结构，一个名字对应一个值，使用=连接，多个名值对之间使用&分割
      - `username=admin&password=123123`，get 请求通过查询字符串将数据发送给服务器，由于查询字符串会在浏览器地址栏中直接显示，所以，它安全性较差，同时，由于 url 地址长度有限制，所以 get 请求无法发送较大的数据。post 请求通过请求体来发送数据，在 chrome 中通过“载荷”可以查看。post 请求通过请求体发送数据，无法在地址栏直接查看，所以安全性较好，请求体的大小没有限制，可以发送任意大小的数据。如果你需要向服务器发送数据，能用 post 尽量使用 post。
    - 第三部分 `HTTP/1.1` 协议的版本
  - 请求头：请求头也是名值对结构，用来告诉服务器我们浏览器的信息
    - 每一个请求头都有它的作用：
    - Accept 浏览器可以接受的文件类型
    - Accept-Encoding 浏览器允许的压缩的编码
    - User-Agent 用户代理，它是一段用来描述浏览器信息的字符串
  - 空行：用来分隔请求头和请求体
  - 请求体：post 请求通过请求体来发送数据
- 响应报文：
  - 响应报文格式如下：响应首行，响应头，空行，响应体
  - 响应首行：`HTTP/1.1 200 OK`
    - 200 响应状态码，ok 对响应状态码的描述
    - 响应状态码的规则：1xx 请求处理中；2xx 表示成功；3xx 表示请求的重定向；4xx 表示客户端错误；5xx 表示服务器的错误
  - 响应头
    - 响应头也是一个一个的名值对结构，用来告诉浏览器响应的信息
    - Content-Type 用来描述响应体的类型`Content-Type: text/html; charset=UTF-8`
    - Content-Length 用来描述响应体大小`Content-Length: 2017`
  - 空行：空行用来分隔响应头和响应体
  - 响应体：响应体就是服务器返回给客户端的内容
  - 网页、css、 js、图片这些资源会作为响应报文中的响应体发送
- 服务器
  - 一个服务器的主要功能：1.可以接收到浏览器发送的请求报文。2.可以向浏览器返回响应报文。

### Express 框架

#### 使用步骤

- 创建 index.js 并编写代码

搭建一个 express 基本框架

- 引入 express：`const express = require("express")`
- 获取服务器的实例（对象）：`const app = express()`
- 启动服务器：`app.listen(3000, () => { console.log("服务器已经启动~") })`
  - `app.listen(端口号, 回调函数)` 用来启动服务器，服务器启动后，我们便可以通过 3000 端口来访问了
  - `协议名://ip地址:端口号/路径`
  - http://localhost:3000 = http://127.0.0.1:3000

```javascript
const express = require('express')
const app = express()
const port = 3000

app.listen(port, () => console.log(`Example app listening on http://localhost:${port}`))
```

#### 路由

- 如果希望服务器可以正常访问，则需要为服务器设置路由，路由可以根据不同的请求方式和请求地址来处理用户的请求
- `app.METHOD('路径，默认根目录', [回调函数])`：METHOD 可以是 get 或 post 等；函数内部第一个参数为请求地址，第二个参数为回调函数
- 路由的回调函数执行时，会接收到三个参数
  - 第一个 request，简写为 req，表示的是用户的请求信息，通过 req 可以获取用户传递数据
  - 第二个 response，简写为 res，表示的服务器发送给客户端的响应信息，可以通过 res 来向客户端返回数据
    - `res.sendStatus()` 向客户端发送响应状态码
    - `res.status()` 用来设置响应状态吗，但是并不发送
    - `res.send()` 设置并发送响应体
    - `res.redirect()` 重定向
- 在路由中，应该做两件事：通过 req 读取用户的请求（request），通过 res 根据用户的请求返回响应（response）

```javascript
const express = require('express')
const app = express()
const port = 3000

// 四个基本路由
app.get('/', (req, res) => res.send('Hello World!'))
app.post('/', (req, res) => res.send('Got a POST request'))
app.put('/user', (req, res) => res.send('Got a PUT request at /user'))
app.delete('/user', (req, res) => res.send('Got a DELETE request at /user'))

app.listen(port, () => console.log(`Example app listening on http://localhost:${port}`))
```

#### 中间件

- 在 express 我们使用 `app.use('路径，默认根目录', [回调函数])` 来定义一个中间件
- 中间件作用和路由很像，用法很像，但是路由不区分请求的方式，只看路径。中间件会在路由之前运行（可以进行路由的权限检查，像保安一样，叫做路由守卫）
- 和路由的区别：会匹配所有请求；路径设置父目录
- `next()` 是回调函数的第三个参数，它是一个函数，调用函数后，可以触发后续的中间件。注意 `next()` 不能在响应处理完毕后调用

```javascript
const express = require('express')
const app = express()
const port = 3000

app.use((req, res, next) => {
  console.log('111', Date.now())
  // res.send("111")
  next() // 放行，我不管了~~
})

app.use((req, res, next) => {
  console.log('222', Date.now())
  // res.send("222")
  next() // 放行，我也不管了~~
})

app.use((req, res, next) => {
  console.log('333', Date.now())
  res.send('<h1>333</h1>')
})

app.get('/', (req, res) => {
  console.log('有人访问我了~')
  res.send('这是我的第一个服务器')
})

app.listen(port, () => console.log(`Example app listening on http://localhost:${port}`))
```

#### 静态资源

- 服务器中的代码，对于外部来说都是不可见的，所以我们写的 html 页面，浏览器无法直接访问。如果希望浏览器可以访问，则需要将页面所在的目录设置静态资源目录
- `express.static([虚拟路径前缀], [路径])`，设置 static 中间件后，浏览器访问时，会自动去 public 目录寻找是否有匹配的静态资源，默认先找 index.html

```javascript
// 正常路径 http://localhost:3000/index.html
app.use(express.static(path.resolve(__dirname, './public')))
// 虚拟路径 http://localhost:3000/static/index.html
app.use('/static', express.static(path.resolve(__dirname, './public')))
```

#### GET 请求传参

查询字符串query（常用于 get 方法）

- 用户通过表单 `/?username=swk&password=123123` 的方式传入的数据，可以通过 `req.query` 来进行获取传入的内容来进行用户验证
- `req.query` 表示查询字符串中的请求参数，`req.query.username`、`req.query.password`
- 缺点：内容会在地址栏上显示出来，却会存在浏览记录里面，造成内容泄露

一个简易登录页面

```javascript
// index.js
const path = require('node:path')
const express = require('express')
const app = express()

// 设置static中间件后，浏览器访问时，会自动去public目录寻找是否有匹配的静态资源，默认先找index.html
app.use(express.static(path.resolve(__dirname, './public')))

// 配置路由，找不到public中的index.html时会显示这个
app.get('/', (req, res) => {
  res.send('怎么办呢？')
})

// 配置路由，处理登录请求
app.get('/login', (req, res) => {
  // 获取到用户输入的用户名和密码
  // req.query 表示查询字符串中的请求参数
  // console.log(req.query.username)
  // console.log(req.query.password)
  console.log('请求已经收到~~')

  // 验证用户输入的用户名和密码是否正确
  if (req.query.username === 'admin' && req.query.password === '123') {
    res.send('<h1>登录成功！</h1>')
  } else {
    res.send('<h1>用户名或密码错误！</h1>')
  }
})

// 启动服务器
app.listen(3000, () => console.log('服务器已经启动~'))
```

#### POST 请求传参

解析请求体（常用于 post 方法）

- 用户通过表单 post 的方式传入的数据，可以通过 `req.body` 来进行获取
- `req.body` 表示 post 的请求参数，`req.body.username`、`req.body.password`
- 默认情况下 express 不会自动解析请求体，需要通过中间件来为其增加功能
- `express.urlencoded()` 来解析请求体，内置中间件
- `express.body-parser` 来解析请求体，第三方中间件

```javascript
// 引入解析请求体的中间件
app.use(express.urlencoded({ extended: true }))

app.post('/login', (req, res) => {
  console.log('POST请求已经收到~~')
  console.log(req.body) // 没有中间件解析请求体，则会返回undefined
})
```

```javascript
// 引入解析请求体的中间件
const bodyParser = require('body-parser')
app.use(bodyParser.urlencoded({ extended: true }))

app.post('/login', (req, res) => {
  console.log('POST请求已经收到~~')
  console.log(req.body) // 没有中间件解析请求体，则会返回undefined
})
```

#### 模板引擎

ejs 是一个模板引擎，它可以将数据和 HTML 模板结合在一起，生成最终的 HTML 页面

- 安装：`npm i ejs -D`、`yarn add ejs -D`
- 使用：`app.set('view engine', 'ejs')` 设置模板引擎为 ejs
- 模板文件：默认在 views 目录下，文件后缀为 .ejs，当作 html 文件书写即可
- 配置路径：`app.set('views', path.resolve(__dirname, './views'))` 设置模板文件存放的目录
- 渲染模板：`res.render('模板文件名', { 数据 })` 渲染模板并返回给客户端

```javascript
// index.js
const path = require('node:path')
const express = require('express')
const app = express()
const port = 3000

// 设置模板引擎为ejs
app.set('view engine', 'ejs')

// 设置模板文件存放的目录
app.set('views', path.resolve(__dirname, './views'))

// 配置路由，处理登录请求
app.get('/login', (req, res) => {
  res.render('login', { title: '登录页面' })
})

// 启动服务器
app.listen(port, () => console.log(`Example app listening on http://localhost:${port}`))
```

#### Router

Router 是 Express 中非常重要的一个概念，它可以帮助我们更好地组织代码，提高代码的可维护性。

```javascript
// 不使用 Router（所有路由写在一起，混乱）
const express = require('express')
const app = express()

// 所有路由直接挂载在 app 上，项目大了会乱成一团
app.get('/user/list', (req, res) => res.send('用户列表'))

app.get('/order/list', (req, res) => res.send('订单列表'))
```

```javascript
// 使用 Router（拆分管理，规范清晰）
const express = require('express')
const app = express()

// 1. 创建用户路由模块
const userRouter = express.Router()
// 给 userRouter 添加具体路由（Route）
userRouter.get('/list', (req, res) => res.send('用户列表'))
userRouter.post('/add', (req, res) => res.send('添加用户'))

// 2. 创建订单路由模块
const orderRouter = express.Router()
orderRouter.get('/list', (req, res) => res.send('订单列表'))

// 3. 把路由模块注册到主应用
app.use('/user', userRouter) // 访问前缀：/user
app.use('/order', orderRouter) // 访问前缀：/order
```

### nodemon

目前，服务器代码修改后必须要重启，我们希望有一种方式，可以自动监视代码的修改，代码修改以后可以自动重启服务器。要实现这个功能，我们需要安装一个模块 nodemon

- 全局安装
  - 安装：`npm i nodemon -g`、`yarn global add nodemon`
    - 注意：推荐使用 npm 全局安装。yarn 进行全局安装时，默认 yarn 的目录并不在环境变量中，需要手动将路径添加到环境变量中：`yarn global bin`，把生成的路径放到环境变量里面即可，`nodemon -v` 显示版本号，表示安装成功
  - 启动：`nodemon // 运行index.js`、`nodemon xxx // 运行指定的js`
- 在项目中安装
  - 安装：`npm i nodemon -D`、`yarn add nodemon -D`
  - 启动：`npx nodemon`

此时，当我们代码修改后，服务器会自动重启，无需手动重启。

### Cookie

cookie 是存储在用户浏览器上的数据，用于记录用户在网站上的行为，以便下次访问时提供更好的用户体验。cookie 是由服务器发送到客户端的一小段数据，客户端会将这些数据存储在本地，并在下次请求时将它们发送回服务器。

- `req.cookies` 获取 cookie
- `res.cookie('key', 'value', { options })` 设置 cookie
  - key：cookie 的名称
  - value：cookie 的值
  - options：cookie 的选项，例如过期时间、路径、域、安全等

cookie 的中间件

- `cookie-parser`：解析 cookie 的中间件，将 cookie 数据解析为对象，挂载到 `req.cookies` 上
- 下载：`npm i cookie-parser`、`yarn add cookie-parser`
- 安装：`const cookieParser = require('cookie-parser')`
- 使用：`app.use(cookieParser())`

```javascript
const express = require('express')
const cookieParser = require('cookie-parser')
const app = express()

app.use(cookieParser())
```

### Session

session 是另一种记录用户信息的方式，它将用户信息存储在服务器端，而不是客户端。session 的数据是存储在服务器上的，因此更加安全，但是也会增加服务器的负担。

session 是服务器中的一个对象，这个对象用来存储用户的数据，每一个 session 对象都有一个唯一的 id，id 会通过 cookie 的形式发送给客户端，客户端每次访问时只需将存储有 id 的 cookie 发回即可获取它在服务器中存储的数据。

session 的中间件

- `express-session`：处理 session 的中间件，将 session 数据存储在内存中，也可以存储在数据库中
- 下载：`npm i express-session`、`yarn add express-session`
- 安装：`const session = require('express-session')`
- 使用：`app.use(session({ secret: 'my_secret', resave: false, saveUninitialized: true }))`
  - secret：用于加密 session 数据的密钥，可以是任意字符串
  - resave：是否每次请求都重新保存 session，默认为 true
  - saveUninitialized：是否保存未初始化的 session，默认为 true

```javascript
const express = require('express')
const session = require('express-session')
const app = express()

app.use(session({ secret: 'my_secret', resave: false, saveUninitialized: true }))
```

### Csrf 攻击

csrf（Cross-Site Request Forgery，跨站请求伪造）是一种攻击方式，攻击者诱导受害者访问一个包含恶意请求的页面，利用受害者的登录状态发起攻击。

- 防护措施：
  - 检查referer：检查请求的来源，如果不是来自可信的域名，则拒绝请求。
  - 使用 token 验证：在请求中添加一个 token，服务器验证 token 是否正确，如果正确则允许请求，否则拒绝请求。token 可以存储在 cookie 中，也可以存储在请求参数中。

## Restful API

### Rest 简介

- Rest：REpresentational State Transfer，表现层状态转化，是一种网络应用程序的设计风格和开发方式。
- API：Application Programming Interface，应用程序编程接口，是一组用于访问应用程序或服务的规则和协议。API 可以用于实现应用程序之间的通信，也可以用于实现应用程序与用户之间的交互。
- RESTful API 是一种设计风格，用于构建网络应用程序。它基于 HTTP 协议，使用不同的 HTTP 方法来表示不同的操作，例如 GET、POST、PUT、DELETE 等，使用 URL 来表示资源，使用 JSON 格式来表示数据。
  - GET：获取资源
  - POST：创建资源
  - PUT：更新资源
  - PATCH：部分更新资源
  - DELETE：删除资源
  - OPTIONS：获取资源支持的请求方法
- **以前编写服务器需要获取数据，渲染页面，然后返回新页面；现在只需要使用 Restful API 提供 JSON 数据，不需要渲染页面，大大简化了后端开发流程。**
