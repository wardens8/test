# 正则表达式 `RegExp`

正则表达式（Regular Expression，简称 RegExp）是一种工具，和其他工具一样，它是人们为了解决某一类专门的问题而发明的。

简单地说，正则表达式是一些用来匹配和处理文本的字符串。

## 一般用法

- 纯文本：匹配纯文本，直接输入即可，和正则关系不大
- `.`：匹配单个任意字符
- `[]`：匹配某一个字符
  - `[Aa]` 用来匹配原始 `A` 或者 `a`
- `[ - ]`：匹配任意字符集合
  - `[0-9]` 匹配从 0 到 9 的所有数字，表示 `[0123456789]`
  - `[A-Z]` 匹配从 A 到 Z 的所有大写字母，表示 `[ABCDEFGHIJKLMNOPQRSTUVWXYZ]`
  - `[a-z]` 匹配从 a 到 z 的所有小写字母，表示 `[abcdefghijklmnopqrstuvwxyz]`
  - `[A-z]` 匹配从 ASCII 字符 A 到 ASCII 字符 z 的所有字母。这个模式一般不常用，因为它还包含着 `[` 和 `^` 等在 ASCII 字符表里排列在 Z 和 a 之间的字符
  - `[A-Za-z0-9]` 匹配从 A 到 Z 的所有大写字母，从 a 到 z 的所有小写字母，从 0 到 9 的所有数字，表示 `[ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789]`
- `^`：取非字符
  - `[0-9]` 匹配 0-9 的所有数字，`[^0-9]` 匹配除 0-9 的所有数字以外的字符
- `\`：转义字符：转义元字符的原义
  - `\.` 表示`.`的意思，`[\b]` 回退（并删除）一个字符 Backspace 键，`\f` 换页符，`\n` 换行符，`\r` 回车符，`\t` 制表符 Tab 键，`\v` 垂直制表符
- 特定字符
  - 匹配数字与非数字 `digit`
    - `\d` 匹配任何一个数字字符（等价于`[0-9]`）
    - `\D` 匹配任何一个非数字字符（等价于`[^0-9]`）
  - 匹配字母和数字与非字母和数字 `word`
    - `\w` 匹配任何一个字母数字字符（大小写均可）或下划线字符（等价于`[a-zA-Z0-9_]`）
    - `\W` 匹配任何一个非字母数字字符（大小写均可）或下划线字符（等价于`[^a-zA-Z0-9_]`）
  - 匹配空白字符与非空白字符 `space`
    - `\s` 匹配任何一个空白字符（等价于`[\f\n\r\t\v]`）
    - `\S` 匹配任何一个非空白字符（等价于`[^\f\n\r\t\v]`）
  - 使用 `POSIX` 字符类
    - `[:alnum:]` 匹配任何一个字母或数字（等价于`[a-zA-Z0-9]`）
    - `[:alpha:]` 匹配任何一个字母（等价于`[a-zA-Z]`）
    - `[:blank:]` 匹配空格或制表符（等价于`[\t ]`）
    - `[:cntrl:]` 匹配 ASCII 控制字符（ASCII 0 到 31，再加上 ASCI 127）
    - `[:digit:]` 匹配任何一个数字（等价于`[0-9]`）
    - `[:graph:]` 和`[:print:]`一样，但不包括空格
    - `[:lower:]` 匹配任何一个小写字母（等价于`[a-z]`）
    - `[:print:]` 匹配任何一个可打印字符
    - `[:punct:]` 匹配既不属于`[:alnum:]`也不属于`[:cntrl:]`的任何一个字符
    - `[:space:]` 匹配任何一个空白字符，包括空格（等价于`[\f\n\r\t\v ]`）
    - `[:upper:]` 匹配任何一个大写字母（等价于`[A-Z]`）
    - `[:xdigit:]` 匹配任何一个十六进制数字（等价于`[a-fA-F0-9]`）
    - 注意：使用的时候是这样的 `[[:***:]]`

## 重复匹配

- `+`：匹配一个或多个字符
  - `[0-9]` 匹配任意单个数字，`[0-9]+` 将匹配一个或多个连续的数字
- `*`：匹配零个或多个字符
  - `[0-9]` 匹配任意单个数字，`[0-9]*` 将匹配零个或多个连续的数字
- `?`：匹配零个或一个字符
  - `[0-9]` 匹配任意单个数字，`[0-9]?` 将匹配零个或一个连续的数字，即可有可无
  - `http://[\w./]+` 匹配 `http` 开头的网址，`https?://[\w./]+` 匹配 `http` 或 `https` 开头的网址
    - 注：`s?` 就是指 `?` 前面的 `s` 可能会出现，也可能不出现，换句话说，`https?://` 既可以匹配 `http://`，也可以匹配 `https://`，但也就仅此而已
- `{}`：为重复匹配次数设定一个精确的值
  - `[0-9]` 匹配任意单个数字，`[0-9]{6}` 将匹配 6 个连续的数字，`[0-9]{n}` 将匹配 n 个连续的数字
- `{ , }`：为重复匹配次数设定一个区间
  - `[0-9]` 匹配任意单个数字，`[0-9]{3,6}` 将匹配 3 到 6 个连续的数字，`[0-9]{3,}` 将匹配至少 3 个连续的数字，`[0-9]{,6}` 将匹配至多 6 个连续的数字，`[0-9]{m,n}` 将匹配 m 至 n 个连续的数字
- 防止过度匹配
  - `*`和`+`都是所谓的“贪婪型”元字符，它们在进行匹配时的行为模式是多多益善而不是适可而止的。它们会尽可能地从一段文本的开头一直匹配到这段文本的末尾，而不是从这段文本的开头匹配到碰到第一个匹配时为止。在不需要这种“贪婪行为”的时候该怎么办？答案是使用这些元字符的“懒惰型”版本（“懒惰”在这里的含义是匹配尽可能少的字符——与“贪婪型”元字符的行为模式刚好相反）。懒惰型元字符的写法很简单，只要给贪婪型元字符加上一个`?`后缀即可
    - `* --> *?`
    - `+ --> +?`
    - `{n,} --> {n,}?`

## 位置匹配

- `\b`：匹配单词边界
  - `cat` 表示 `cat`，`\bcat\b` 表示 `cat`
    - 注：`\b` 表示 `\w` 和 `\W` 之间的位置。`\B` 表示 `\w` 和 `\w` 之间或 `\W` 和 `\W` 之间的位置
- `^` /`$`：匹配字符串的边界，`^`用来定义字符串的开头，`$`用来定义字符串的结尾
  - `^\s*` 匹配一个字符串的开头位置和随后的零个或多个空白字符
  - `\s*$` 匹配一个字符串的结尾位置和之前的零个或多个空白字符
  - 注意：`^`是几个有着多种用途的元字符之一。只有当它出现在一个字符集合里（被放在`[`和`]`之间）并紧跟在左方括号`[`的后面时，它才能发挥“求非”作用。如果是在一个字符集合的外面并位于一个模式的开头，`^`将匹配字符串的开头。
- `(?m)`：分行匹配模式
  - 有许多正则表达式都支持使用一些特殊的元字符去改变另外一些元字符行为的做法，用来启用分行匹配模式（multiline mode）的`(?m)`记号就是一个能够改变其他元字符行为的元字符序列。
  - 警告：有许多正则表达式实现不支持`(?m)`。

## 分组捕获

用 `()` 把一段正则括起来，视为一个整体，方便后续引用或提取内容。这部分内容就叫做子表达式。子表达式是一个更大的表达式的一部分；把一个表达式划分为一系列子表达式的目的是为了把那些子表达式当作一个独立元素来使用

- `()`：子表达式必须用 `(` 和 `)` 括起来。通俗的讲，就是用括号括起来表示一个整体使用
- `|`：或操作符，`19|20` 将匹配数字序列 19 或 20。
- 子表达式允许嵌套。事实上，子表达式允许多重嵌套，这种嵌套的层次在理论上没有限制，但在实际工作中还是应该遵循适可而止的原则。

子表达式的用途：定义回溯引用（backreference）

- `\1`：用来表示前一个出现的子表达式
  - `[ ]+(\w+)[ ]+\1`，此处的 `\1` 用来匹配前面的 `(\w+)` 部分
  - 注意：在 JavaScript 中，使用的是 `$1`

## 前后查找

- `(?=)`：正向前查找
  - `.*(?=:)` 是匹配`:`之前的内容
- `(?!)`：负向前查找
- `(?<=)`：正向后查找
  - `(?<=\$)[0-9.]+` 是匹配`$`之后的内容
- `(?<!)`：负向后查找

## JavaScript 中的正则表达式

创建正则表达式

- 字面量：`/abc/`、`/abc/gim`
- 对象：`new RegExp('abc')`、`new RegExp('abc', 'gim')`
- 修饰符：
  - `g`：全局匹配
  - `i`：忽略大小写
  - `m`：多行匹配

正则表达式的使用

- `str.match(/ /)` 匹配正则表达式，返回匹配结果数组

```javascript
const str = '123 abc 456'
console.log(str.match(/\d+/g)) // ["123", "456"]
```

# 正则表达式——word通配符

## 一般用法

- 纯文本：匹配纯文本，直接输入即可，和正则关系不大
- `?`：匹配单个任意字符
- `*`：匹配任意 0 个或多个字符
- `[]`：匹配某一个字符
  - `[Aa]` 用来匹配原始`A`或者`a`
- `[ - ]`：匹配任意字符集合
  - `[0-9]` 匹配从 0 到 9 的所有数字，表示 `[0123456789]`
  - `[A-Z]` 匹配从 A 到 Z 的所有大写字母，表示 `[ABCDEFGHIJKLMNOPQRSTUVWXYZ]`
  - `[a-z]` 匹配从 a 到 z 的所有小写字母，表示 `[abcdefghijklmnopqrstuvwxyz]`
  - `[A-z]` 匹配从 ASCII 字符 A 到 ASCII 字符 z 的所有字母。这个模式一般不常用，因为它还包含着 `[` 和 `^` 等在 ASCII 字符表里排列在 Z 和 a 之间的字符
  - `[A-Za-z0-9]` 匹配从 A 到 Z 的所有大写字母，从 a 到 z 的所有小写字母，从 0 到 9 的所有数字，表示 `[ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789]`
- `!`：取非字符
  - `[0-9]` 匹配 0-9 的所有数字，`[!0-9]` 匹配除 0-9 的所有数字以外的字符
- `\`：转义字符：转义元字符的原义
- 特殊字符
  - `^13`/`^p`：换行符
  - `^t`：制表符
  - 其他特殊字符均可以在 `使用通配符-特殊格式` 中进行查看使用

## 重复匹配

- `@`：匹配一个或多个字符
  - `[0-9]` 匹配任意单个数字，`[0-9]@` 将匹配一个或多个连续的数字

- `{ , }`：为重复匹配次数设定一个区间
  - `[0-9]` 匹配任意单个数字，`[0-9]{3,6}` 将匹配 3 到 6 个连续的数字，`[0-9]{3,}` 将匹配至少 3 个连续的数字，`[0-9]{,6}` 将匹配至多 6 个连续的数字，`[0-9]{m,n}` 将匹配 m 至 n 个连续的数字
- word 中没有非贪婪匹配，默认贪婪

## 位置匹配

- `<>`：匹配单词边界
  - `cat`表示`cat`，`<cat>`表示` cat`
    - 注：`\b` 表示 `\w` 和 `\W` 之间的位置。`\B` 表示 `\w` 和 `\w` 之间或 `\W` 和 `\W` 之间的位置
- `^`/`$`：匹配字符串的边界，`^` 用来定义字符串的开头，`$` 用来定义字符串的结尾

  - `\s*$` 匹配一个字符串的结尾位置和之前的零个或多个空白字符
  - 注意：`^`是几个有着多种用途的元字符之一。只有当它出现在一个字符集合里（被放在`[`和`]`之间）并紧跟在左方括号`[`的后面时，它才能发挥“求非”作用。如果是在一个字符集合的外面并位于一个模式的开头，`^`将匹配字符串的开头。

- `()`：子表达式必须用 `(` 和 `)` 括起来。通俗的讲，就是用括号括起来表示一个整体使用
