## 基本用法

> 注：用法右键都可以实现。快捷键在菜单栏都有。

标题

```markdown
段落内容直接写

# 一级标题（最大，适配文档大标题）

## 二级标题（常用，适配章节标题）

### 三级标题（常用，适配小节标题）

#### 四级标题

##### 五级标题

###### 六级标题（最小）

<!-- 文件目录 -->

[toc] / [TOC]
```

基础文本格式

```markdown
_斜体内容_

**加粗内容**

**_粗斜体内容_**

~~删除线内容~~

==高亮内容==（部分编辑器支持，如 VS Code）

> 引用内容
```

列表

```markdown
- 无序列表
  - 列表 1
  - 列表 2
  - 列表 3

1. 有序列表
   1. 列表 1
   2. 列表 2
   3. 列表 3

- [ ] 任务列表
- [x] 任务列表
```

链接与图片

```markdown
[链接显示文本](链接地址)
[GitHub 官网](https://github.com)

![图片加载失败提示语](图片地址)
![笔记封面](./images/cover.png)（相对路径）
![logo](https://static.char123.com/images/favicon.ico)
```

分割线

```markdown
---
```

表格

```markdown
| 表头 1 | 表头 2 | 表头 3 |
| ------ | ------ | ------ |
| 内容 1 | 内容 2 | 内容 3 |
| 内容 4 | 内容 5 | 内容 6 |

| 表头 1 |  表头 2  | 表头 3 |
| :----- | :------: | -----: |
| 左对齐 | 居中对齐 | 右对齐 |
| 内容 1 |  内容 2  | 内容 3 |
| 内容 4 |  内容 5  | 内容 6 |
```

脚注

```markdown
这是一段需要注释的文本[^1]
[^1]: 这里是脚注的内容（文档末尾显示）
```

代码块

- 单行：前后各一个单引号 `123`
- 多行：前后各三个单引号

```markdown
我是代码块内容
```

公式块

- 单行：前后各一个美元符号 $123$
- 多行：前后各两个美元符号

$$
我是公式块内容
$$

## 浏览器显示

在浏览器中下载 `Markdown Reader` / `Markdown Viewer` 插件添加至浏览器，即可在浏览器中查看 Markdown 文件

- `Markdown Reader`（自带目录）✅️
  - 插件网址：https://markdownreader.github.io/
- `Markdown Viewer`（不带目录）
  - 插件网址：https://github.com/simov/markdown-viewer
- 安装
  - 除谷歌浏览器外，其他浏览器可直接在插件网址下载安装，或者在各自插件商店搜索下载安装。安装完成后，点击插件详细信息，勾选允许访问文件 URL 后，即可在浏览器中查看 Markdown 文件，浏览器会自动转换成 HTML 文件
  - 谷歌浏览器可在下面网址下载插件。解压压缩包，打开扩展程序，打开开发者工具，将插件直接拖入即可安装。安装完成后，点击插件详情，勾选允许访问文件网址后，即可在浏览器中查看 Markdown 文件，浏览器会自动转换成 HTML 文件
    - 直接下载：<a href="./Markdown Reader 3.6.4.zip" download>Markdown Reader 3.6.4.zip</a>
    - Crx 搜搜：https://www.crxsoso.com/
    - 插件小屋：https://www.chajianxw.com/
- 使用
  - 安装完成后可点击右上角设置进行修改，建议打开深色模式，开启代码自动换行
  - 也可以自定义桌面样式

```css
h1 {
  text-align: center;
}
```

## 设置修改

- `Format On Save` 勾选上可以在保存时自动格式化代码
- `Default Formatter` 选择默认格式化工具 prettier
- `Word Wrap` 选择 on 在视区宽度换行
- `Tab Size` 写 2 表示缩进全部为2个字符
- `Color Theme` 选择主题
- `Icon Theme` 选择icon主题
- `Explorer:confirm drag and drop` 选择是否确认拖拽
- `Explorer:confirm delete` 选择是否确认删除
- `prettier:Semi` 选择 false，去掉末尾分号
- `prettier:Single Quote` 选择 true，用单引号代替双引号

点击右上角可以进入 `setting.json`，不想设置也可以直接将下列代码放入json文件内

```json
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.wordWrap": "on",
  "editor.tabSize": 2,
  "workbench.colorTheme": "Monokai",
  "workbench.iconTheme": "vscode-icons",
  "explorer.confirmDragAndDrop": false,
  "explorer.confirmDelete": false,
  "[html]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "[css]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "[markdown]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "[javascript]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "[less]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "[jsonc]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "prettier.semi": false,
  "prettier.singleQuote": true,
  "Codegeex.License": "",
  "Codegeex.Privacy": false
}
```

## 代码片段修改

左下角设置 --> 代码片段

在不同类型代码后面设置对应的 json 文件，例如是 HTML 类型的代码就在 `html.json` 内设置，是 Javascript 类型的代码就在 `javascript.json` 内设置

代码片段介绍

```
Place your snippets for javascript here. Each snippet is defined under a snippet name and has a prefix, body and description. The prefix is what is used to trigger the snippet and the body will be expanded and inserted. Possible variables are: $1, $2 for tab stops, $0 for the final cursor position, and ${1:label}, ${2:another} for placeholders. Placeholders with the same ids are connected.
将您的javascript代码片段放在这里。每个代码段都在一个代码段名称下定义，并且有一个前缀、主体和描述。前缀是用来触发代码片段的，主体将被扩展和插入。可能的变量有:$1，$2用于制表位，$0用于最终光标位置，${1:label}、${2:another}用于占位符。带有连接了相同的id。

Example:
"Print to console": {
  "prefix": "log",
  "body": [
    "console.log('$1');",
    "$2"
  ],
  "description": "Log output to console",
}
```

书写格式：可以用`$1`、`$2`等进行占位，可以快速定位进行书写，tab 可以快速跳至下一个位置

```json
{
  // 设置的代码片段的标识
  "html": {
    // 前缀，书写时需要输入的片段缩写，用来触发代码片段的
    "prefix": "html",
    // 主体（必备），被缩写的代码片段，要求每一行都需要被""包裹，具体的""需要\转义
    "body": [
      "<!DOCTYPE html>",
      "<html lang=\"en\">",
      "  <head>$1</head>",
      "  <body>$2</body>",
      "</html>"
    ],
    // 描述，代码片段的具体描述
    "description": "一段简短的html片段"
  }
}
```
