## 闭包与高阶函数

### 闭包

- 闭包（Closure）是指有权访问另一个函数作用域中的变量的函数。闭包是由函数以及创建该函数的环境组合而成。闭包让你可以在一个内层函数中访问到其外层函数的作用域
- 闭包的本质：闭包是 JavaScript 中函数嵌套产生的特殊现象：内层函数可以访问外层函数作用域中的变量，即使外层函数已经执行完毕并退出
- 核心原理：JavaScript 有「词法作用域」（定义时决定作用域，而非执行时），当内层函数被外部引用时，外层函数的执行上下文不会被垃圾回收机制（GC）销毁，从而保留了外层作用域的变量
- 闭包的作用：闭包可以用来封装私有变量，避免全局变量的污染，实现模块化开发
- 闭包的缺点：闭包会常驻内存，会增大内存使用量，过多的闭包可能会导致内存泄露
- 闭包的创建：当一个函数被创建并传递或赋值给另一个变量时，闭包就会被创建
  - 函数的嵌套
  - 内层函数引用了外层函数的局部变量
  - 内部函数要作为返回值返回
- 闭包的生命周期：
  1.  闭包在外部函数调用时产生，外部函数每次调用都会产生一个全新的闭包
  2.  在内部函数丢失时销毁（内部函数被垃圾回收了，闭包才会消失）

```javascript
function outer() {
  // 外层函数的局部变量
  const name = '闭包示例'
  // 内层函数，访问外层变量
  function inner() {
    console.log(name) // 即使outer执行完，inner仍能访问name
  }
  // 返回内层函数，形成闭包
  return inner
}

// 执行outer，得到inner函数
const fn = outer()
// 执行inner，依然能访问outer的name变量
fn() // 输出：闭包示例
```

闭包的应用场景：闭包是前端开发中高频使用的特性，核心价值是私有化变量、保留状态

1. 私有化变量 / 方法（模块化）：避免全局变量污染，模拟 “私有属性”

```javascript
function createCounter() {
  // 私有变量，外部无法直接访问
  let count = 0

  return {
    increment: () => count++,
    decrement: () => count--,
    getCount: () => count,
  }
}

const counter = createCounter()
counter.increment()
console.log(counter.getCount()) // 1
console.log(counter.count) // undefined（无法直接访问）
```

2. 延迟执行 / 保留状态：比如定时器、事件回调中保留上下文

```javascript
function setMsg(msg) {
  // 1秒后执行，仍能访问msg
  setTimeout(() => {
    console.log(msg)
  }, 1000)
}
setMsg('闭包保留状态') // 1秒后输出：闭包保留状态
```

3. 函数柯里化 / 防抖节流（后续详细讲）：闭包是柯里化、防抖节流的核心实现基础

闭包的内存泄漏问题

- 为什么会泄漏：正常情况下，函数执行完毕后，其执行上下文会被 GC 回收；但闭包会让外层函数的作用域被内层函数引用，导致外层作用域无法被回收，如果闭包被长期引用（比如挂载到 window），就会造成内存泄漏
- 常见场景 & 解决方案

```javascript
// 错误示例：闭包被全局变量引用，外层作用域永远无法回收
let globalFn
function leakFn() {
  const bigData = new Array(1000000) // 大体积数据
  globalFn = () => console.log(bigData)
}
leakFn()

// 正确方案：使用完后手动解除引用，让GC回收
globalFn = null // 解除引用，bigData可被回收
```

> 关键注意点：闭包本身不是内存泄漏的原因，不合理的长期引用才是；在浏览器中，DOM 元素与闭包相互引用也容易泄漏（比如闭包引用了 DOM，DOM 又引用了闭包），解决方案是 DOM 销毁时解除闭包引用。

### 递归

- 递归（Recursion）：函数直接或间接调用自身，是编程中的一种重要思想。递归分为两个阶段：递推（不断调用自身，直到满足边界条件）和回归（逐层返回，计算结果）
- 递归的三个要素：1. 递归的终止条件；2. 递归的返回值；3. 递归的单次执行逻辑
- 递归的优缺点：
  - 优点：代码简洁、可读性强；
  - 缺点：递归调用层级过多，容易导致栈溢出（StackOverflowError）

```javascript
// 递归求阶乘
function factorial(n) {
  if (n === 1) return 1
  return n * factorial(n - 1)
}
console.log(factorial(5)) // 120

// 递归求斐波那契数列（生兔子问题）
function fibonacci(n) {
  if (n === 1 || n === 2) return 1
  return fibonacci(n - 1) + fibonacci(n - 2)
}
console.log(fibonacci(10)) // 55
```

### 高阶函数

- 高阶函数（Higher-Order Function）：满足以下任一条件的函数就是高阶函数：1.接受一个或多个函数作为参数；2.返回一个新函数。简单说：操作函数的函数就是高阶函数。
- 常用场景：
  1.  回调函数（最基础场景）：将函数作为参数传入，在合适的时机执行
  2.  封装工具函数（复用逻辑）：将通用逻辑封装成高阶函数，返回新函数
- 原生高阶函数示例：JavaScript 内置了大量高阶函数，比如：
  - 数组方法：map、filter、reduce、sort；
  - 定时器：setTimeout、setInterval；
  - Promise：then、catch、finally。

```javascript
// 回调函数
function add(a, b, callback) {
  console.log('a + b =', a + b)
  callback()
}
add(1, 2, function () {
  console.log('我是回调函数，实现了异步操作')
})
```

### 函数柯里化

函数柯里化（Currying）

- 定义：将接收多个参数的函数转化为一系列接收单个参数的函数的过程。比如：`fn(a, b, c) → fn(a)(b)(c)`
- 手写思路 & 实现
  - 核心思路：用闭包保留已传入的参数；判断参数总数是否达到原函数的参数个数，达到则执行原函数，否则返回新函数继续接收参数。

```javascript
// 通用柯里化函数
function curry(fn) {
  // 原函数的参数个数
  const fnArgLength = fn.length

  // 递归函数，保留已传入参数
  function curried(...args) {
    // 情况1：参数足够，执行原函数
    if (args.length >= fnArgLength) {
      return fn.apply(this, args)
    }

    // 情况2：参数不足，返回新函数继续接收参数
    return function (...newArgs) {
      // 合并已传入的参数和新参数，递归调用
      return curried.apply(this, [...args, ...newArgs])
    }
  }

  return curried
}

// 测试
function sum(a, b, c) {
  return a + b + c
}

const curriedSum = curry(sum)
console.log(curriedSum(1)(2)(3)) // 6
console.log(curriedSum(1, 2)(3)) // 6
console.log(curriedSum(1)(2, 3)) // 6
```

- 应用场景
  - 参数复用：固定部分参数，生成新函数；
  - 延迟执行：分步接收参数，最后执行；
  - 适配函数：将多参数函数适配为单参数函数（比如适配高阶函数）。

### 防抖与节流

- 防抖（Debounce）
  - 核心逻辑：触发事件后，延迟 n 秒执行回调；如果在 n 秒内再次触发事件，则重置延迟时间。通俗说：“频繁触发时，只执行最后一次”。
  - 手写思路 & 实现

```javascript
/**
 * 防抖函数
 * @param {Function} fn - 要防抖的函数
 * @param {number} delay - 延迟时间（毫秒）
 * @param {boolean} immediate - 是否立即执行（首次触发时执行，后续重置）
 * @returns {Function} 防抖后的函数
 */
function debounce(fn, delay = 500, immediate = false) {
  // 用闭包保存定时器ID
  let timer = null

  return function (...args) {
    // 每次触发，先清除之前的定时器
    if (timer) clearTimeout(timer)

    // 立即执行的情况
    if (immediate && !timer) {
      fn.apply(this, args)
    }

    // 重置定时器，延迟执行
    timer = setTimeout(() => {
      // 非立即执行的情况，延迟后执行
      if (!immediate) {
        fn.apply(this, args)
      }
      // 执行完清空定时器
      timer = null
    }, delay)
  }
}

// 测试：监听窗口resize
window.addEventListener(
  'resize',
  debounce(() => {
    console.log('窗口大小变化（防抖）')
  }, 500),
)
```

- 节流（Throttle）
  - 核心逻辑：触发事件后，每隔 n 秒只能执行一次回调，无论期间触发多少次。通俗说：“频繁触发时，固定频率执行”。
  - 手写思路 & 实现（时间戳版）

```javascript
/**
 * 节流函数（时间戳版）
 * @param {Function} fn - 要节流的函数
 * @param {number} interval - 间隔时间（毫秒）
 * @returns {Function} 节流后的函数
 */
function throttle(fn, interval = 500) {
  // 用闭包保存上一次执行的时间
  let lastTime = 0

  return function (...args) {
    // 当前时间
    const now = Date.now()

    // 如果当前时间 - 上一次执行时间 >= 间隔时间，执行函数
    if (now - lastTime >= interval) {
      fn.apply(this, args)
      // 更新上一次执行时间
      lastTime = now
    }
  }
}

// 测试：监听鼠标移动
document.addEventListener(
  'mousemove',
  throttle(() => {
    console.log('鼠标移动（节流）')
  }, 500),
)
```

- 防抖 vs 节流

|   特性   |            防抖            |              节流               |
| :------: | :------------------------: | :-----------------------------: |
| 核心逻辑 |     最后一次触发后执行     |          固定频率执行           |
| 适用场景 | 搜索框输入、按钮防重复点击 | 滚动加载、鼠标移动、窗口 resize |
| 执行时机 |         延迟后执行         |        间隔时间到就执行         |

总结

- 闭包：核心是 “内层函数引用外层作用域变量，保留状态”，应用于私有化变量、延迟执行，需注意解除长期引用避免内存泄漏；
- 高阶函数：定义是 “操作函数的函数”（参数 / 返回值为函数），常用场景是回调函数、封装工具函数；
- 柯里化 / 防抖节流：柯里化核心是 “分步接收参数 + 闭包保留状态”；防抖是 “最后一次触发执行”，节流是 “固定频率执行”，二者均依赖闭包实现。
- 这些知识点是前端面试和实际开发的高频考点，建议你手动敲一遍代码，结合实际场景（比如搜索框防抖、滚动节流）加深理解。

## 面向对象 OOP

面向对象程序设计（Object Oriented Programming, OOP）：一种编程思想，将功能、数据封装成对象，方便调用。内部的具体内容怎么实现，外部不用关心，只需要知道怎么调用即可，保护了内部数据的安全。

### 类和对象

- 类（Class）：抽象的模板，通过 `class` 关键字创建，定义了对象的属性和方法
- 对象（Object）：类的实例，通过 `new` 关键字创建，是类的具体表现
- ES6 语法糖：本质依然是函数，`class` 语法糖只是让对象创建、继承更加简洁易读

```javascript
// ES5 构造函数（旧类）
var Person = (function () {
  function Person(name, age) {
    this.name = name
    this.age = age
  }
  Person.prototype.sayHi = function () {
    console.log('我是' + this.name)
  }
  Person.staticProperty = 'xxx'
  Person.staticMethod = function () {}
  return Person
})()

// ES6 class 类
class Person {
  constructor(name, age) {
    this.name = name
    this.age = age
  }
  sayHi() {
    console.log('我是' + this.name)
  }
}

// 实例化调用，两种定义方式调用方法一致
const person1 = new Person('张三', 18)
const person2 = new Person('李四', 20)
```

- 实例属性：实例化对象后，通过 `实例对象.属性名` 访问的属性
- 类属性/静态属性：通过 `类名.属性名` 访问的属性
- 实例方法：实例化对象后，通过 `实例对象.方法名` 调用的方法
- 类方法/静态方法：通过 `类名.方法名` 调用的方法
- 检测方法：`实例对象 instanceof 类名`

```javascript
class Person {
  // 实例属性
  name = '实例属性'
  // 类属性/静态属性
  static staticProperty = '静态属性'
  // 实例方法
  instanceMethod() {
    console.log('实例方法')
  }
  // 类方法/静态方法
  static staticMethod() {
    console.log('静态方法')
  }
}

// 实例属性、方法
const person = new Person()
console.log(person.name) // 实例属性
person.instanceMethod() // 实例方法

// 类属性、方法
console.log(Person.staticProperty) // 静态属性
Person.staticMethod() // 静态方法

// 检测方法
console.log(person instanceof Person) // true
```

- 构造函数 `constructor`：实例化对象时自动执行的函数，用于初始化实例属性

```javascript
class Person {
  constructor(name, age) {
    this.name = name
    this.age = age
  }
}
const person = new Person('张三', 18)
console.log(person.name) // 张三
console.log(person.age) // 18
```

### 三大特性

封装（安全性）

- 将功能、数据封装成对象，方便调用。内部的具体内容怎么实现，外部不用关心，只需要知道怎么调用即可，保护了内部数据的安全
- 对象就是一个用来存储不同属性的容器（装），对象不仅存储属性，还要负责数据的安全（封）
- 直接添加到对象中的属性，并不安全，因为它们可以被任意的修改
- 如何确保数据的安全：
  - 私有化数据：将需要保护的数据设置为私有，只能在类内部使用，在属性前面加 `#`
  - 提供 `getter` 和 `setter` 方法来开放对数据的操作：可以控制属性的读写权限、可以在方法中对属性的值进行验证

```javascript
// getter 方法，用来读取属性
get 属性名 () {
  return this.#属性
}
// setter 方法，用来设置属性
set 属性名 (参数) {
  this.#属性 = 参数
}
```

```javascript
class Person {
  // 实例使用#开头就变成了私有属性，私有属性只能在类内部访问，私有属性需要先在前面定义
  #name
  #age
  #gender
  constructor(name, age, gender) {
    this.#name = name
    this.#age = age
    this.#gender = gender
  }
  sayHello() {
    console.log(this.#name)
  }

  // 旧getter方法，用来读取属性
  getName() {
    return this.#name
  }
  // 旧setter方法，用来设置属性
  setName(name) {
    this.#name = name
  }

  // 新getter方法，用来读取属性
  get gender() {
    return this.#gender
  }
  // 新setter方法，用来设置属性
  set gender(gender) {
    this.#gender = gender
  }
}

const p1 = new Person('孙悟空', 18, '男')
// 旧方法
p1.getName()
p1.setName('猪八戒')
// 新方法
p1.gender
p1.gender = '女'
```

继承（拓展性）

- 子类继承父类，复用父类代码
- ES6 语法糖：`class extends`
- `OCP` 开闭原则：程序应该对修改关闭，对扩展开放。指对父类对象代码禁止修改，对子类对象可以创建同名方法来修改

```javascript
// 父类
class Person {
  constructor(name, age) {
    this.name = name
    this.age = age
  }
  sayHi() {
    console.log('我是' + this.name)
  }
}

// 子类继承父类
class Student extends Person {
  constructor(name, age, score) {
    // 重写构造函数，必须用super()调用父类构造函数，初始化父类属性
    super(name, age)
    this.score = score
  }
  // 子类方法
  study() {
    console.log('学习')
  }
  // 重写父类方法：通过创建同名方法来覆盖父类的方法
  sayHi() {
    // super.sayHi() // 调用父类方法
    console.log('我是' + this.name + '，我正在学习')
  }
}

// 实例化子类
const student = new Student('张三', 18, 100)
console.log(student.name) // 张三
console.log(student.age) // 18
console.log(student.score) // 100
student.sayHi() // 我是张三
student.study() // 学习
```

多态（灵活性）

- 同一个方法，不同的对象有不同的实现
- 多态的前提：继承、方法重写

```javascript
// 父类
class Animal {
  constructor(name) {
    this.name = name
  }
  speak() {
    console.log(this.name + '发出声音')
  }
}

// 子类继承父类
class Dog extends Animal {
  speak() {
    console.log(this.name + '汪汪汪')
  }
}
class Cat extends Animal {
  speak() {
    console.log(this.name + '喵喵喵')
  }
}

// 实例化子类
const dog = new Dog('旺财')
const cat = new Cat('咪咪')

// 调用方法，不同的对象有不同的实现
dog.speak() // 旺财汪汪汪
cat.speak() // 咪咪喵喵喵
```

### 对象的结构

- 对象中存储属性的区域实际有两个：对象自身和原型对象
  - 对象自身
    - 直接通过对象所添加的属性，位于对象自身中
    - 在类中通过 x = y 的形式添加的属性，位于对象自身中
  - 原型对象
    - 对象中还有一些内容，会存储到其他的对象里（原型对象）
    - 在对象中会有一个属性用来存储原型对象，这个属性叫做 `__proto__`
    - 原型对象也负责为对象存储属性，当我们访问对象中的属性时，会优先访问对象自身的属性，对象自身不包含该属性时，才会去原型对象中寻找
- 会添加到原型对象中的情况：
  - 在类中通过 `xxx(){}` 方式添加的方法，位于原型中
  - 主动向原型中添加的属性或方法

```javascript
class Person {
  // 这些内容会添加到对象自身中
  constructor(name, age) {
    this.name = name
    this.age = age
  }
  sayHi = () => console.log('Hi，我是', this.name)
  // 这些内容会添加到原型中
  sayHello() {
    console.log('Hello，我是', this.name)
  }
}

const p = new Person('孙悟空', 18, '男')
console.log(p.name, p.age) // 孙悟空 18
p.sayHi() // Hi，我是 孙悟空
p.sayHello() // Hello，我是 孙悟空
```

### 原型和原型链

原型（prototype）

- 原型是 JavaScript 中函数的一个属性，每个函数都有一个原型属性，指向一个对象
  - 构造函数的原型对象：`构造函数.prototype`
  - 实例对象的原型对象：`实例对象.__proto__`、`Object.getPrototypeOf(实例对象)`
  - 且构造函数的原型对象和实例对象的原型对象指向的是同一个对象 `构造函数.prototype === 实例对象.__proto__`
  - 所有函数的原型（prototype）都是对象，所有对象的 `__proto__` 最终指向 `Object.prototype`
  - 只有 `new` 出来的才是实例对象
- 原型对象中的数据：对象中的数据（属性、方法等）；默认有一个 `constructor` 指向构造函数本身

```javascript
class Person {
  name = '孙悟空'
  age = 18
  sayHello() {
    console.log('Hello，我是', this.name)
  }
}
const p = new Person()
// 构造函数的原型对象
console.log(Person.prototype)
// 实例对象的原型对象
console.log(p.__proto__) // 不安全，容易被修改
console.log(Object.getPrototypeOf(p)) // 安全，不易被修改
console.log(Object.getPrototypeOf(p) === p.__proto__) // true
console.log(Person.prototype === p.__proto__) // true
// constructor 指向构造函数本身
console.log(p.constructor === Person)
```

原型链（prototype chain）

- 实例对象有原型，原型对象也有原型，这样就构成了一条原型链，根据对象的复杂程度不同，原型链的长度也不同
  - `p`对象的原型链：`p`对象 --> `obj`对象（Person） --> `obj`对象原型 --> `null`
  - `obj`对象的原型链：`obj`对象 --> `obj`对象原型 --> `null`
  - 中间的对象为前一个的原型对象，后一个的实例对象
- 原型链：读取对象属性时，会优先对象自身属性，如果对象中有，则使用，没有则去对象的原型中寻找，如果原型中有，则使用，没有则去原型的原型中寻找，直到找到 `Object` 对象的原型（`Object` 的原型没有原型【为`null`】），如果依然没有找到，则返回 `undefined`
  - 作用域链，是找变量的链，找不到会报错
  - 原型链，是找属性的链，找不到会返回 `undefined`

```javascript
class Person {
  name = '孙悟空'
  age = 18
  sayHello() {
    console.log('Hello，我是', this.name)
  }
}

const p = new Person()
console.log(p) // Person { name: '孙悟空', age: 18 }
console.log(p.__proto__) // Object { sayHello: f }
console.log(p.__proto__.__proto__) // Object { } 的原型对象
console.log(p.__proto__.__proto__.__proto__) // null
```

原型的作用

- 所有的同类型对象它们的原型对象都是同一个，也就意味着，同类型对象的原型链是一样的
- 原型的作用：原型就相当于是一个公共的区域，可以被所有该类实例访问。可以将该类实例中，所有的公共属性（方法）统一存储到原型中，这样我们只需要创建一个属性，即可被所有实例访问
- 在对象中有些值是对象独有的，像属性（name，age，gender）每个对象都应该有自己值，但是有些值对于每个对象来说都是一样的，像各种方法，对于一样的值没必要重复的创建

```javascript
class Person {
  name = '孙悟空'
  age = 18
  sayHello() {
    console.log('Hello，我是', this.name)
  }
}
const p1 = new Person()
const p2 = new Person()
console.log(p1 === p2) // false
console.log(p1.__proto__ === p2.__proto__) // true
```

- JavaScript 中继承就是通过原型来实现的，当继承时，子类的原型就是一个父类的实例

```javascript
class Animal {}
class Cat extends Animal {}
class TomCat extends Cat {}

const cat = new Cat()
console.log(cat) // Cat 实例
console.log(cat.__proto__) // Cat.prototype，也就是 Cat 的原型对象
console.log(cat.__proto__.__proto__) // Animal.prototype，Object 也就是 Animal 的原型对象
console.log(cat.__proto__.__proto__.__proto__) // Object.prototype，也就是 Object 的原型对象
console.log(cat.__proto__.__proto__.__proto__.__proto__) // null
// cat --> Cat.prototype --> Animal.prototype --> Object.prototype --> null

const tomCat = new TomCat()
console.log(tomCat) // TomCat 实例
console.log(tomCat.__proto__) // TomCat.prototype，也就是 TomCat 的原型对象
console.log(tomCat.__proto__.__proto__) // Cat.prototype，也就是 Cat 的原型对象
console.log(tomCat.__proto__.__proto__.__proto__) // Animal.prototype，Object 也就是 Animal 的原型对象
console.log(tomCat.__proto__.__proto__.__proto__.__proto__) // Object.prototype，也就是 Object 的原型对象
console.log(tomCat.__proto__.__proto__.__proto__.__proto__.__proto__) // null
// tomCat --> TomCat.prototype --> Cat.prototype --> Animal.prototype --> Object.prototype --> null
```

修改原型

- 大部分情况下，我们是不需要修改原型对象
  - 千万不要通过类的实例去修改原型：1.通过一个对象影响所有同类对象，这么做不合适；2.修改原型先得创建实例，麻烦；3.危险。
- 处理通过 `__proto__` 能访问对象的原型外，还可以通过类的 `prototype` 属性，来访问实例的原型，修改原型时，最好通过通过类去修改。
  - 好处：1.一修改就是修改所有实例的原型；2.无需创建实例即可完成对类的修改。
  - 原则：1.原型尽量不要手动改；2.要改也不要通过实例对象去改，通过 `类.prototype` 属性去修改；3.最好不要直接给 `prototype` 去赋值。

```javascript
class Person {
  name = '孙悟空'
  age = 18
  sayHello() {
    console.log('Hello，我是', this.name)
  }
}

// 通过类.prototype 修改原型
Person.prototype.fly = () => {
  console.log('我在飞！')
}

const p = new Person()
const p2 = new Person()
// 通过对象修改原型，向原型中添加方法，修改后所有同类实例都能访问该方法 不要这么做
p.__proto__.run = () => {
  console.log('我在跑~')
}
p2.run() // 我在跑~
// 直接为对象赋值了一个新的原型 不要这么做
p.__proto__ = new Dog()
```

两个方法

- `实例对象 instanceof 类` 判断一个对象是否是一个类的实例，返回布尔值（实际上检查原型链）

```javascript
class Person {}
class Animal {}
class Dog extends Animal {}

const p = new Person()
const d = new Dog()

console.log(p instanceof Person) // true
console.log(p instanceof Animal) // false
console.log(p instanceof Object) // true
console.log(p instanceof Dog) // false

console.log(d instanceof Person) // false
console.log(d instanceof Animal) // true
console.log(d instanceof Object) // true
console.log(d instanceof Dog) // true
```

- `属性名 in 对象` 判断一个对象是否包含某个属性，返回布尔值（包括对象自身和原型对象）
- `对象.hasOwnProperty(属性名)` 判断一个对象是否包含某个属性，返回布尔值（只包括对象自身，不包括原型对象）（实例方法已不推荐使用）
- `Object.hasOwn(对象, 属性名)` ES13/ES2022新增，判断一个对象是否包含某个属性，返回布尔值（只包括对象自身，不包括原型对象）（静态方法推荐使用）

```javascript
class Person {
  name = '孙悟空'
  age = 18
  sayHello() {
    console.log('Hello，我是', this.name)
  }
}

const p = new Person()
console.log('name' in p) // true
console.log('age' in p) // true
console.log('sayHello' in p) // true
console.log(p.hasOwnProperty('name')) // true
console.log(p.hasOwnProperty('age')) // true
console.log(p.hasOwnProperty('sayHello')) // false
console.log(Object.hasOwn(p, 'name')) // true
console.log(Object.hasOwn(p, 'age')) // true
console.log(Object.hasOwn(p, 'sayHello')) // false
```

### `new` 运算符

- `new` 运算符的作用：创建一个类的实例对象，并返回该实例对象。
- `new` 运算符的执行过程：1.创建一个普通的JS对象（Object对象{}），为了方便，称其为新对象；2.将构造函数的prototype属性设置为新对象的原型；3.使用实参来执行构造函数，并且将新对象设置为函数中的this；4.如果构造函数返回的是一个非原始值，则该值会作为new运算的返回值返回（千万不要这样做）。如果构造函数的返回值是一个原始值或者没有指定返回值，则新的对象将会作为返回值返回。

- 学习对象：1.明确这个对象代表什么，有什么用；2.如何获取到这个对象；3.如何使用这个对象（对象中的属性和方法）。

## 深浅拷贝与循环引用

### 简单类型与复杂类型

简单类型与复杂类型

- 简单类型又叫做基本数据类型或者值类型，复杂类型又叫做引用类型
- 值类型：简单数据类型/基本数据类型，在存储时变量中存储的是值本身，因此叫做值类型
- 引用类型：复杂数据类型，在存储时变量中存储的仅仅是地址（引用），因此叫做引用数据类型

堆和栈

- 栈（操作系统）：**简单数据类型存放到栈里面**，由操作系统自动分配释放存放函数的参数值、局部变量的值等。其操作方式类似于数据结构中的栈
- 堆(操作系统)：**复杂数据类型存放到堆里面**，一般由程序员分配释放，若程序员不释放，由垃圾回收机制回收

  > 注意：JavaScript 中没有堆栈的概念，通过堆栈的方式，可以让大家更容易理解代码的一些执行方式，便于将来学习其他语言。

两种类型的内存分配

- 简单类型的内存分配：值类型变量的数据直接存放在变量（栈空间）中
- 复杂类型的内存分配：引用类型变量（栈空间）里存放的是地址，真正的对象实例存放在堆空间中

两种类型传参

- 简单类型传参：函数的形参也可以看做是一个变量，当我们把一个值类型变量作为参数传给函数的形参时，其实是把变量在栈空间里的值复制了一份给形参，那么在方法内部对形参做任何修改，都不会影响到的外部变量
- 复杂类型传参：函数的形参也可以看做是一个变量，当我们把引用类型变量传给形参时，其实是把变量在栈空间里保存的堆地址复制给了形参，形参和实参其实保存的是同一个堆地址，所以操作的是同一个对象

```javascript
function fn(a) {
  a++
  console.log(a)
}
var x = 10
fn(x)
console.log(x)
```

```javascript
function Person(name) {
  this.name = name
}
function f1(x) {
  // x = p
  console.log(x.name) // 2．这个输出什么 ?  刘德华
  x.name = '张学友'
  console.log(x.name) // 3．这个输出什么 ?  张学友
}
var p = new Person('刘德华')
console.log(p.name) // 1．这个输出什么 ?  刘德华
f1(p)
console.log(p.name) // 4.这个输出什么 ?  张学友
```

### 深拷贝与浅拷贝

赋值、浅拷贝、深拷贝

- 赋值（传递引用地址，原对象修改，赋值对象也修改）
- 浅拷贝（拷贝第一层属性，引用类型属性共享引用）
  - 浅拷贝 `shallow copy`：只复制对象的第一层属性，如果属性值是引用类型，则复制的是引用地址，而不是引用对象本身。因此，如果修改了原对象中的引用类型属性，那么浅拷贝得到的新对象中的对应属性也会受到影响。
  - 浅拷贝的实现方式：
    - 使用扩展运算符 `...` 方法
    - 使用 `Object.assign()` 方法
    - 使用 `Array.from()` 方法
    - 使用 `arr.slice()` 方法
    - 使用 `arr.concat()` 方法
- 深拷贝（拷贝所有层级属性，原对象与拷贝对象完全独立，互不影响）
  - 深拷贝 `deep copy`：不仅复制对象的第一层属性，还会递归复制对象中的所有引用类型属性，直到复制到最底层为止。因此，深拷贝得到的新对象与原对象是完全独立的，修改新对象不会影响原对象，反之亦然。
  - 深拷贝的实现方式：
    - 使用 `JSON.parse()`、`JSON.stringify()`（注意：这种方法不能处理函数、`undefined`、`Symbol`、循环引用等特殊情况）
    - 使用递归函数实现深拷贝（手动遍历所有层级，处理基本类型和引用类型，可解决JSON方法的局限性）
    - 使用 `structureClone()` 方法（新API，支持循环引用、RegExp，不支持函数）
    - 使用第三方库，如 `lodash` 的 `cloneDeep` 方法
- 注意：原始值（如数字、字符串、布尔值）在赋值时是深拷贝，因为原始值是不可变的，所以复制原始值不会影响原值

```javascript
// 浅拷贝
const obj = { a: 1, b: { c: 2 } }
const shallowCopy1 = { ...obj }
const shallowCopy2 = Object.assign({}, obj)
const arr = [1, 2, 3]
const shallowCopy3 = [...arr]
const shallowCopy4 = Array.from(arr)
const shallowCopy5 = arr.slice()
const shallowCopy6 = arr.concat()

// 深拷贝
const obj = { a: 1, b: { c: 2 } }
const deepCopy1 = JSON.parse(JSON.stringify(obj))
const deepCopy2 = structureClone(obj)
```

### 循环引用

WeakMap 解决循环引用

- 递归深拷贝时，用WeakMap缓存已拷贝的对象，遇到已缓存对象直接返回，避免循环引用导致的栈溢出（如 `obj1 = {a: obj2}`，`obj2 = {b: obj1}`）。

## 异步编程

### 异步基础

进程和线程

- 进程（process）：是计算机中程序关于某数据集合上的一次运行活动，是系统进行资源分配和调度的基本单位，是操作系统结构的基础。在早期面向进程设计的计算机结构中，进程是程序的基本执行实体；在当代面向线程设计的计算机结构中，进程是线程的容器。程序是指令、数据及其组织形式的描述，进程是程序的实体。【厂房】
- 线程（thread）：是操作系统能够进行运算调度的最小单位，被包含在进程之中，是进程中的实际运作单位。一条线程指的是进程中一个单一顺序的控制流，一个进程中可以并发多个线程，每条线程并行执行不同的任务。【工人】

同步和异步

- JavaScript 是单线程的，只能同时执行一个任务，其他任务需要排队等待。
- 同步：代码顺序执行，前一句执行完才执行后一句，会阻塞。
- 异步：代码不顺序执行，不会阻塞，如定时器、AJAX。

调用栈和任务队列

- 调用栈（call stack）：用于存储函数调用关系，先进后出（像枪压子弹）。
- 任务队列（task queue）：用于存储异步任务，先进先出，分为宏任务队列和微任务队列。
- 宏任务（Macrotask/Task）：`script` / `setTimeout` / `setInterval` / `I/O` / `UI渲染` / `setImmediate（Node）`，执行优先级低于微任务。
- 微任务（Microtask/Job）：`Promise.then` / `MutationObserver` / `queueMicrotask()` / `process.nextTick（Node）`，执行优先级高于宏任务。
- 事件循环（event loop）：JS引擎通过事件循环来处理异步任务，分为宏任务队列和微任务队列。事件循环的顺序：
  - 执行一个宏任务（通常是 script 脚本本身）。
  - 执行过程中，遇到宏任务就把它放到宏任务队列，遇到微任务就把它放到微任务队列。
  - 当前宏任务执行完毕后，立即检查微任务队列，并执行里面所有的微任务。
  - 所有微任务执行完毕后，如有需要，进行页面渲染。
  - 然后，从宏任务队列中取出一个任务，开始新一轮的循环。
- 执行顺序：同步代码 → 微任务队列（全部执行完毕） → 宏任务队列（执行一个） → UI渲染 → 重复上述步骤，直到所有任务执行完毕。

```javascript
const promise = Promise.resolve('成功')
// const promise = Promise.reject('失败')

// 同步
console.log('同步代码')
// 微任务
promise
  .then((res) => console.log('then1')) // then1
  .then((res) => console.log('then2')) // then2
  .catch((err) => console.log('catch1')) // catch1
// 微任务
promise
  .then((res) => console.log('then3')) // then3
  .catch((err) => console.log('catch2')) // catch2
// 微任务
queueMicrotask(() => console.log('queueMicrotask')) // queueMicrotask
// 宏任务
setTimeout(() => console.log('宏任务'), 0)

// 全部微任务的执行顺序是：then1 then3 queueMicrotask then2 catch2 catch1
// resolove 执行结果：同步代码 then1 then3 queueMicrotask then2 宏任务，微任务只执行then部分
// reject 执行结果：同步代码 queueMicrotask catch2 catch1 宏任务，微任务只执行catch部分
```

```javascript
// 运行顺序小测试
console.log(1)
setTimeout(() => console.log(2))
Promise.resolve().then(() => console.log(3))
Promise.resolve().then(() => setTimeout(() => console.log(4)))
Promise.resolve().then(() => console.log(5))
setTimeout(() => console.log(6))
console.log(7)
// 运行结果：1 7 3 5 2 6 4
```

### Promise

回调地狱

- 回调地狱（callback hell）：多层嵌套的回调函数会形成回调地狱，代码难以阅读和维护。
- 解决回调地狱的方法：Promise、async/await。

Promise

- Promise 是 ES6 引入的一种异步编程解决方案，用于处理异步操作，避免回调地狱。
- Promise 是一个构造函数，用于创建一个 Promise 对象，返回的也是一个 Promise 对象，该对象表示一个异步操作的最终完成（或失败）及其结果值。
- Promise 对象创建时需要传入一个执行器函数（executor），该函数有两个参数：
  - resolve：异步操作成功时的回调函数，接收异步操作的结果作为参数
  - reject：异步操作失败时的回调函数，接收异步操作失败的原因作为参数

```javascript
// 创建一个 Promise 对象
const promise = new Promise((resolve, reject) => {
  // 异步操作成功时调用 resolve，将结果传递给 then 方法的回调函数
  // resolve('成功')
  // 异步操作失败时调用 reject，将错误信息传递给 catch 方法的回调函数
  // reject('失败')
})
```

- Promise 对象读取的方法：
  - `.then(onFulfilled, onRejected)`：注册成功和失败的回调函数，返回一个新的 Promise 对象。
  - `.catch(onRejected)`：注册失败的回调函数，返回一个新的 Promise 对象。
  - `.finally(onFinally)`：注册一个无论成功或失败都会执行的回调函数，返回一个新的 Promise 对象。
  - 注意：上述方法返回的都是新的 Promise 对象，所以可以进行链式调用来解决回调地狱的问题。

```javascript
const promise = new Promise((resolve, reject) => {
  resolve('成功')
})

// .then()
promise.then(
  (result) => {
    console.log(result) // resolve 存储的数据
  },
  (error) => {
    console.log(error) // reject 存储的数据
  },
)
// .catch()
promise.catch((error) => {
  console.log(error) // reject 存储的数据
})
// .finally()
promise.finally(() => {
  console.log('无论成功失败都会执行') // 无论成功失败都会执行
})
```

```javascript
// Promise 的链式调用
promise
  .then(() => {})
  .then(() => {})
  .catch(() => {})
  .finally(() => {})
```

- Promise 对象中有两个隐藏的属性：
  - PromiseState：表示 Promise 对象的状态，有三种状态：pending（等待中）、fulfilled（已成功）、rejected（已失败）。状态无论成功还是失败，一旦确定，就不能再改变，状态不可逆。
  - PromiseResult：表示 Promise 对象存入的结果，可以是任意类型，当 Promise 对象的状态变为 fulfilled 或 rejected 时，PromiseResult 的值会被设置。
- 流程：
  - 当 Promise 创建时，PromiseState 初始值为 pending（等待中）
    - 当通过 resolve 存储数据时，PromiseState 变为fulfilled（已成功），PromiseResult 变为存储的数据
    - 当通过 reject 存储数据或出错时，PromiseState 变为rejected（已失败），PromiseResult 变为存储的数据或异常对象
  - 当我们通过 then 读取数据时，相当于为 Promise 设置了回调函数
    - 如果 PromiseState 变为 fulfilled，则调用 then 的第一个回调函数来返回数据
    - 如果 PromiseState 变为 rejected，则调用 then 的第二个回调函数来返回数据

```javascript
const promise0 = new Promise((resolve, reject) => {})
const promise1 = new Promise((resolve, reject) => resolve('成功'))
const promise2 = new Promise((resolve, reject) => reject('失败'))

console.log(promise0)
// Promise {[[PromiseState]]: 'pending', [[PromiseResult]]: undefined}
console.log(promise1)
// Promise {[[PromiseState]]: 'fulfilled', [[PromiseResult]]: '成功'}
console.log(promise2)
// Promise {[[PromiseState]]: 'rejected', [[PromiseResult]]: '失败'}
```

- Promise 的静态方法：
  - `Promise.resolve(value)`：返回一个已解决的 Promise 对象，相当于 `new Promise((resolve, reject) => resolve(value))`。
  - `Promise.reject(reason)`：返回一个已拒绝的 Promise 对象，相当于 `new Promise((resolve, reject) => reject(reason))`。
  - `Promise.all(iterable)`：返回一个 Promise 对象，当 iterable 中的所有 Promise 对象都成功时，该 Promise 对象才会成功，否则会失败。
  - `Promise.allSettled(iterable)`：返回一个 Promise 对象，无论 iterable 中的所有 Promise 对象成功或失败，该 Promise 对象都会全部返回。
  - `Promise.race(iterable)`：返回一个 Promise 对象，无论 iterable 中的所有 Promise 对象成功或失败，该 Promise 对象都会返回最快的一个。
  - `Promise.any(iterable)`：返回一个 Promise 对象，无论 iterable 中的所有 Promise 对象成功或失败，该 Promise 对象都会返回最快的已解决的一个。

```javascript
// Promise.all()
Promise.all([Promise.resolve(123), Promise.resolve(456)]).then((result) => {
  console.log(result) // [123, 456]
})

// Promise.allSettled()
Promise.allSettled([Promise.resolve(123), Promise.reject(456)]).then((result) => {
  console.log(result) // [{status: 'fulfilled', value: 123}, {status: 'rejected', reason: 456}]
})

// Promise.race()
Promise.race([Promise.reject(123), Promise.resolve(456)])
  .then((result) => {
    console.log(result)
  })
  .catch((error) => {
    console.log(error) // 123
  })

// Promise.any()
Promise.race([Promise.reject(123), Promise.resolve(456)])
  .then((result) => {
    console.log(result) // 456
  })
  .catch((error) => {
    console.log(error) // 123
  })
```

### async/await

- async：全称为 asynchronous，中文意思是异步的，声明一个函数是异步的，该函数返回一个 Promise 对象。

```javascript
// 正常同步函数
function fn1() {
  return 'hello'
}
console.log(fn1()) // hello

// async 异步函数
async function fn2() {
  return 'hello'
}
console.log(fn2()) // Promise {<fulfilled>: 'hello'}
```

- await：全称为 asynchronous wait，中文意思是异步等待，用于等待一个 Promise 对象的状态变为 fulfilled，并返回该 Promise 对象的结果。await 的使用，使得异步代码看起来像同步代码。
  - 当我们通过 await 去调用异步函数时，它会暂停代码的运行，直到异步代码执行有结果时，才会将结果返回
  - 注意：
    - await 只能用于 async 声明的异步函数中，或ES模块的顶级作用域中
    - await 阻塞的知识异步函数内部的代码，不会影响外部代码
    - 通过 await 调用异步代码时，需要通过 `try-catch` 来处理异常

```javascript
function sum(a, b) {
  return new Promise((resolve) => {
    resolve(a + b)
  })
}

// Promise.then()方法的链式调用
async function fn1() {
  sum(123, 456)
    .then((r) => r + 7)
    .then((r) => r + 8)
    .then((r) => r + 9)
    .then((r) => console.log(r))
}
fn1()

// async/await的链式调用
async function fn2() {
  let r = await sum(123, 456)
  r = await sum(r, 7)
  r = await sum(r, 8)
  r = await sum(r, 9)
  console.log(r)
}
fn2()

// async/await的链式调用，需要try-catch处理异常
async function fn3() {
  try {
    let r = await sum(123, 456)
    r = await sum(r, 7)
    r = await sum(r, 8)
    r = await sum(r, 9)
    console.log(r)
  } catch (error) {
    console.log(error)
  }
}
fn3()
```

- 注意：
  - 如果 async 里面没有 await，则里面的代码会同步执行
  - 如果 async 里面有 await，则里面的代码会异步执行，且当使用 await 调用函数后，后面的代码会在 await 函数执行完毕后进入微任务队列异步执行

```javascript
// async 里面没有 await，则里面的代码会同步执行
async function fn1() {
  console.log(1)
  console.log(2)
  console.log(3)
}
fn1()
console.log(4)
// 1 2 3 4

// async 里面有 await，则里面的代码会异步执行
async function fn2() {
  console.log(1)
  await console.log(2)
  // await 后面的代码会进入微任务队列异步执行
  console.log(3)
}
fn2()
console.log(4)
// 1 2 4 3
```

### 错误处理

错误类型

- SyntaxError（语法错误，如括号不匹配）
- ReferenceError（引用未声明的变量）
- TypeError（类型错误，如调用非函数）
- RangeError（范围错误，如数组长度为负数）
- URIError（URI相关错误，如decodeURI错误）
- EvalError（eval相关错误，极少出现）

错误捕获

- try：包裹可能报错的代码
- catch：捕获错误并处理
- finally：无论是否报错都执行（常用于释放资源，如关闭定时器、清理事件监听）

```javascript
try {
  // 可能会报错的代码
} catch (error) {
  // 捕获错误并进行处理
} finally {
  // 无论是否报错都执行的代码
}
```

错误抛出

- throw：抛出异常、自定义错误，中断程序执行，配合catch捕获，用于自定义错误场景（如表单验证失败）

```javascript
throw new Error('自定义错误信息')

try {
  throw new Error('自定义错误信息')
} catch (error) {
  console.log(error)
}
```

## 性能优化❌

- 重绘与回流（重排）：重绘（元素样式改变，位置不变，如color改变）；回流（元素位置、尺寸改变，如width改变，会触发重绘）；回流代价高于重绘，需减少触发。
- 事件委托：利用事件冒泡，将子元素的事件绑定到父元素上，减少事件绑定次数（如列表项点击事件，绑定到ul上），优化性能。
- DOM 缓存与批量操作：频繁获取DOM元素时，将DOM元素缓存到变量中（避免重复查询DOM）；批量修改DOM时，使用文档碎片（document.createDocumentFragment()）或display:none隐藏元素后修改，减少回流。
- 内存泄漏场景：意外全局变量（未声明的变量挂载到window）、脱离DOM的元素（元素已删除，但仍有引用）、定时器未清理、事件监听未移除、闭包滥用，需及时避免和清理。
- 避免闭包滥用：合理使用闭包，不需要时及时解除引用，防止内存泄漏。

## 常见设计模式❌

- 单例模式：保证一个类只有一个实例，全局可访问（如弹窗组件、全局缓存，通过闭包或静态属性实现）。
- 策略模式：封装不同算法，根据场景切换算法，避免多重if-else（如表单验证，不同字段使用不同验证规则）。
- 适配器模式：将一个接口转换为另一个接口，使不兼容的接口可以一起工作（如不同后端接口返回格式不同，用适配器统一格式）。
- 装饰器模式：在不修改原函数/类的前提下，扩展其功能（如给函数添加日志、权限校验，ES7装饰器语法可简化实现）。
- 发布订阅模式 / 观察者模式：发布者发布事件，订阅者订阅事件，发布者触发事件时，所有订阅者接收通知（如事件总线、Vue响应式原理）。
- 工厂模式：封装对象创建逻辑，根据参数返回不同类型的对象（如创建不同类型的组件，简化实例化过程）。

## 安全基础❌

- XSS 原理与防御：XSS（跨站脚本攻击），注入恶意脚本到页面，窃取用户信息；防御（输入转义、使用textContent而非innerHTML、开启CSP内容安全策略、设置HttpOnly Cookie）。
- CSRF 原理与防御：CSRF（跨站请求伪造），利用用户已登录状态，伪造请求操作用户账户；防御（添加Token验证、验证Referer来源、设置SameSite Cookie）。
- CSP 内容安全策略（基础）：通过HTTP响应头Content-Security-Policy，限制页面可加载的资源（如只允许加载指定域名的脚本、样式），防止XSS攻击。
