# 构建工具

## 构建工具简介

当我们习惯了在 node 中编写代码的方式后，在回到前端编写 html、css、js 这些东西会感觉到各种的不便。比如：不能放心的使用模块化规范（浏览器兼容性问题）、即使可以使用模块化规范也会面临模块过多时的加载问题。

我们就迫切的希望有一款工具可以对代码进行打包，将多个模块打包成一个文件。这样一来即解决了兼容性问题，又解决了模块过多的问题。

构建工具就起到这样一个作用，通过构建工具可以将使用 ESM 规范编写的代码转换为旧的 JS 语法，这样可以使得所有的浏览器都可以支持代码。

## Webpack

- Webpack 官网：https://webpack.js.org/
- Webpack 中文网：https://www.webpackjs.com/

### 使用步骤

1. 初始化项目

```bash
npm init -y
yarn init -y
```

2. 安装依赖核心 `webpack`、命令行 `webpack-cli`

```bash
npm i -D webpack webpack-cli
yarn add -D webpack webpack-cli
```

3. 在项目中创建 src 目录，在 src 中创建 `index.js` 然后编写代码
4. 执行 `yarn webpack` 来对代码进行打包（打包后观察 dist 目录）

```bash
npx webpack
yarn webpack
```

### 配置文件 `webpack.config.js`

```javascript
module.exports = {
  mode: 'production',
  entry: './src/index.js',
  output: {},
  module: {},
}
```

- `mode`：设置打包的模式

```js
module.exports = {
  mode: 'production', // 表示生产模式，会压缩代码
  mode: 'development', // 表示开发模式，不会压缩代码
}
```

- `entry`：用来指定打包时的主文件，默认 `./src/index.js`

```js
module.exports = {
  // 默认情况
  entry: './src/index.js'
  // 单个文件
  entry: './hello/hello.js',
  // 数组
  entry: ['./src/a.js', './src/b.js'],
  // 对象，将a打包为自定义a，将b打包为自定义b
  entry: {
    a: './src/a.js',
    b: './src/b.js',
  },
}
```

- `output`：用来指定打包后的目录

```js
const path = require('path')
module.exports = {
  // 配置代码打包后的地址
  output: {
    // path表示打包结束后的文件路径，默认为dist文件夹。指定打包的目录，必须要绝对路径
    path: path.resolve(__dirname, 'dist'),
    // filename表示打包后的文件名，默认为main.js
    filename: 'main.js',
    // filename后面可以跟entry的name，id和哈希值
    filename: '[name]-[id]-[hash].js',
    // clean表示自动清理打包目录，true/false
    clean: true,
  },
}
```

- `module`：webpack 默认情况下，只会处理 js 文件，如果我们希望它可以处理其他类型的文件，则要为其引入 loader。以 css 为例：使用 css-loader 可以处理 js 中的样式，使用 style-loader 可以展示 js 中的样式，使用 ts-loader 可以处理 ts 中的内容
  - 安装：`npm i -D css-loader style-loader ts-loader`
  - 引入 CSS：`import('./style/index.css')`、`require('./style/index.css')`

```js
module.exports = {
  // 用来配置loader
  module: {
    // 配置loader的规则
    rules: [
      {
        // 正则表达式
        test: /\.css$/i,
        // 使用的loader，从后往前写
        use: ['style-loader', 'css-loader'],
      },
      {
        // 正则表达式
        test: /\.ts$/i,
        // 使用的loader
        use: 'ts-loader',
      },
      {
        // 正则表达式
        test: /\.(jpg|png|gif)$/i,
        // 图片直接资源类型的数据，可以通过指定type来处理，asset文件夹用来存放图片直接资源
        type: 'asset/resource',
      },
    ],
  },
}
```

- `babel`：在编写 js 代码时，经常需要使用一些 js 中的新特性，而新特性在旧的浏览器中兼容性并不好。此时就导致我们无法使用一些新的特性。但是我们现在希望能够使用新的特性，我们可以采用折中的方案。依然使用新特性编写代码，但是代码编写完成时我们可以通过一些工具将新代码转换为旧代码。babel 就是这样一个工具，可以将新的 js 语法转换为旧的 js，以提高代码的兼容性。我们如果希望在 webpack 支持 babel，则需要向 webpack 中引入 babel 的 loader
  - 英文官网：https://babeljs.io/
  - 中文官网：https://www.babeljs.cn/
  - 安装：`npm i -D babel-loader @babel/core @babel/preset-env`
  - 在 package.json 中设置兼容列表，https://github.com/browserslist/browserslist 可以查看相应选项

```js
module.exports = {
  module: {
    rules: [
      {
        // 正则匹配js或mjs文件
        test: /\.m?js$/,
        // 排除这两个目录下的内容
        exclude: /(node_modules|bower_components)/,
        use: {
          // 使用babel-loader
          loader: 'babel-loader',
          // babel-loader的配置预设环境
          options: {
            presets: ['@babel/preset-env'],
          },
        },
      },
    ],
  },
}
```

```json
// package.json
// 兼容的浏览器的列表， "defaults"表示默认
{
  "browserslist": ["defaults"]
}
```

- `plugin`：插件，用来为 webpack 来扩展功能
  - `html-webpack-plugin`：这个插件可以在打包代码后，自动在打包目录生成 html 页面
    - 安装依赖`npm i -D html-webpack-plugin`
    - 配置插件
  - 插件的一般流程都是如此，先安装依赖，在配置插件即可

```js
// 引入该插件
const HTMLPlugin = require('html-webpack-plugin')
module.exports = {
  // 用来配置插件
  plugins: [
    new HTMLPlugin({
      // 设置标题，一般不需要，了解即可
      title: 'Hello Webpack',
      // 设置模版路径
      template: './src/index.html',
      // 模板导出的路径，会在多界面时使用，entry需要修改为多文件
      filename: './home/index.html',
      // 该模板匹配的js文件，会在多界面时使用，entry需要修改为多文件
      chunks: ['main'],
    }),
    // 配置多个页面时，配置多个HTMLPlugin即可
    // new HTMLPlugin({ ... })
  ],
}
```

- `webpack-dev-server`：开发服务器
  - 终端输入 `npx webpack --watch` 可以随时监视文件的变化，但是该方法是以文件形式打开，所以并不会使用该方法
  - 安装：`npm i -D webpack-dev-server`
  - 启动：`npx webpack server`、`npx webpack serve --open`
- `source-map`：配置源码的映射，配置完成之后可以在浏览器看到打包前的源码

```js
module.exports = {
  // 显示源码的映射工具
  devtool: 'inline-source-map',
}
```

- 关闭性能警告：如果只是临时不想看到性能警告，可以在 webpack 配置中修改体积阈值

```js
module.exports = {
  performance: {
    // 完全关闭性能警告
    hints: false,
    // 单个资源最大500KiB
    maxAssetSize: 512000,
    // 入口资源最大500KiB
    maxEntrypointSize: 512000,
  },
}
```

### Webpack 完整使用

1. 初始化项目

```bash
# 初始化项目
npm init -y
yarn init -y

# 创建src文件夹
mkdir src

# 创建index.html
touch src/index.html

# 创建index.js
touch src/index.js
```

2. 安装全部依赖

```bash
# 分别安装依赖
npm i -D webpack webpack-cli
npm i -D css-loader less-loader style-loader ts-loader
npm i -D babel-loader @babel/core @babel/preset-env
npm i -D html-webpack-plugin
npm i -D webpack-dev-server

# 一步安装全部依赖
npm i -D webpack webpack-cli css-loader less-loader style-loader ts-loader babel-loader @babel/core @babel/preset-env html-webpack-plugin webpack-dev-server
```

3. 配置 `webpack.config.js`

```js
// webpack.config.js 单页面
const path = require('path')
const HTMLPlugin = require('html-webpack-plugin')
module.exports = {
  mode: 'production',
  entry: './src/index.js',
  output: {
    path: path.resolve(__dirname, 'dist'),
    filename: 'main.js',
    clean: true,
  },
  module: {
    rules: [
      { test: /\.css$/i, use: ['style-loader', 'css-loader'] },
      { test: /\.less$/i, use: ['style-loader', 'css-loader', 'less-loader'] },
      { test: /\.ts$/i, use: 'ts-loader' },
      { test: /\.(jpg|png|gif)$/i, type: 'asset/resource' },
      {
        test: /\.m?js$/,
        exclude: /(node_modules|bower_components)/,
        use: {
          loader: 'babel-loader',
          options: {
            presets: ['@babel/preset-env'],
          },
        },
      },
    ],
  },
  plugins: [
    new HTMLPlugin({
      template: './src/index.html',
    }),
  ],
  devtool: 'inline-source-map',
  performance: {
    hints: false,
  },
}
```

```js
// webpack.config.js 多页面
// 打包后的每个页面存在于单个文件夹下，且每个文件夹内容均为index.html和index.js，具体情况再做修改
const path = require('path')
const HTMLPlugin = require('html-webpack-plugin')
module.exports = {
  mode: 'production',
  entry: {
    page1: './src/page1/index.js',
    page2: './src/page2/index.js',
  },
  output: {
    path: path.resolve(__dirname, 'dist'),
    filename: './[name]/index.js',
    clean: true,
  },
  module: {
    rules: [
      { test: /\.css$/i, use: ['style-loader', 'css-loader'] },
      { test: /\.less$/i, use: ['style-loader', 'css-loader', 'less-loader'] },
      { test: /\.ts$/i, use: 'ts-loader' },
      { test: /\.(jpg|png|gif)$/i, type: 'asset/resource' },
      {
        test: /\.m?js$/,
        exclude: /(node_modules|bower_components)/,
        use: {
          loader: 'babel-loader',
          options: {
            presets: ['@babel/preset-env'],
          },
        },
      },
    ],
  },
  plugins: [
    new HTMLPlugin({
      template: './src/page1/index.html',
      filename: './page1/index.html',
      chunks: ['page1'],
    }),
    new HTMLPlugin({
      template: './src/page2/index.html',
      filename: './page2/index.html',
      chunks: ['page2'],
    }),
    // ...
  ],
  devtool: 'inline-source-map',
  performance: {
    hints: false,
  },
}
```

4. 配置 `package.json`

```json
// package.json补充
{
  "scripts": {
    "build": "webpack",
    "watch": "webpack --watch",
    "serve": "webpack serve",
    "open": "webpack serve --open"
  },
  "browserslist": ["defaults"]
}
```

5. 启动项目

```bash
# 可用命令
npx webpack # 进行打包 npm run build
npx webpack --watch # 进行打包，并进行热更新 npm run watch
npx webpack serve # 开启服务器进行打包 npm run serve
npx webpack serve --open # 开启服务器进行打包，并自动打开网页 npm run open
```

## Vite

- Vite 英文官网：https://vite.dev/
- Vite 中文官网：https://cn.vitejs.dev/
- Vite 中文网：https://vitejs.cn/

### 使用步骤

1. 初始化项目

```bash
npm init -y
yarn init -y
```

2. 安装依赖核心 `vite`

```bash
npm i -D vite
yarn add -D vite
```

3. Vite的源码目录就是项目根目录，创建一个像这样的 `index.html` 文件

```html
<!-- ES模块化，需要 type="module" -->
<script type="module" src="./src/index.js"></script>

<p>Hello Vite!</p>
```

4. 执行 `yarn vite` 来对代码进行打包

```bash
npx vite
yarn vite
```

5. 所有的 vite 命令

```json
// package.json
"scripts": {
  "dev": "vite", // 启动开发服务器
  "build": "vite build", // 打包代码
  "preview": "vite preview" // 预览打包后dist代码
},
```

6. 快速搭建一个vite项目

```bash
npm create vite@latest
yarn create vite
```

### 配置文件 `vite.config.js`

- 更改vite的兼容性

```javascript
import { defineConfig } from 'vite'
import legacy from '@vitejs/plugin-legacy'

// defineConfig写不写都可以，写上可以显示代码提示
export default defineConfig({
  plugins: [
    legacy({
      targets: ['defaults'],
    }),
  ],
})
```

- 需要注意的是：Vite 项目打包后的 dist 目录中的文件，不能直接在本地查看，需要部署至服务器上才可查看，或者 `npm run preview` 来进行查看
