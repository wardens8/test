## ES6/ES2015 新特性

### 块级作用域与变量声明

`var` 的问题

- 作用域：只有全局作用域和函数作用域，无块级作用域
- 变量提升：声明会被提升到当前作用域顶部（只提升声明，不提升赋值）
- 重复声明：允许同一作用域内重复声明同一个变量，后声明会覆盖前一个

```javascript
if (true) {
  var a = 10 // 看似在 if 块里，实际是全局/函数作用域
}
console.log(a) // 输出 10（能访问到，因为没有块级作用域限制）
```

```javascript
console.log(b) // 输出 undefined（提升了声明，未赋值）
var b = 20
// 等价于：var b; console.log(b); b = 20;
```

```javascript
var c = 30
var c = 40
console.log(c) // 输出 40（无报错，覆盖了）
```

```javascript
// var 的循环问题
for (var i = 0; i < 5; i++) {
  setTimeout(() => {
    console.log(i) // 输出 5（循环结束后，i 已经是 5 了）
  })
}
console.log(i) // 输出 5（数据泄露到全局作用域，能访问到）
```

块级作用域

- 以 `{}` 为边界的作用域。ES5 只有「全局作用域」和「函数作用域」，没有块级作用域，这也是 var 的核心问题；ES6 新增块级作用域，let/const 正是基于块级作用域设计的
- 替代传统 var 变量污染，解决循环、条件块作用域问题

`let` 声明变量

- 块级作用域：只在声明的 `{}` 内有效，外部无法访问
- 无变量提升：必须先声明再使用，存在「暂时性死区」（temporal dead zone，简称 TDZ），声明前访问会报错
- 不可重复声明：同一作用域内不能重复声明同一个变量（包括不能和 var/const 重复）

```javascript
if (true) {
  let a = 10
}
console.log(a) // 报错：a is not defined（块外访问不到）
```

```javascript
{
  console.log(b) // 报错：Cannot access 'b' before initialization
  let b = 20
}
```

```javascript
{
  let c = 30
  let c = 40 // 报错：Identifier 'c' has already been declared
  var c = 50 // 同样报错（同一作用域）
}
```

`const` 声明常量

- `const` 完全继承 `let` 的所有特性（块级作用域、无变量提升、不可重复声明），额外增加两个规则
- 声明即初始化：声明时必须赋值，不能只声明不赋值
- 值不可修改（关键易错点）：基本类型（数字、字符串、布尔等）值完全不能改。引用类型（对象、数组等）仅内存地址不可变，内部属性/元素可以改

```javascript
const a // 报错：Missing initializer in const declaration
const b = 20 // 正确
```

```javascript
const c = 30
c = 40 // 报错：Assignment to constant variable.

const obj = { name: 'Tom' }
obj.name = 'Jack' // 不报错（地址没变，只是内部属性改了）
obj = { age: 18 } // 报错（要修改地址，不允许）

const arr = [1, 2, 3]
arr.push(4) // 不报错（数组地址没变）
arr = [5, 6] // 报错（修改地址）
```

### 解构赋值

- 数组的解构赋值
- 对象的解构赋值
- 字符串的解构赋值
- 数值和布尔值的解构赋值
- 函数参数的解构赋值
- 圆括号问题
- 用途

---

- 解构赋值：从数组或对象中提取数据，并赋值给新的变量。本质上属于“模式匹配”，把数组/对象的结构拆解，然后将对应位置的值赋值给变量
- 核心规则：只要等号两边的模式相同，左边的变量就会被赋予对应的值
- 作用：简化提取数据（不用再通过索引 / 属性名取值），减少代码量，提高可读性

数组的解构赋值（有序）

```javascript
// 基本用法
let [a, b, c] = [1, 2, 3]
console.log(a, b, c) // 输出 1 2 3

// 嵌套数组
let [a, [b, c]] = [1, [2, 3]]
console.log(a, b, c) // 输出 1 2 3

// 不完全解构1
let [a, b] = [1, 2, 3]
console.log(a, b) // 输出 1 2

// 不完全解构2
let [a, b] = [1]
console.log(a, b) // 输出 1 undefined

// 空缺位置
let [a, , c] = [1, 2, 3]
console.log(a, c) // 输出 1 3

// 剩余运算符
let [a, ...b] = [1, 2, 3, 4]
console.log(a, b) // 输出 1 [2, 3, 4]

// 默认值
let [a = 1, b = 2] = [3]
console.log(a, b) // 输出 3 2（a 取传入值，b 取默认值）
// 注意：undefined 会触发默认值，null 不会触发默认值
let [a = 1, b = 2] = [null, undefined]
console.log(a, b) // 输出 null 2（a 取传入值，b 取传入值）
```

对象的解构赋值（无序）

```javascript
// 基本用法
let { a, b } = { a: 1, b: 2 }
console.log(a, b) // 输出 1 2

// 嵌套对象
let {
  a,
  b: { c },
} = { a: 1, b: { c: 2 } }
console.log(a, c) // 输出 1 2

// 重命名变量
let { a: num1, b: num2 } = { a: 1, b: 2 }
console.log(num1, num2) // 输出 1 2
// 这实际上说明，对象的解构赋值是下面形式的简写
let { a: a, b: b } = { a: 1, b: 2 }

// 不完全解构1
let { a, b } = { a: 1, b: 2, c: 3 }
console.log(a, b) // 输出 1 2

// 不完全解构2
let { a, b } = { a: 1 }
console.log(a, b) // 输出 1 undefined

// 剩余运算符
let { a, ...b } = { a: 1, b: 2, c: 3 }
console.log(a, b) // 输出 1 { b: 2, c: 3 }

// 默认值
let { a = 1, b = 2 } = { a: 3 }
console.log(a, b) // 输出 3 2（a 取传入值，b 取默认值）
// 注意：undefined 会触发默认值，null 不会触发默认值
let { a = 1, b = 2 } = { a: null, b: undefined }
console.log(a, b) // 输出 null 2（a 取传入值，b 取默认值）
```

字符串的解构赋值

```javascript
// 基本用法
let [a, b, c, d, e] = 'hello'
console.log(a, b, c, d, e) // 输出 h e l l o

// 剩余运算符
let [a, ...b] = 'hello'
console.log(a, b) // 输出 h ['e', 'l', 'l', 'o']
```

函数参数的解构赋值

```javascript
// 基本用法1
function add([a, b]) {
  return a + b
}
console.log(add([1, 2])) // 输出 3
// 基本用法2
function add({ a, b }) {
  return a + b
}
console.log(add({ a: 1, b: 2 })) // 输出 3

// 默认值
function add({ a = 1, b = 2 } = {}) {
  return a + b
}
console.log(add({ a: 3 })) // 输出 5（a 取传入值，b 取默认值）
```

### 模板字符串

- 模板字符串：增强版的字符串，用反引号（`）标识，可包含变量、表达式、多行字符串，无需\n拼接
- 核心规则：用 `${}` 语法嵌入变量或表达式，支持换行、任意字符，无需拼接
- 嵌套模板字符串：模板字符串可以嵌套使用，适合处理复杂的字符串逻辑

```javascript
// 基本用法
let name = 'Tom'
let age = 18
let str = `My name is ${name}, I'm ${age} years old.` // 输出：My name is Tom, I'm 18 years old.

// 多行字符串
let str = `
  <div>
    <p>Hello</p>
    <p>World</p>
  </div>
`
```

```javascript
// 嵌入变量
let name = 'Tom'
let age = 18
let str = `My name is ${name}, I'm ${age} years old.` // 输出：My name is Tom, I'm 18 years old.

// 嵌入表达式
let a = 1
let b = 2
let str = `a + b = ${a + b}` // 输出：a + b = 3

// 函数调用
const getAge = () => 18
const str = `年龄：${getAge()}` // 年龄：18

// 三元表达式
const isAdult = true
const str = `是否成年：${isAdult ? '是' : '否'}` // 是否成年：是
```

- 模板字符串 vs 传统字符串拼接

```javascript
// 传统拼接（易出错，可读性差）
const user = { name: 'Tom', age: 18, gender: '男' }
const info1 = '姓名：' + user.name + '，年龄：' + user.age + '，性别：' + user.gender

// 模板字符串（简洁清晰）
const info2 = `姓名：${user.name}，年龄：${user.age}，性别：${user.gender}`
// 结果：姓名：Tom，年龄：18，性别：男
```

### 箭头函数

- 箭头函数：ES6 允许使用“箭头”（=>）定义函数，箭头函数提供了一种更简洁的函数书写方式
- 基本语法：`() => {}`，箭头函数用箭头（`=>`）来定义函数，箭头左边是参数，右边是函数体
- 简化写法：如果函数体只有一个表达式，可以省略大括号和 return，直接返回表达式的值
- 简化返回对象：如果返回对象，需要用小括号包裹对象，否则会被解析成代码块
- 简化回调函数：箭头函数可以简化回调函数的写法，使代码更简洁易读
- 箭头函数的 this 指向：箭头函数没有自己的 this，它在定义时 this 就已经生成，无法改变，它会捕获其所在上下文的 this 值，作为自己的 this 值。因此，箭头函数中的 this 值是固定的，不会受函数调用方式的影响

```javascript
// 基本用法
const add = (a, b) => {
  return a + b
}

// 简化函数体
const add = (a, b) => a + b

// 简化返回对象
const user = (name, age) => ({ name, age })

// 简化回调函数
const arr = [1, 2, 3, 4, 5]
const newArr = arr.map((item) => item * 2)
console.log(newArr) // 输出 [2, 4, 6, 8, 10]
```

this 指向汇总


### 数值的拓展

- 二进制和八进制表示法：ES6 提供了二进制和八进制数值的新的写法，分别用前缀0b（或0B）和0o（或0O）表示
- `Number.isFinite()`：用来检查一个数值是否为有限的（finite），即不是 Infinity
- `Number.isNaN()`：用来检查一个值是否为 NaN
- `Number.parseInt()`/`Number.parseFloat()`：ES6 将全局方法 parseInt() 和 parseFloat() 移到了 Number 对象上
- `Number.isInteger()`：用来判断一个值是否为整数
- `Number.isSafeInteger()`：用来判断一个整数是否在 JavaScript 能表示的安全范围内
- `Math.trunc()`：用于去除一个数的小数部分，返回整数部分
- `Math.sign()`：用来判断一个数到底是正数、负数、还是零。它会返回五种值，正数返回+1；负数返回-1；0返回0；-0返回-0；其他值返回NaN
- `Math.cbrt()`：用于计算一个数的立方根
- `Math.hypot()`：用于计算所有参数的平方和的平方根

### 函数的拓展

- 函数参数默认值：ES6 允许为函数的参数设置默认值，即直接写在参数定义的后面。但是此时length属性将失真，不再包括指定默认值的参数

```javascript
// ES5 传统写法
function log(x, y) {
  y = y || 'World'
  console.log(x, y)
}

// ES6 写法
function log(x, y = 'World') {
  console.log(x, y)
}
```

- rest 参数：用于获取函数的多余参数，这样就不需要使用 arguments 对象了

```javascript
// ES5 传统写法
function add() {
  var sum = 0
  for (var i = 0; i < arguments.length; i++) {
    sum += arguments[i]
  }
  return sum
}

// ES6 写法
function add(...args) {
  var sum = 0
  for (var i = 0; i < args.length; i++) {
    sum += args[i]
  }
  return sum
}
```

- 严格模式：从 ES5 开始，函数内部可以设定为严格模式。ES7/ES2016 做了一点修改，规定只要函数参数使用了默认值、解构赋值、或者扩展运算符，那么函数内部就不能显式设定为严格模式，否则会报错
- name 属性：函数的 name 属性返回函数的函数名
- 尾调用优化：尾调用（Tail Call）是函数式编程的一个重要概念，本身非常简单，就是指某个函数的最后一步是调用另一个函数。ES6 对函数的尾调用做了优化，尾调用优化可以节省内存，提高性能

### 数组的拓展

- 扩展运算符（spread）：扩展运算符（spread）是三个点（`...`）。它好比 rest 参数的逆运算，将一个数组转为用逗号分隔的参数序列

```javascript
// ES5 传统写法
console.log.apply(console, [1, 2, 3]) // 输出：1 2 3

// ES6 写法
console.log(...[1, 2, 3]) // 输出：1 2 3

// 应用1：复制数组
const arr1 = [1, 2, 3]
const arr2 = [...arr1] // 输出：[1, 2, 3]
// 应用2：合并数组
const arr3 = [4, 5, 6]
const arr4 = [...arr1, ...arr3] // 输出：[1, 2, 3, 4, 5, 6]
// 应用3：与解构赋值结合
const [first, ...rest] = [1, 2, 3, 4, 5] // 输出：first: 1, rest: [2, 3, 4, 5]
// 应用4：字符串
console.log(...'hello') // 输出：h e l l o
```

### 对象的拓展

- 属性简写：ES6 允许在对象中直接写入变量和函数，作为对象的属性和方法，这被称为属性简写（Property Shorthand）

```javascript
// ES5 传统写法
const name = '张三'
const age = 18
const obj = {
  name: name,
  age: age,
  say: function () {
    console.log('Hello, my name is ' + this.name + ', I am ' + this.age + ' years old.')
  },
}

// ES6 写法
const name = '张三'
const age = 18
const obj = {
  name,
  age,
  say() {
    console.log('Hello, my name is ' + this.name + ', I am ' + this.age + ' years old.')
  },
}
```

- 属性名表达式：ES6 允许字面量定义对象时，将表达式放在括号内，作为对象的属性名。但是注意，属性名表达式与简洁表示法，不能同时使用，会报错

```javascript
// ES5 传统写法
const obj = { name: '张三', age: 18, say() {} }

// ES6 写法
const key = 'name'
const obj = { [key]: '张三', ['age']: 18, ['say']() {} }
```

- 方法的name属性：函数的name属性，返回函数名。对象方法也是函数，因此也有name属性
- super关键字：this关键字总是指向函数所在的当前对象，ES6 又新增了另一个类似的关键字super，指向当前对象的原型对象
- 对象的扩展运算符：扩展运算符（spread）用于取出参数对象的所有可遍历属性，拷贝到当前对象之中

### Symbol

- Symbol 是 ES6 引入的一种新的原始数据类型，表示独一无二的值。即使是用相同的参数创建的两个 Symbol 值也不会相等
- Symbol 值通过 `Symbol()` 函数生成。可以在括号内传入一个字符串参数，表示对 Symbol 的描述，主要是为了在控制台显示，或者转为字符串时，比较容易区分 `Symbol([description])`

```javascript
const s1 = Symbol()
const s2 = Symbol()
console.log(s1 === s2) // 输出：false
```

- Symbol 作为属性名，不能使用点运算符

```javascript
const name = Symbol('name')
const obj = {}
obj.name = '张三' // 普通属性 {name: '张三'}
obj[name] = '张三' // Symbol 属性 {Symbol(name): '张三'}
```

- Symbol 作为属性名，该属性不会出现在 for...in、for...of 循环中，也不会被 Object.keys()、Object.getOwnPropertyNames()、JSON.stringify() 返回。但是，它也不是私有属性，有一个 Object.getOwnPropertySymbols 方法，可以获取指定对象的所有 Symbol 属性名

```javascript
const obj = {
  [Symbol('name')]: '张三',
  [Symbol('age')]: 18,
}
console.log(Object.getOwnPropertySymbols(obj)) // 输出：[Symbol(name), Symbol(age)]
```

### Set 和 Map 数据结构

Set

- ES6 提供了新的数据结构 Set。它类似于数组，但是成员的值都是唯一的，没有重复的值。
- 创建Set：`new Set([1, 2, 3, 4, 4])`，Set 结构不会添加重复的值，所以可以直接利用 Set 去重

```javascript
const s = new Set()
console.log(s) // 输出：Set(0) {}

const s = new Set([1, 2, 3, 4, 5])
console.log(s) // 输出：Set(5) {1, 2, 3, 4, 5}
```

- Set 实例的属性和方法
  - `Set.prototype.constructor` 构造函数，默认就是 Set 函数
  - `Set.prototype.size` 返回 Set 实例的成员总数
  - `Set.prototype.add(value)` 添加某个值，返回 Set 结构本身
  - `Set.prototype.delete(value)` 删除某个值，返回一个布尔值，表示删除是否成功
  - `Set.prototype.has(value)` 返回一个布尔值，表示该值是否为 Set 的成员
  - `Set.prototype.clear()` 清除所有成员，没有返回值

```javascript
const set = new Set([1, 2, 3, 4, 5])
console.log(set) // Set { 1, 2, 3, 4, 5 }
console.log(set.size) // 5

set.add(6)
console.log(set) // Set { 1, 2, 3, 4, 5, 6 }

set.delete(6)
console.log(set) // Set { 1, 2, 3, 4, 5 }

console.log(set.has(5)) // true

set.clear()
console.log(set) // Set {}
```

- Set 结构的遍历：遍历顺序就是插入顺序
  - `Set.prototype.keys()` 返回键名的遍历器
  - `Set.prototype.values()` 返回键值的遍历器
  - `Set.prototype.entries()` 返回键值对的遍历器
  - `Set.prototype.forEach()` 使用回调函数遍历每个成员
  - 注意：由于 Set 结构没有键名，只有键值（或者说键名和键值是同一个值），所以keys方法和values方法的行为完全一致

```javascript
const set = new Set(['red', 'green', 'blue'])
for (let item of set.keys()) {
  console.log(item) // red green blue
}
for (let item of set.values()) {
  console.log(item) // red green blue
}
for (let item of set.entries()) {
  console.log(item) // ["red", "red"] ["green", "green"] ["blue", "blue"]
}
set.forEach((key, value) => {
  console.log(key, value) // red red green green blue blue
})
// 由于 Set 结构没有键名，这意味着，可以省略values方法，直接用for...of循环遍历 Set
for (let item of set) {
  console.log(item) // red green blue
}
```

- 利用 Set 数据结构去重

```javascript
const arr = [1, 2, 3, 4, 5, 5, 6, 6, 7, 8, 9, 9]
const set = new Set(arr)
const newArr = [...set]
console.log(newArr) // 输出：[1, 2, 3, 4, 5, 6, 7, 8, 9]
```

Map

- ES6 提供了 Map 数据结构。它类似于对象，也是键值对的集合，但是“键”的范围不限于字符串，各种类型的值（包括对象）都可以当作键。也就是说，Object 结构提供了“字符串—值”的对应，Map 结构提供了“值—值”的对应，是一种更完善的 Hash 结构实现。如果你需要“键值对”的数据结构，Map 比 Object 更合适
- 创建 Map：`new Map([[key1, value1], [key2, value2]])`

```javascript
const map = new Map()
console.log(map) // 输出：Map(0) {}

const map = new Map([
  ['name', '张三'],
  ['age', 18],
])
console.log(map) // 输出：Map(2) { 'name' => '张三', 'age' => 18 }
```

- Map 实例的属性和方法
  - `Map.prototype.size` 返回 Map 实例的成员总数
  - `Map.prototype.set(key, value)` 设置键名 key 对应的键值为 value，然后返回整个 Map 结构。如果 key 已经有值，则键值会被更新，否则就新生成该键
  - `Map.prototype.get(key)` 读取 key 对应的键值，如果找不到 key，返回 undefined
  - `Map.prototype.has(key)` 返回一个布尔值，表示某个键是否在当前 Map 对象之中
  - `Map.prototype.delete(key)` 删除某个键，返回 true。如果删除失败，返回 false
  - `Map.prototype.clear()` 清除所有成员，没有返回值

```javascript
const map = new Map()
console.log(map.size) // 输出：0

map.set('name', '张三')
map.set('age', 18)
console.log(map) // 输出：Map(2) { 'name' => '张三', 'age' => 18 }

console.log(map.get('name')) // 输出：张三
console.log(map.get('age')) // 输出：18
console.log(map.get('gender')) // 输出：undefined

console.log(map.has('name')) // 输出：true
console.log(map.has('gender')) // 输出：false

map.delete('name')
console.log(map) // 输出：Map(1) { 'age' => 18 }
console.log(map.has('name')) // 输出：false

map.clear()
console.log(map) // 输出：Map(0) {}
```

- Map 结构的遍历：遍历顺序就是插入顺序
  - `Map.prototype.keys()` 返回键名的遍历器
  - `Map.prototype.values()` 返回键值的遍历器
  - `Map.prototype.entries()` 返回键值对的遍历器
  - `Map.prototype.forEach()` 使用回调函数遍历每个成员

```javascript
const map = new Map([
  ['name', '张三'],
  ['age', 18],
])
for (let item of map.keys()) {
  console.log(item) // name age
}
for (let item of map.values()) {
  console.log(item) // 张三 18
}
for (let item of map.entries()) {
  console.log(item) // ["name", "张三"] ["age", 18]
}
map.forEach((value, key) => {
  console.log(key, value) // name 张三 age 18
})
```

- Map 结构转为数组结构，比较快速的方法是使用扩展运算符（...）

```javascript
const map = new Map([
  ['name', '张三'],
  ['age', 18],
])
const arr = [...map]
console.log(arr) // 输出：[["name", "张三"], ["age", 18]]
```

## ES7/ES2016 新特性

### 幂运算符 `**`

用幂运算符 `**` 替代 `Math.pow`，支持链式运算与赋值简写

```javascript
// 传统写法
const square = Math.pow(2, 3) // 8
const cube = Math.pow(3, 3) // 27

// 幂运算符
const square = 2 ** 3 // 8
const cube = 3 ** 3 // 27

// 链式运算
const result = 2 ** (3 ** 2) // 512（3 的平方再乘以 2）

// 赋值简写
let num = 2
num **= 3 // 8（相当于 num = num ** 3）
```

### `Array.prototype.includes()`

判断数组是否包含指定元素，修复 `indexOf` 无法判断 `NaN` 的问题

```javascript
const arr = [1, 2, 3, NaN]
console.log(arr.indexOf(NaN)) // -1（无法判断 NaN）
console.log(arr.includes(NaN)) // true
```

### 大整数 BigInt

由于 JavaScript 中 Number 类型的数字精度有限，当需要表示大于 2^53 - 1 的整数时，JavaScript 会失去精度。因此 ES11 新增了一种数据类型 BigInt，用于表示大于 2^53 - 1 的整数，可以表示任意大的整数，末位加 n 表示 BigInt 类型

```javascript
// 基本用法
let a = 123n
let b = BigInt(123)
console.log(a, b) // 输出 123 123n

// 注意事项
const arr = [1n, 2n, 3n]
console.log(arr) // 输出 [1n, 2n, 3n]
```

- 空值合并运算符：`??`，用于为变量设置默认值，如果变量为 null 或 undefined，则返回默认值，否则返回变量值
